## 4. 详细问题分析

### 4.1 巨型文件: risk_calculation_engine.py

**文件路径**: `src/risk/models/risk_calculation_engine.py`  
**当前规模**: 2,472行, 98.01KB  
**问题严重度**: 🔴 **极高**

#### 问题描述

1. **文件过大**: 2,472行是推荐上限(800行)的**309%**
2. **维护困难**: 如此大的文件极难维护和理解
3. **测试困难**: 难以进行充分的单元测试
4. **协作冲突**: 多人修改容易产生冲突
5. **性能影响**: 文件加载和解析耗时

#### 拆分建议（紧急🔴）

**建议拆分为5-6个模块**:

```
models/
├── risk_calculation_engine.py      # 核心引擎 (预计400行)
├── position_risk_calculator.py     # 仓位风险计算
├── volatility_risk_calculator.py   # 波动率风险计算
├── liquidity_risk_calculator.py    # 流动性风险计算
├── correlation_risk_calculator.py  # 相关性风险计算
├── market_risk_calculator.py       # 市场风险计算
└── operational_risk_calculator.py  # 操作风险计算
```

**预期效果**:
- risk_calculation_engine.py: 2,472行 → 400行 (-84%)
- 提升可维护性: +80%
- 提升测试覆盖: +60%
