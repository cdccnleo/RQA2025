## 🔧 拆分方法论

### 标准拆分流程

基于已完成的12个成功拆分示例，标准流程：

**1. 分析阶段**（10%时间）
- 阅读完整代码
- 识别主要职责
- 绘制依赖关系图
- 确定拆分边界

**2. 规划阶段**（10%时间）
- 设计新模块结构
- 定义模块职责
- 规划导入关系
- 评估风险点

**3. 提取阶段**（40%时间）
- 提取数据模型
- 提取工具函数
- 提取业务逻辑
- 创建新模块文件

**4. 更新阶段**（20%时间）
- 更新原文件导入
- 创建入口文件
- 更新__init__.py
- 调整依赖关系

**5. 测试阶段**（20%时间）
- 导入测试
- 单元测试
- 集成测试
- 回归测试

### 三种拆分模式

**模式1: 管理器委托模式**
```
原文件（1000行）→ 
  - models.py（数据模型）
  - manager1.py（管理器1）
  - manager2.py（管理器2）
  - core.py（核心逻辑）
  - 原文件（入口，100行）
```

**模式2: 入口点模式**
```
原文件（800行）→
  - component1.py（组件1）
  - component2.py（组件2）
  - utils.py（工具函数）
  - 原文件（入口+协调，150行）
```

**模式3: 技术栈分离模式**
```
原文件（1500行）→
  - models.py（Pydantic模型）
  - protocols.py（协议定义）
  - implementation.py（具体实现）
  - 原文件（入口，100行）
```
