## 🚨 重要更新：报告数量控制与命名优化

### 报告数量控制策略
- **每个目录最多保留10个最新报告**

- **使用语义化命名**
- **添加版本控制而非时间戳**

## 📝 命名规范（更新版）

### 1. 文件命名规范（新标准）

#### 基本格式（移除日期）
```
{category}_{type}_{subject}_{version}.{extension}
```

#### 命名组件说明
- **category**: 报告类别（project, technical, business, operational, research）
- **type**: 报告类型（progress, completion, analysis, test, audit, etc.）
- **subject**: 报告主题（deployment, performance, security, etc.）
- **version**: 版本标识（v1, v2, latest, final）
- **extension**: 文件扩展名（.md, .json, .html）

#### 命名示例（新标准）
```
project_progress_deployment_latest.md
technical_test_performance_v2.json
business_analysis_trading_final.md
operational_monitoring_system_v1.md
research_ml_integration_latest.md
```

#### 禁止的命名模式
- ❌ `project_progress_deployment_20250727_v1.md`
- ❌ `technical_test_performance_20250727.json`
- ❌ `business_analysis_trading_20250727.md`
- ❌ `operational_monitoring_system_20250727.md`

### 2. 版本控制策略

#### 版本标识规则
- **latest**: 最新版本（推荐用于当前活跃报告）
- **v1, v2, v3**: 主要版本更新
- **final**: 最终版本（项目完成或功能稳定）
- **draft**: 草稿版本（开发中）

#### 版本更新规则
- 重大变更：增加主版本号（v1 → v2）
- 功能增强：增加次版本号（v1.1 → v1.2）
- 问题修复：增加修订版本号（v1.1.1 → v1.1.2）

## 📊 报告分类标准

### 1. 项目报告 (project/)
- **进度报告**: 项目里程碑、任务完成情况
- **完成报告**: 项目或功能模块完成总结
- **部署报告**: 系统部署和上线报告
- **架构报告**: 系统架构设计和变更报告

### 2. 技术报告 (technical/)
- **测试报告**: 单元测试、集成测试、性能测试结果
- **性能报告**: 系统性能分析和优化报告
- **安全报告**: 安全审计、漏洞扫描、风险评估
- **质量报告**: 代码质量、技术债务分析
- **优化报告**: 性能优化、架构优化总结

### 3. 业务报告 (business/)
- **分析报告**: 业务数据分析、趋势分析
- **交易报告**: 交易策略、交易执行分析
- **回测报告**: 策略回测结果、回测一致性验证
- **合规报告**: 监管合规、风险控制报告

### 4. 运维报告 (operational/)
- **监控报告**: 系统监控、告警分析
- **部署报告**: 环境部署、配置变更
- **通知报告**: 系统通知、团队沟通
- **维护报告**: 系统维护、故障处理

### 5. 研究报告 (research/)
- **机器学习集成**: ML模型集成、AI功能开发
- **深度学习**: 神经网络、深度学习应用
- **强化学习**: 强化学习算法、策略优化
- **持续优化**: 自动化优化、智能调优

## 📋 报告管理规范（更新版）

### 1. 数量控制策略
- **每个目录最多保留10个最新报告**
- **自动归档超过30天的报告**
- **合并相似功能的报告**
- **定期清理临时和测试报告**

### 2. 版本控制
- 使用语义化版本号（v1.0.0）
- 重大变更增加主版本号
- 功能增强增加次版本号
- 问题修复增加修订版本号

### 3. 归档策略
- **自动归档**: 超过30天的报告自动移至archive/
- **年度归档**: 每年底将旧报告移至archive/YYYY/
- **废弃归档**: 不再使用的报告移至archive/deprecated/
- **保留策略**: 重要报告永久保留，临时报告定期清理

### 4. 质量要求
- **内容准确性**: 数据准确，分析合理
- **格式规范性**: 遵循模板规范
- **及时性**: 重要事件24小时内生成报告
- **完整性**: 包含必要的元数据和附录
