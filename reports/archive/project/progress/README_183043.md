# Testing - Quality

## 📋 目录说明

此目录用于存放testing相关的quality报告。

## 📁 文件组织

- 按日期命名：`quality_内容_YYYYMMDD.md`
- 按版本命名：`quality_v版本号_YYYYMMDD.md`
- 按项目命名：`quality_项目名_YYYYMMDD.md`

## 📝 报告模板

请参考 `assets/templates/` 目录下的报告模板。

## 📊 质量要求

1. 报告标题清晰明确
2. 包含执行时间和版本信息
3. 提供详细的分析结果
4. 包含改进建议和后续计划
