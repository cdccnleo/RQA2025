```bash

# 收集并输出到文件

pytest tests/unit/infrastructure/ --co
```

coverage run -m pytest tests/

coverage html

## 📝 测试编写最佳实践

### 测试文件组织
```
tests/unit/[module]/
├── test_[component].py           # 单元测试
├── test_[component]_integration.py  # 集成测试
└── test_[component]_functional.py   # 功能测试
```

### 测试用例模板（Pytest风格）

#### 1. 基础单元测试
```python
import pytest
from src.module.component import MyClass

class TestMyClass:
    """MyClass的测试类"""
    
    @pytest.fixture
    def instance(self):
        """创建测试实例"""
        return MyClass()
    
    def test_basic_functionality(self, instance):
        """测试基本功能"""
        # Arrange
        input_data = "test"
        
        # Act
        result = instance.process(input_data)
        
        # Assert
        assert result is not None
        assert isinstance(result, str)
```

#### 2. 参数化测试
```python
@pytest.mark.parametrize("input,expected", [
    ("test1", "result1"),
    ("test2", "result2"),
    ("test3", "result3"),
])
def test_with_parameters(input, expected):
    """参数化测试"""
    result = process_function(input)
    assert result == expected
```

#### 3. 异常测试
```python
def test_exception_handling():
    """测试异常处理"""
    with pytest.raises(ValueError) as exc_info:
        invalid_function()
    
    assert "expected error message" in str(exc_info.value)
```

#### 4. Mock测试
```python
from unittest.mock import Mock, patch

def test_with_mock():
    """使用mock的测试"""
    mock_dependency = Mock()
    mock_dependency.method.return_value = "mocked"
    
    result = function_using_dependency(mock_dependency)
    
    assert result == "expected"
    mock_dependency.method.assert_called_once()
```

### 测试命名规范
- **测试文件**: `test_[module_name].py`
- **测试类**: `Test[ClassName]`
- **测试方法**: `test_[what_is_being_tested]`
- **使用描述性名称**: `test_get_user_returns_none_when_not_found`

## 💡 常见问题和解决方案

### Q1: 如何修复测试收集错误？
**A**: 
1. 查看错误信息，确定错误类型
2. 导入错误：检查import路径和模块存在性
3. 语法错误：使用linter检查代码
4. 依赖错误：安装缺失的依赖包

### Q2: 如何修复Result对象相关测试？
**A**: 
```python
# 错误代码
self.assertTrue(result.success)
self.assertIsNone(result.error)

# 修复方案
self.assertGreater(result.row_count, 0)
# 或
self.assertTrue(len(result.data) > 0)
```

### Q3: 测试运行很慢怎么办？
**A**: 
```bash
# 使用pytest-xdist并行运行
pytest tests/ -n auto

# 只运行特定模块
pytest tests/unit/infrastructure/utils/ -n auto
```

### Q4: 如何快速定位失败测试？
**A**: 
```bash
# 只运行失败的测试
pytest --lf

# 先运行失败的测试
pytest --ff

# 详细输出
pytest -vv
```

### Q5: 如何生成覆盖率报告？
**A**: 
```bash
# 生成HTML报告
pytest tests/ --cov=src --cov-report=html

# 在浏览器中打开
# htmlcov/index.html
```
