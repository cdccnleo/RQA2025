## 💡 关键决策记录

### 决策1: API Gateway保留aiohttp版本

**选项**:
- A: 保留aiohttp版本 (1,137行，功能完整，正在使用)
- B: 保留Flask版本 (526行，代码简洁，未使用)

**决策**: 选择A

**理由**:
1. aiohttp版本正在被使用（有别名文件引用）
2. 功能更完整（限流、熔断、健康检查）
3. Flask版本有依赖问题（缺少service_communicator）
4. 删除未使用版本风险更低

### 决策2: 事件总线保留两个实现

**方案**:
- 保留主实现 + 编排器专用实现
- 删除未使用的独立实现

**理由**:
1. 职责不同：系统级 vs 编排器级
2. 避免重依赖：编排器不需要完整功能
3. 符合设计原则：各司其职，互不干扰

### 决策3: 服务文件统一到integration/services/

**选择**: integration/services/

**理由**:
1. 功能更完整（文件更大）
2. 位置更合理（符合"集成服务层"职责）
3. 架构一致性（与整体设计相符）
