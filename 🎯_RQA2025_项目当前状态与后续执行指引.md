## 💻 测试编写标准

### 标准测试模板

```python
#!/usr/bin/env python
# -*- coding: utf-8 -*-
"""
Trading层 - XXX模块完整测试（Week X）
方案B Month Y任务：深度测试XXX模块
目标：XXX模块从<5%提升到40%+
"""

import pytest
from datetime import datetime
from unittest.mock import Mock, patch

# 导入实际项目代码（使用绝对导入）
from src.trading.xxx.xxx_module import XXXClass

pytestmark = [pytest.mark.timeout(30)]


class TestXXXInstantiation:
    """测试XXX实例化"""
    
    def test_xxx_default_instantiation(self):
        """测试默认实例化"""
        obj = XXXClass()
        assert obj is not None
        # 验证关键属性
        assert hasattr(obj, 'key_attribute')


class TestXXXCoreFunction:
    """测试XXX核心功能"""
    
    @pytest.fixture
    def xxx_obj(self):
        """创建测试对象"""
        return XXXClass()
    
    def test_core_function(self, xxx_obj):
        """测试核心功能"""
        result = xxx_obj.core_function(param1, param2)
        assert result is not None
        # 验证结果
```

### 测试编写原则

1. ✅ **使用绝对导入**: `from src.xxx import YYY`
2. ✅ **测试核心功能优先**: 不追求100%，关注关键业务逻辑
3. ✅ **合理的通过率**: 75-85%即可，不必完美
4. ✅ **清晰的测试组织**: 按功能分类，使用test class
5. ✅ **适当的断言**: 验证关键属性和返回值

## 🔧 常见问题解决

### 问题1: 导入失败
```python
# ❌ 不推荐（相对导入可能失败）
from ...core.xxx import YYY

# ✅ 推荐（绝对导入更可靠）
from src.trading.core.xxx import YYY
```

### 问题2: pytest-cov无数据
```bash
# 单模块覆盖率可能失败
pytest test_xxx.py --cov=src.xxx

# 使用整层覆盖率作为替代
pytest tests/unit/trading/ --cov=src/trading
```

### 问题3: 实例化失败
```python
# 使用try-except处理导入
try:
    from src.xxx import YYY
except ImportError:
    YYY = None

# 在测试中跳过
def test_xxx():
    if YYY is None:
        pytest.skip("YYY not available")
    # 测试代码
```
