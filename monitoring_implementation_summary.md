## 📊 数据流程

### 数据收集流程
1. 用户点击"测试连接"按钮
2. 后端执行真实API调用
3. 记录开始时间
4. 执行API调用
5. 计算响应时间（结束时间 - 开始时间）
6. 保存到监控数据存储
7. 更新数据源状态

### 数据查询流程
1. 前端请求 `/api/v1/data-sources/metrics`
2. 后端从监控数据存储查询真实数据
3. 计算最近7天的平均值
4. 如果没有真实数据，使用估算值
5. 返回给前端显示

## 📝 使用说明

### 查看真实监控数据
1. 测试数据源连接（点击"测试"按钮）
2. 等待测试完成
3. 刷新页面查看更新的性能指标
4. 多次测试后，系统会计算平均值

### 验证真实数据
1. 检查 `data/monitoring/` 目录下的JSON文件
2. 查看浏览器控制台的DEBUG日志
3. 对比多次测试的延迟数据变化

## ⚠️ 注意事项

1. **数据积累**: 真实监控数据需要时间积累，初始阶段可能仍显示估算值
2. **数据量**: 系统自动保留最近30天的数据，更早的数据会被清理
3. **性能影响**: 监控数据收集对性能影响很小（<1ms）
4. **向后兼容**: 如果没有真实数据，系统会自动使用估算值，确保功能正常
