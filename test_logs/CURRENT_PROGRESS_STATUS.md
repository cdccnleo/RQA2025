## 🎯 质量优先原则（严格遵循）

### ✅ **核心原则**

1. **测试通过率优先** (P0 - 最高优先级) ✅

   - **只有在通过率100%后，才考虑提升覆盖率**

2. **核心模块优先** (P1 - 高优先级)
   - 优先确保核心模块的高覆盖率（≥80%）
   - 核心模块必须充分测试

3. **稳定性优先** (P2 - 中优先级)
   - 确保测试稳定可靠
   - 避免间歇性失败

4. **质量优先** (贯穿始终) ✅
   - 宁可覆盖率略低，也要确保代码质量
   - 测试质量比数量更重要

### ✅ **严格验证流程**

```
Phase 1: 测试通过率验证 (P0) → 必须100%通过
    ↓ (只有100%通过后)
Phase 2: 代码覆盖率验证 (P1) → 目标≥85%（Risk层≥90%）
    ↓ (覆盖率达标后)
Phase 3: 核心模块验证 (P1) → 核心模块≥80%
    ↓ (核心模块达标后)
Phase 4: 稳定性验证 (P2) → 测试稳定可靠
    ↓ (稳定性验证通过后)
Phase 5: 投产准备验证 → 确认所有指标达标
```

**重要**: 每个阶段必须完全通过，才能进入下一个阶段。
