## 💎 核心经验与最佳实践

### 成功经验

1. **系统性诊断优先** ⭐⭐⭐⭐⭐
   - 花20%时间分析问题
   - 找到根本原因再动手
   - 避免盲目修复

2. **接口优先原则** ⭐⭐⭐⭐⭐
   - 先读源码理解接口
   - 再编写符合接口的测试
   - 避免假设导致的失败

3. **Mock后端策略** ⭐⭐⭐⭐
   - 创建符合接口的轻量Mock
   - 快速、可控、隔离
   - 避免依赖真实存储

4. **增量迭代改进** ⭐⭐⭐⭐⭐
   - 每轮聚焦1-2个模块
   - 验证效果再推进
   - 持续积累成果

5. **及时清理冗余** ⭐⭐⭐⭐
   - 删除失败的旧测试
   - 保持代码库整洁
   - 提升维护性

### 避坑指南

❌ **不要假设接口** - 先读代码
❌ **不要过度Mock** - 适当使用集成测试
❌ **不要测试私有方法** - 测试公共API
❌ **不要忽视清理** - 及时删除冗余
❌ **不要孤立推进** - 持续验证效果

### 可复用模式

```python
# 模式1: 创建符合接口的Mock
class MockXXX:
    def __init__(self, ...):
        # 实现所有接口方法
        pass
    
    def required_method(self):
        # 完整实现，不留抽象
        return expected_result

# 模式2: Duck typing验证
assert result is not None
assert hasattr(result, 'required_attr')
# 而不是: assert isinstance(result, SpecificClass)

# 模式3: 兼容性初始化
def __init__(self, optional_param=None):
    self._param = optional_param or default
```
