## 💡 关键经验

### 发现1：文件名不匹配是最严重问题

**案例**:
```
导入：api_documentation_enhancer_refactored.py
0%文件：api_documentation_enhancer.py

结果：测试了错误的文件！
```

**解决方案**:
- 检查0%文件的实际路径
- 确保测试导入正确的文件
- 避免_refactored后缀混淆

### 发现2：pytest.skip()隐藏了真实问题

**问题**:
```python
try:
    from xxx import XXX
    assert XXX is not None
except Exception:
    pytest.skip("测试跳过")
```

**效果**:
- 测试显示"跳过"而非"失败"
- 真实的ImportError被隐藏
- 未贡献任何覆盖

**改进**:
```python
# 直接导入，让错误暴露
from xxx import XXX
assert XXX is not None
```

### 发现3：类名必须精确匹配

**错误**:
```python
DocumentationSearchEngine  # 猜测的类名
TestGenerationCoordinator  # 猜测的类名
```

**正确**:
```python
SearchEngine              # 实际的类名
APITestSuiteCoordinator   # 实际的类名
```

**方法**:
```bash
# 使用grep查找实际类名
grep "class.*SearchEngine" src/.../search_engine.py
```

## 💡 方法论验证

### ✅ 有效的方法

1. **grep查找实际类名**
   ```bash
   grep "class.*XXX" src/.../*.py
   ```

2. **直接导入，暴露错误**
   ```python
   from xxx import XXX  # 不用try-except包裹
   ```

3. **添加深度测试**
   ```python
   obj.method(args)  # 真正调用方法
   assert result == expected  # 验证结果
   ```

### ⚠️ 经验教训

1. **不要猜测类名**
   - 必须grep查找实际定义
   - 避免DocumentationXXX这种猜测

2. **不要过度使用pytest.skip()**
   - 只在明确情况下skip
   - 让真实错误暴露出来

3. **单独测试仅供诊断**
   - 不能作为覆盖率依据
   - 必须运行完整测试验证
