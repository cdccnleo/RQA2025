# Health模块测试覆盖率统计说明报告

## 🔍 问题：为何Health测试统计包含Utils模块？

### 问题描述

用户发现：运行Health模块测试时，覆盖率报告中包含了`src/infrastructure/utils/`的文件。

**这是为什么？**

---

## 📊 原因解释

### 根本原因：代码依赖关系

```
┌─────────────────────────────────────────────┐
│ Health模块代码 (src/infrastructure/health/) │
│             ↓ import                        │
│ Utils工具类 (src/infrastructure/utils/)     │
│             ↓ 被执行                        │
│ Pytest-cov统计所有被执行的代码               │
│             ↓ 包含                          │
│ 覆盖率报告 = Health代码 + Utils依赖          │
└─────────────────────────────────────────────┘
```

### pytest-cov的行为

**默认行为**: 
- `pytest-cov`会统计**所有被测试执行到的代码**
- 包括主模块代码
- **也包括被导入的依赖模块代码**
- 这反映了真实的系统覆盖情况

**为什么这样设计**？
- 依赖模块的质量同样影响系统
- 完整的覆盖率应该包含所有相关代码
- 这是更保守、更全面的评估

## 📋 统计方法说明

### 如何得到纯Health统计？

**方法**: 从JSON覆盖率数据中过滤

```python
# 读取覆盖率JSON
with open('coverage.json') as f:
    data = json.load(f)

# 只保留health文件，排除utils
health_only = {}
for filepath, file_data in data['files'].items():
    if 'infrastructure/health' in filepath:
        if 'infrastructure/utils' not in filepath:
            health_only[filepath] = file_data

# 计算纯Health覆盖率
total_lines = sum(file['summary']['covered_lines'] + 
                  file['summary']['missing_lines'] 
                  for file in health_only.values())
covered_lines = sum(file['summary']['covered_lines'] 
                    for file in health_only.values())
coverage = covered_lines / total_lines * 100
```

### 为什么配置文件omit无效？

**原因**:
- `omit`配置主要用于排除不想测量的源文件
- 但如果这些文件已经在`source`路径下被执行
- 某些版本的pytest-cov仍会统计

**解决方案**:
- 手动从JSON数据中过滤（本项目采用）
- 或使用`--cov`时不指定utils路径
- 或升级pytest-cov版本尝试更精确控制
