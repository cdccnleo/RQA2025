# 如何运行基础设施层新增测试

## 🚀 快速开始


## 📊 生成覆盖率报告

### 方法1: 使用pytest-cov生成HTML报告

```bash
# 生成覆盖率HTML报告
pytest tests/unit/infrastructure/ \
    --cov=src/infrastructure \
    --cov-report=html:test_logs/htmlcov \
    -n auto

# 查看报告
start test_logs/htmlcov/index.html  # Windows
# 或
open test_logs/htmlcov/index.html   # macOS/Linux
```

### 方法2: 生成控制台覆盖率报告

```bash
pytest tests/unit/infrastructure/ \
    --cov=src/infrastructure \
    --cov-report=term-missing \
    -n auto
```

## 🔄 CI/CD集成

### 添加到CI流水线

```yaml
# .github/workflows/infrastructure-tests.yml
name: Infrastructure Tests

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Set up Python
        uses: actions/setup-python@v2
        with:
          python-version: '3.9'
      - name: Install dependencies
        run: |
          pip install -r requirements.txt
      - name: Run infrastructure tests
        run: |
          pytest tests/unit/infrastructure/ \
            --cov=src/infrastructure \
            --cov-report=xml \
            --cov-fail-under=85 \
            -n auto
      - name: Upload coverage
        uses: codecov/codecov-action@v2
```

## 💡 最佳实践

### 1. 运行测试前
```bash
# 确保环境激活
conda activate rqa

# 确保依赖安装
pip install pytest pytest-cov pytest-xdist
```

### 2. 日常开发
```bash
# 开发新功能时，只运行相关模块测试
pytest tests/unit/infrastructure/api/ -v

# 提交代码前，运行全部测试
pytest tests/unit/infrastructure/ -n auto
```

### 3. 调试失败测试
```bash
# 显示详细错误信息
pytest tests/unit/infrastructure/ -vv --tb=long

# 进入调试模式
pytest tests/unit/infrastructure/ --pdb
```

## ❓ 常见问题

### Q1: 测试运行很慢怎么办？
**A**: 使用并行执行 `-n auto` 参数，可以提速3-5倍。

### Q2: 如何只运行某个测试类？
**A**: 使用 `-k` 参数，例如：
```bash
pytest tests/unit/infrastructure/ -k "TestServiceStatus"
```

### Q3: 如何查看测试覆盖了哪些代码？
**A**: 生成HTML覆盖率报告，可以逐行查看：
```bash
pytest --cov=src/infrastructure --cov-report=html
```

### Q4: 测试失败了怎么办？
**A**: 
1. 查看详细错误信息：`pytest -vv --tb=long`
2. 检查是否有依赖问题
3. 确认测试环境配置正确
