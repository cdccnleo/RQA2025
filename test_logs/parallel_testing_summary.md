## 📝 建议

### 使用并行测试的场景

1. **大规模测试套件** ✅
   - 策略服务层：1,806个测试
   - 风险控制层：770个测试
   - 使用 `-n auto` 自动检测CPU核心数

2. **独立测试用例** ✅
   - 单元测试通常可以并行执行
   - 避免共享状态和资源竞争

3. **覆盖率检查** ✅
   - 快速获取覆盖率数据
   - 适合批量检查多个层级

### 避免并行测试的场景

1. **有导入问题的测试** ⚠️
   - 核心服务层（导入错误）
   - 使用 `-n 0` 串行执行

2. **有共享状态的测试** ⚠️
   - 需要串行执行的集成测试
   - 使用 `-n 0` 串行执行
