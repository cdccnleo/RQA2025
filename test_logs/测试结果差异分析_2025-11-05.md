## 🔍 差异原因分析

### 可能原因1：测试范围不同

**单独测试**:
```python

pytest tests/unit/infrastructure/api
# 覆盖率统计src/infrastructure/api
--cov=src/infrastructure/api
```

**整体测试**:
```python
# 运行所有tests
pytest tests/unit/infrastructure/
# 可能统计了更多的源文件
```

### 可能原因2：代码执行路径不同

**假设**:
- 单独测试：只执行api模块相关代码
- 整体测试：执行所有模块，包括未被测试的依赖代码

## 💡 经验教训

### 关键发现

1. **测试范围问题**
   - 单独模块测试可能统计范围过窄
   - 只覆盖了被测试直接调用的代码
   - 未考虑模块间依赖和间接调用

2. **评估方法错误**
   - 不应依赖单独模块测试结果
   - 应以完整测试为准
   - 单独测试仅作为诊断工具

3. **工作成果**
   - 虽然整体覆盖率提升不大
   - 但测试基础设施得到加强
   - 为后续提升打下基础
