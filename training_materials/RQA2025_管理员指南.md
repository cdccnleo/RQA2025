## 2. 用户管理

### 2.1 创建用户

#### 命令行方式
```bash
# 创建普通用户
python scripts/create_user.py --username trader01 --email trader01@company.com --role trader

# 创建管理员用户
python scripts/create_user.py --username admin01 --email admin@company.com --role admin
```

#### 管理后台方式
1. 登录管理后台
2. 进入"用户管理" -> "创建用户"
3. 填写用户信息：
   - 用户名
   - 邮箱地址
   - 用户角色
   - 初始密码
4. 点击"创建"

### 2.2 管理用户权限

#### 角色定义
- **admin**：系统管理员，拥有所有权限
- **trader**：交易员，可以进行交易操作和数据查看
- **analyst**：分析师，可以查看数据和生成报告
- **viewer**：观察者，只读权限

#### 修改用户角色
```bash
# 修改用户角色
python scripts/update_user.py --username trader01 --role analyst
```

### 2.3 重置用户密码
```bash
# 重置用户密码
python scripts/reset_password.py --username trader01
```

## 4. 监控管理

### 4.1 告警规则配置

#### 添加告警规则
```python
from infrastructure.monitoring.alert_system import AlertRule, AlertLevel, AlertChannel

rule = AlertRule(
    rule_id="high_cpu_usage",
    name="CPU使用率过高",
    condition="cpu_percent > 80",
    level=AlertLevel.WARNING,
    channels=[AlertChannel.EMAIL, AlertChannel.CONSOLE],
    enabled=True,
    cooldown=300
)

alert_system.add_alert_rule(rule)
```

#### 查看告警历史
```bash
# 查看最近24小时的告警
python scripts/view_alerts.py --hours 24

# 按级别筛选告警
python scripts/view_alerts.py --level CRITICAL
```

### 4.2 性能监控

#### 查看系统性能
```bash
# 查看当前性能指标
python scripts/system_status.py

# 查看历史性能数据
python scripts/performance_report.py --days 7
```

#### 设置性能阈值
```python
from infrastructure.monitoring.application_monitor import ApplicationMonitor

monitor = ApplicationMonitor()
monitor.set_alert_threshold('cpu_percent', 75.0)
monitor.set_alert_threshold('memory_percent', 80.0)
```

## 6. 故障排除

### 6.1 常见问题

#### 数据库连接失败
```bash
# 检查数据库状态
systemctl status mysql

# 检查连接配置
cat /etc/rqa2025/config/production_config.json

# 重启数据库
systemctl restart mysql
```

#### 系统响应慢
```bash
# 查看系统负载
top

# 查看数据库连接数
mysqladmin processlist

# 检查磁盘IO
iostat -x 1 10
```

#### 内存不足
```bash
# 查看内存使用
free -h

# 查看进程内存使用
ps aux --sort=-%mem | head -10

# 清理缓存
echo 3 > /proc/sys/vm/drop_caches
```

### 6.2 日志分析

#### 查看系统日志
```bash
# 查看应用日志
tail -f /var/log/rqa2025/app.log

# 查看错误日志
tail -f /var/log/rqa2025/error.log

# 搜索特定错误
grep "ERROR" /var/log/rqa2025/app.log | tail -10
```

#### 分析性能日志
```bash
# 查看慢查询
tail -f /var/log/rqa2025/slow_queries.log

# 分析响应时间
python scripts/analyze_performance.py --log-file /var/log/rqa2025/performance.log
```

## 8. 应急响应

### 8.1 应急预案

#### 严重故障处理流程
1. **立即响应**
   - 通知相关人员
   - 启动应急小组
   - 评估故障影响范围

2. **问题诊断**
   - 查看系统日志
   - 检查系统状态
   - 定位故障根源

3. **恢复操作**
   - 执行备份恢复
   - 重启相关服务
   - 验证系统功能

4. **后续处理**
   - 分析故障原因
   - 改进预防措施
   - 更新应急预案
