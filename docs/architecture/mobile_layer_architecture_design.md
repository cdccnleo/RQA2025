## 🎯 架构概述

### 核心定位

移动端层是RQA2025量化交易系统的移动设备适配层，为移动设备用户提供完整的量化交易体验。通过响应式设计和移动优化，提供便捷、安全、高效的移动端交易服务。

### 设计原则

1. **移动优先**: 专为移动设备优化用户体验
2. **轻量化**: 减少网络传输和本地存储开销
3. **安全性**: 加强移动端安全防护措施
4. **离线支持**: 提供基本的离线功能支持
5. **跨平台**: 支持多种移动设备和平板

## 🏗️ 总体架构

### 架构层次结构

```
src/mobile/                           # 移动端层 ⭐
├── core/                             # 核心功能 ⭐ (2个文件)
│   ├── mobile_trading.py            # 移动端交易主程序
│   └── mobile_trading_main.py       # 模块统一接口
├── trading/                          # 交易功能 ⭐ (1个文件)
│   └── trading.py                   # 交易订单处理
├── portfolio/                        # 投资组合 ⭐ (1个文件)
│   └── portfolio.py                 # 投资组合管理
├── market/                           # 市场数据 ⭐ (1个文件)
│   └── market_data.py               # 市场数据服务
├── user/                             # 用户管理 ⭐ (1个文件)
│   └── user_management.py           # 用户账户管理
└── utils/                            # 工具函数 ⭐ (1个文件)
    └── utilities.py                  # 移动端工具函数
```
