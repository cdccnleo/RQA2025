# 代码审查清单

## 概述

本文档提供了RQA2025项目的代码审查清单，用于确保代码质量、风格一致性和最佳实践的遵循。

## 审查维度

### 🔍 功能正确性
- [ ] 代码实现是否满足需求规格
- [ ] 边界条件和异常情况是否正确处理
- [ ] 单元测试是否覆盖主要功能路径
- [ ] 集成测试是否验证了组件协作

### 🏗️ 架构一致性
- [ ] 是否遵循项目架构模式和设计原则
- [ ] 接口定义是否符合标准规范
- [ ] 依赖关系是否合理，避免循环依赖
- [ ] 是否使用了合适的设计模式

### 🧹 代码质量
- [ ] 是否存在代码重复（使用智能检测工具验证）
- [ ] 圈复杂度是否超过阈值（推荐<10）
- [ ] 认知复杂度是否过高（推荐<15）
- [ ] 方法长度是否合理（推荐<50行）

### 📋 代码风格
- [ ] 是否符合PEP 8编码规范
- [ ] 命名是否清晰准确，遵循驼峰/下划线规范
- [ ] 注释是否完整，包含必要的功能说明
- [ ] 文档字符串是否符合Google/NumPy格式

### 🔒 安全性
- [ ] 是否存在SQL注入、XSS等安全漏洞
- [ ] 敏感信息是否正确处理（不硬编码）
- [ ] 输入验证是否充分
- [ ] 错误信息是否避免泄露敏感信息

### ⚡ 性能优化
- [ ] 是否存在明显的性能瓶颈
- [ ] 大数据结构操作是否优化
- [ ] 数据库查询是否高效
- [ ] 缓存策略是否合理

## 审查流程

### 1. 自动化检查
```bash
# 运行自动化质量检查
python scripts/automated_quality_check.py --output review_report.json src/

# 生成趋势报告
python scripts/generate_quality_trend_report.py --days 7
```

### 2. 手动审查要点

#### 重复代码检查
- 使用智能重复检测工具验证
- 关注相似度>80%的代码段
- 检查是否有提取公共方法的机会

#### 复杂度评估
- 查看圈复杂度分布
- 识别高复杂度方法
- 评估重构价值

#### 接口一致性
- 检查接口定义规范
- 验证继承关系正确性
- 确认抽象方法实现完整

### 3. 审查反馈

#### 问题分类
- **🔴 Critical**: 必须修复的严重问题
- **🟡 Major**: 建议修复的重要问题
- **🟢 Minor**: 可选改进的小问题
- **ℹ️ Info**: 信息性建议

#### 反馈格式
```
## 审查结果

### ✅ 通过项
- 功能实现完整
- 测试覆盖充分

### 🔴 必须修复
1. **高复杂度方法**
   - `complex_method()` 圈复杂度为15
   - 建议拆分为多个小方法

### 🟡 建议改进
1. **代码重复**
   - 第45-55行与第120-130行存在重复
   - 建议提取公共方法

### 🟢 可选优化
1. **性能优化**
   - 考虑添加缓存机制
```

## 质量门禁标准

### 强制要求
- [ ] 单元测试覆盖率 > 80%
- [ ] 克隆组数量 < 50个
- [ ] 质量评分 > 0.7
- [ ] 无严重安全漏洞
- [ ] 通过自动化质量检查

### 推荐标准
- [ ] 圈复杂度 < 10
- [ ] 方法长度 < 30行
- [ ] 注释覆盖率 > 70%
- [ ] 无代码重复

## 常见问题

### Q: 如何处理发现的重复代码？
A: 使用智能检测工具的建议，提取公共方法或创建工具类。

### Q: 复杂度过高的方法如何处理？
A: 拆分为多个小方法，使用策略模式，或添加注释说明复杂度合理性。

### Q: 接口不一致如何解决？
A: 检查基类定义，确保所有实现类都正确实现了抽象方法。

### Q: 性能问题如何识别？
A: 使用性能分析工具，关注大数据结构操作和数据库查询。
