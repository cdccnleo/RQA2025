# 报告文档规范

## 📋 文档时效性原则

### 1. 报告更新策略
- **优先更新现有报告**: 在已有报告基础上更新，避免创建新报告
- **版本连续性**: 保持报告版本号的连续性 (v1.0 → v2.0 → v3.0)
- **时效性标记**: 在报告末尾添加最新更新时间和版本信息

### 2. 避免冗余报告
- **检查现有报告**: 在创建新报告前，检查是否已有相似报告
- **合并相似内容**: 将相似内容合并到现有报告中
- **删除过时报告**: 定期清理过时或重复的报告

### 3. 报告命名规范
- **版本号**: 使用语义化版本号 (v1.0, v2.0, v3.0)
- **日期标记**: 在文件名中包含日期信息
- **功能标识**: 文件名应清晰表达报告内容

## 📊 报告结构标准

### 1. 标准报告结构
```markdown
# 报告标题 vX.0

## 🎯 执行摘要
- 报告目的和范围
- 关键成果和结论

## 📊 详细内容
- 数据和分析
- 图表和说明

## 📝 结论
- 主要发现
- 建议和下一步

## 🆕 最新更新 (vX.0)
- 更新时间和内容
- 版本变更说明

---
*报告更新时间: YYYY-MM-DD*  
*报告版本: vX.0*  
*状态: [状态]* ✅
```

### 2. 更新标记规范
- **🆕 最新更新**: 标记最新更新内容
- **📅 更新时间**: 明确标注更新时间
- **📋 版本信息**: 包含版本号和状态

## 🔄 报告维护流程

### 1. 创建新报告前
1. **检查现有报告**: 搜索是否有相似内容的报告
2. **评估更新需求**: 确定是否需要新报告还是更新现有报告
3. **选择更新策略**: 决定是创建新版本还是更新现有版本

### 2. 更新现有报告
1. **保留历史内容**: 保留原有内容，在末尾添加更新
2. **版本号递增**: 更新版本号 (v4.0 → v5.0)
3. **更新标记**: 添加最新更新章节和时间标记

### 3. 报告清理
1. **定期检查**: 每月检查报告目录
2. **识别冗余**: 找出重复或过时的报告
3. **安全删除**: 删除确认过时的报告

- **状态后缀**: 可添加状态后缀 (如 `_final`, `_draft`)

## 🚫 避免的做法

### 1. 不要创建冗余报告
- ❌ 为相同内容创建多个报告
- ❌ 忽略现有报告直接创建新报告
- ❌ 不检查现有报告就创建新报告

### 2. 不要忽略版本管理
- ❌ 不更新版本号
- ❌ 不标记更新时间
- ❌ 不保留历史内容

### 3. 不要忽略文档索引
- ❌ 创建报告后不更新索引
- ❌ 不记录报告位置和用途
- ❌ 不维护报告目录结构
