# RQA2025 ELK/Grafana 日志可视化对接与仪表盘模板指南

## 1. Filebeat采集配置模板
```yaml
filebeat.inputs:
  - type: log
    enabled: true
    paths:
      - /path/to/logs/automation/*.log
    json.keys_under_root: true
    json.add_error_key: true

output.elasticsearch:
  hosts: ["http://localhost:9200"]
  index: "automation-logs-%{+yyyy.MM.dd}"
```
- 建议将`logs/automation/`目录下所有结构化日志采集到Elasticsearch，按天分索引。

## 2. Elasticsearch字段映射建议
- 关键字段：timestamp、event_type、level、platform、detail
- 建议为event_type/level/platform等字段设置keyword类型，便于聚合与筛选。

## 3. Grafana/Kibana仪表盘模板
- 可在Kibana/Grafana中创建如下典型仪表盘：
  - 日志总量、同步成功率、失败率（折线/饼图）
  - 各平台同步异常分布（饼图/柱状图）
  - 告警次数与类型趋势（折线图）
  - 自愈操作触发频率（柱状图）
  - 近7天/30天自动化健康度（热力图）
- 支持按平台、时间、event_type、level等多维度筛选与分析。
- 仪表盘JSON模板可导出/导入，便于团队复用。

## 4. 导入与对接说明
- Filebeat/Logstash采集日志推送到Elasticsearch后，在Kibana/Grafana中添加数据源。
- 导入仪表盘模板，配置字段映射与可视化。
- 可结合告警规则，自动推送异常到IM/邮件等。

## 5. 持续优化建议
