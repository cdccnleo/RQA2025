# 数据采集映射标准文档

## 1. 文档目的

本文档定义了 RQA2025 项目中数据采集映射的统一标准，确保历史数据采集与日常增量数据采集使用一致的映射关系，提高数据质量和系统可靠性。

## 2. 数据源标准

### 2.1 数据源标识 (source_id)

**标准数据源标识**：
- `akshare_stock_a` - A股数据采集
- `akshare_stock_hk` - 港股数据采集
- `akshare_stock_us` - 美股数据采集

**禁止使用**：
- 动态生成的数据源标识（如 `historical_collection_688702`）
- 包含具体股票代码的数据源标识

## 3. 数据类型标准

### 3.1 数据类型 (data_type)

**标准数据类型**：
- `daily` - 日线数据
- `weekly` - 周线数据
- `monthly` - 月线数据
- `quarterly` - 季度数据
- `yearly` - 年度数据

**禁止使用**：
- 非标准数据类型（如 `price`、`volume`、`fundamental`）
- 模糊或不明确的数据类型

## 4. 字段映射标准

### 4.1 核心字段映射

| 字段名 | 数据类型 | 描述 | 映射要求 |
|--------|----------|------|----------|
| `source_id` | VARCHAR(50) | 数据源标识 | 必须使用标准数据源标识 |
| `symbol` | VARCHAR(20) | 股票代码 | 必须使用标准股票代码格式 |
| `date` | DATE | 数据日期 | 必须使用标准日期格式 |
| `data_type` | VARCHAR(20) | 数据类型 | 必须使用标准数据类型 |
| `open_price` | NUMERIC(15,6) | 开盘价 | 必须映射 |
| `high_price` | NUMERIC(15,6) | 最高价 | 必须映射 |
| `low_price` | NUMERIC(15,6) | 最低价 | 必须映射 |
| `close_price` | NUMERIC(15,6) | 收盘价 | 必须映射 |
| `volume` | BIGINT | 成交量 | 必须映射 |
| `amount` | NUMERIC(20,2) | 成交额 | 必须映射 |
| `pct_change` | NUMERIC(10,4) | 涨跌幅 | 必须映射 |
| `change` | NUMERIC(15,6) | 涨跌额 | 必须映射 |
| `turnover_rate` | NUMERIC(10,4) | 换手率 | 必须映射 |
| `amplitude` | NUMERIC(10,4) | 振幅 | 必须映射 |
| `data_source` | VARCHAR(50) | 数据来源 | 建议设置为 'akshare' |
| `collected_at` | TIMESTAMP WITH TIME ZONE | 采集时间 | 建议设置为 CURRENT_TIMESTAMP |
| `persistence_timestamp` | TIMESTAMP WITH TIME ZONE | 持久化时间 | 建议设置为 CURRENT_TIMESTAMP |

### 4.2 字段映射完整性要求

**必须映射的关键字段**：
- `volume` - 成交量
- `amount` - 成交额
- `pct_change` - 涨跌幅
- `change` - 涨跌额
- `turnover_rate` - 换手率
- `amplitude` - 振幅

**字段完整性标准**：
- 所有关键字段必须有有效值
- 不允许关键字段为空或 NULL
- 数值字段必须使用正确的数据类型和精度

## 5. 数据持久化标准

### 5.1 PostgreSQL 表结构

**标准表结构**：`akshare_stock_data`

**表结构要求**：
- 必须包含所有核心字段
- 必须建立合适的索引
- 必须设置唯一约束确保数据不重复

### 5.2 索引优化

**建议索引**：
- 主键索引：`id`
- 复合索引：`(symbol, date, data_type)`
- 日期索引：`(date DESC)`
- 数据源索引：`(source_id, collected_at DESC)`

### 5.3 数据质量控制

**数据验证规则**：
- 检查数据类型是否符合标准
- 检查关键字段是否完整
- 检查数据格式是否正确
- 检查数据是否重复

## 6. 数据采集流程标准

### 6.1 历史数据采集

**采集标准**：
- 使用标准数据源标识
- 使用标准数据类型
- 确保所有关键字段完整映射
- 批量采集时保持数据一致性

### 6.2 日常增量数据采集

**采集标准**：
- 使用与历史数据相同的映射标准
- 定期增量采集，避免数据丢失
- 采集后验证数据完整性

## 7. 错误处理与异常情况

### 7.1 数据缺失处理

**处理策略**：
- 对于缺失数据，不允许写入 NULL 值
- 建议使用合理的默认值或跳过该条记录
- 记录缺失数据的情况以便后续处理

### 7.2 数据格式错误处理

**处理策略**：
- 检测并修正数据格式错误
- 对于无法修正的错误，记录并跳过
- 定期清理格式错误的数据

## 8. 数据验证与监控

### 8.1 数据验证

**验证标准**：
- 检查数据源标识是否符合标准
- 检查数据类型是否符合标准
- 检查关键字段是否完整
- 检查数据范围是否合理

### 8.2 数据监控

**监控指标**：
- 数据采集成功率
- 数据完整性率
- 数据一致性率
- 数据延迟率

## 9. 实施指南

### 9.1 现有数据清理

**清理步骤**：
1. 识别不符合标准的数据
2. 备份需要清理的数据
3. 删除不符合标准的数据
4. 验证清理结果

### 9.2 新数据采集

**实施步骤**：
1. 使用标准数据源标识
2. 使用标准数据类型
3. 确保字段完整映射
4. 验证数据完整性
5. 定期监控数据质量
