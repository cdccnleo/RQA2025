# 回测数据加载优化总结

## 📋 优化概述

根据量化交易系统的严格要求，已优化回测引擎的数据加载逻辑，移除了所有模拟数据生成，实现了真实数据加载和自动数据采集功能。

## ✅ 完成的优化

### 1. 添加PostgreSQL数据查询函数 ✅

**文件**: `src/gateway/web/postgresql_persistence.py`

**新增函数**: `query_stock_data_from_postgresql()`

**功能**:
- 从PostgreSQL数据库查询股票数据
- 支持按股票代码和日期范围查询
- 返回标准OHLCV格式的DataFrame
- 自动处理日期索引和列名映射

**代码位置**: `src/gateway/web/postgresql_persistence.py` 第289-390行

### 2. 移除模拟数据生成 ✅

**文件**: `src/strategy/backtest/backtest_service.py`

**修改前**:
- 使用 `np.random` 生成模拟价格数据
- 使用随机游走模型生成价格序列
- 完全脱离真实数据

**修改后**:
- ✅ 完全移除所有 `np.random` 调用
- ✅ 移除所有模拟数据生成逻辑
- ✅ 改为从PostgreSQL数据库加载真实数据

### 3. 实现真实数据加载 ✅

**新增方法**: `_load_from_storage()`

**功能**:
- 从PostgreSQL数据库加载股票数据
- 支持多个股票代码批量加载
- 处理日期范围查询
- 错误处理和日志记录

**代码位置**: `src/strategy/backtest/backtest_service.py` 第251-280行

### 4. 实现数据完整性检查 ✅

**新增方法**: `_check_data_completeness()`

**功能**:
- 检查每个股票的数据完整性
- 计算缺失的日期范围
- 识别完全缺失数据的股票
- 计算数据完整度百分比

**代码位置**: `src/strategy/backtest/backtest_service.py` 第282-340行

### 5. 实现自动数据采集 ✅

**新增方法**: 
- `_get_data_source_config()` - 获取数据源配置
- `_trigger_data_collection()` - 触发数据采集

**功能**:
- 当检测到缺失数据时，自动获取数据源配置
- 调用数据采集API进行数据采集
- 支持增量采集模式
- 采集完成后重新加载数据

**代码位置**: 
- `src/strategy/backtest/backtest_service.py` 第342-365行 (`_get_data_source_config`)
- `src/strategy/backtest/backtest_service.py` 第367-410行 (`_trigger_data_collection`)

### 6. 优化主数据加载流程 ✅

**修改方法**: `_load_market_data()`

**新流程**:
1. 确定股票代码列表（从配置或策略配置获取）
2. 从PostgreSQL数据库加载数据
3. 检查数据完整性
4. 如果缺失，自动触发数据采集
5. 重新加载数据
6. 验证最终数据，如果仍为空则抛出异常

**代码位置**: `src/strategy/backtest/backtest_service.py` 第201-250行

## 🎯 核心改进

### 1. 数据来源真实化

- **数据加载**: 从PostgreSQL数据库加载真实数据
- **数据格式**: 标准OHLCV格式（open, high, low, close, volume）
- **数据索引**: 使用日期作为索引，便于时间序列分析

### 2. 自动数据采集

- **触发条件**: 检测到缺失数据时自动触发
- **采集模式**: 增量采集模式（只采集缺失部分）
- **数据源配置**: 自动从配置管理器获取akshare_stock配置
- **采集后处理**: 自动重新加载数据

### 3. 数据完整性保障

- **完整性检查**: 检查每个股票的日期范围完整性
- **缺失识别**: 识别完全缺失和部分缺失的数据
- **缺失范围计算**: 精确计算缺失的日期范围
- **完整度报告**: 计算并记录数据完整度百分比

### 4. 错误处理完善

- **数据库连接失败**: 记录警告，返回空数据
- **数据源配置缺失**: 记录警告，跳过自动采集
- **数据采集失败**: 记录错误，抛出异常
- **最终数据验证**: 如果数据仍为空，抛出异常阻止回测

## 📝 使用说明

### 回测配置要求

回测配置需要包含以下信息：

```python
config = BacktestConfig(
    backtest_id="backtest_001",
    strategy_id="strategy_001",
    start_date="2024-01-01",
    end_date="2024-12-31",
    symbols=["000001", "000002"],  # 可选，如果不指定则从策略配置获取
    initial_capital=1000000.0,
    commission=0.0003
)
```

### 数据源配置要求

需要在 `data/data_sources_config.json` 中配置 `akshare_stock` 数据源：

```json
{
  "id": "akshare_stock",
  "name": "AKShare A股数据",
  "type": "股票数据",
  "enabled": true,
  "config": {
    "enable_incremental": true
  }
}
```

### 工作流程

1. **回测启动**: 用户调用 `run_backtest(backtest_id)`
2. **数据加载**: 系统从PostgreSQL加载数据
3. **完整性检查**: 检查数据是否完整
4. **自动采集**: 如果缺失，自动触发数据采集
5. **重新加载**: 采集完成后重新加载数据
6. **数据验证**: 验证最终数据，如果为空则失败
7. **回测执行**: 使用真实数据进行回测
