## 系统架构

### 分层设计
```
┌─────────────────────────────────────────────────────────────┐
│                    应用层 (Application Layer)                │
├─────────────────────────────────────────────────────────────┤
│                    业务层 (Business Layer)                  │
├─────────────────────────────────────────────────────────────┤
│                    服务层 (Service Layer)                   │
├─────────────────────────────────────────────────────────────┤
│                  基础设施层 (Infrastructure Layer)           │
├─────────────────────────────────────────────────────────────┤
│                    数据层 (Data Layer)                      │
└─────────────────────────────────────────────────────────────┘
```

### 核心模块

#### 基础设施层 (Infrastructure Layer)
- **日志管理**: 统一的日志记录和管理
- **配置管理**: 集中化配置管理
- **错误处理**: 异常处理和重试机制
- **监控系统**: 服务监控、告警管理、自动化任务
- **存储系统**: 数据存储和缓存
- **健康检查**: 系统健康状态监控

#### 数据层 (Data Layer)
- **数据获取**: 实时和历史数据获取
- **数据存储**: 高效的数据存储方案
- **数据预处理**: 数据清洗和特征工程

#### 服务层 (Service Layer)
- **交易服务**: 订单管理和执行
- **风控服务**: 风险控制和监控
- **分析服务**: 数据分析和报告

#### 业务层 (Business Layer)
- **策略管理**: 交易策略的开发和部署
- **投资组合**: 投资组合管理
- **绩效分析**: 交易绩效评估

#### 应用层 (Application Layer)
- **Web界面**: 用户友好的Web界面
- **API接口**: RESTful API服务
- **移动应用**: 移动端应用

## 快速开始

### 环境要求
- Python 3.9+
- Redis 6.0+ (缓存)
- PostgreSQL 12+ (主数据库)
- InfluxDB 2.0+ (时序数据库，新增)
- Docker (可选)
- Node.js 14+ (前端开发，可选)

### 安装步骤

1. **克隆项目**
```bash
git clone https://github.com/your-org/RQA2025.git
cd RQA2025
```

2. **创建虚拟环境**
```bash
conda create -n rqa python=3.9
conda activate rqa
```

3. **安装依赖**
```bash
pip install -r requirements.txt
```

4. **配置环境**
```bash
cp config/config.example.json config/config.json
# 编辑配置文件
```

5. **运行测试**
```bash
python scripts/testing/run_tests.py --module infrastructure --lightweight
```

6. **启动服务**
```bash
python main.py
```
