**问题**：

- 获取股票列表失败时，使用硬编码的默认股票代码：`["000001", "000002", "600000", "600036", "000858"]`

**修复**：
- ✅ 改为采集所有A股股票数据（`stock_list['code'].tolist()`）
- ✅ 获取股票列表失败时，返回空数据并记录错误，不再使用硬编码默认值
- ✅ 添加明确的日志说明：量化交易系统要求使用真实数据

**代码位置**：`src/gateway/web/api.py` 第2353-2363行

**修复前**：
```python
if not symbols:
    stock_list = ak.stock_info_a_code_name()
    symbols = stock_list['code'].head(10).tolist()  # 只采集10只
except Exception as e:
    symbols = ["000001", "000002", "600000", "600036", "000858"]  # 硬编码
```

**修复后**：
```python
if not symbols:
    stock_list = ak.stock_info_a_code_name()
    symbols = stock_list['code'].tolist()  # 采集所有股票
    logger.info(f"未指定股票代码，将采集所有A股数据，共 {len(symbols)} 只股票")
except Exception as e:
    logger.error("无法获取股票列表，量化交易系统要求使用真实数据，不能使用默认股票代码")
    return []  # 返回空数据，不使用硬编码
```

## 🔍 修复原则

### 量化交易系统要求

1. **零模拟数据**：不能使用任何随机生成的模拟数据
2. **零硬编码**：不能使用硬编码的默认股票代码或数据样本
3. **真实数据优先**：必须使用真实数据源接口
4. **明确状态说明**：数据不可用时，明确返回状态说明，不使用模拟数据

### 修复策略

1. **数据采集函数**：
   - 移除所有 `random` 模块的使用

2. **数据样本函数**：

3. **性能指标**：
