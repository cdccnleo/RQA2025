## 使用方法

### 开发环境

在应用启动时自动执行健康检查：
- 应用启动后会自动打印健康检查报告
- 发现问题会在日志中记录详细错误信息

### CI/CD流程

1. **GitHub Actions自动检查**:
   - 代码推送或PR时自动运行
   - 检查路由健康状态
   - 运行相关测试

2. **手动执行检查**:
   ```bash
   # 快速健康检查
   python scripts/check_routes_health.py
   
   # 完整验证
   python scripts/ci_route_verification.py
   ```

3. **本地测试**:
   ```bash
   pytest tests/dashboard_verification/test_route_health_check.py -v
   ```

## 集成到应用启动

在 `api.py` 中已自动集成：
```python
# 执行路由健康检查
try:
    from .route_health_check import print_routes_health_report
    print_routes_health_report(app)
except Exception as e:
    print(f"⚠️ 路由健康检查执行失败: {e}")
```

## 优势

1. **早期发现问题**: 在应用启动时就能发现路由注册问题
2. **详细错误信息**: 提供完整的堆栈跟踪，便于快速定位问题
3. **自动化验证**: 集成到CI/CD流程，确保代码质量
4. **易于维护**: 集中管理预期路由列表，易于更新
