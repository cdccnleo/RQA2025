# 文档维护规范

## 📋 概述

本文档定义了RQA2025项目的文档维护规范，确保文档结构清晰、内容准确、更新及时。

## 🏗️ 文档组织结构

### 按代码结构组织
文档目录结构应与`src`目录结构保持一致：

```
docs/
├── core/                    # 对应 src/core/
├── trading/                 # 对应 src/trading/
├── data/                    # 对应 src/data/
├── models/                  # 对应 src/models/
├── features/                # 对应 src/features/
├── adapters/                # 对应 src/adapters/
├── services/                # 对应 src/services/
├── engine/                  # 对应 src/engine/
├── backtest/                # 对应 src/backtest/
├── ensemble/                # 对应 src/ensemble/
├── tuning/                  # 对应 src/tuning/
├── acceleration/            # 对应 src/acceleration/
├── utils/                   # 对应 src/utils/
├── infrastructure/          # 对应 src/infrastructure/
├── architecture/            # 架构文档
├── configuration/           # 配置文档
├── deployment/              # 部署文档
├── testing/                 # 测试文档
├── reports/                 # 报告文档
├── api/                     # API文档
└── DOCUMENT_INDEX.md        # 文档索引
```

### 每个模块目录结构
每个模块目录应包含：

```
docs/[module]/
├── README.md               # 模块概览（必需）
├── [component].md          # 组件文档
├── [feature].md            # 功能文档
└── [submodule]/            # 子模块目录
    └── README.md           # 子模块概览
```

## 📝 文档编写规范

### 文档模板
每个模块的README.md应包含以下部分：

```markdown
# [模块名]模块文档

## 📋 模块概述
模块功能描述

## 🏗️ 模块结构
代码结构说明

## 📚 文档索引
相关文档链接

## 🔧 使用指南
快速开始和最佳实践

## 📊 架构图
模块架构图

## 🧪 测试
测试相关内容

## 📈 性能指标
性能指标说明

---

**最后更新**: YYYY-MM-DD  
**维护者**: [团队名]  
**状态**: ✅ 活跃维护
```

### 文档命名规范
- 文件名使用小写字母和下划线
- 模块名使用英文
- 文档类型用后缀标识（如：_guide.md, _api.md）

### 文档内容规范
- 使用中文编写
- 代码示例使用Python
- 图表使用ASCII或Mermaid
- 链接使用相对路径

## 🔄 文档更新流程

### 新增功能时
1. 在对应模块目录创建文档
2. 更新模块README.md
3. 更新DOCUMENT_INDEX.md
4. 提交代码和文档

### 修改功能时
1. 更新相关文档
2. 检查文档链接有效性
3. 更新最后修改时间
4. 提交文档变更

### 删除功能时
1. 删除相关文档
2. 更新文档索引
3. 清理无效链接
4. 提交清理变更
