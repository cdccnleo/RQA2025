# 股票名称持久化功能

## 功能概述

股票名称持久化功能允许系统在通过查询接口获取股票名称后，将其保存到配置文件中，以便后续展示时避免重复查询，提升用户体验。

## 实现原理

### 数据格式变更

**旧格式（字符串数组）：**
```json
{
  "custom_stocks": ["002837", "688702"]
}
```

**新格式（对象数组）：**
```json
{
  "custom_stocks": [
    {"code": "002837", "name": "英维克"},
    {"code": "688702", "name": "盛科通信"}
  ]
}
```

### 前端修改

#### 1. 股票池更新逻辑 (`updateSelectedStocksField`)

修改 `web-static/data-sources-config.html` 中的 `updateSelectedStocksField` 函数：
- 从股票标签中提取代码和名称信息
- 将数据保存为对象数组格式：`[{code: "002837", name: "英维克"}]`
- 保持向后兼容性

#### 2. 配置加载逻辑 (`populateStockPoolConfig`)

修改配置加载时对 `custom_stocks` 的处理：
- 支持新的对象数组格式 `{code: "002837", name: "英维克"}`
- 向后兼容旧的字符串数组格式 `["002837"]`
- 当使用旧格式时，提供默认的股票名称映射

### 后端修改

#### 1. 数据收集器 (`data_collectors.py`)

修改 `collect_from_akshare_adapter` 函数：
- 从 `custom_stocks` 配置中提取股票代码
- 支持对象格式和字符串格式的兼容处理
- 只提取 `code` 字段用于实际的数据采集

#### 2. 配置验证 (`datasource_routes.py`)

修改 `validate_data_source_config` 函数：
- 更新股票代码验证逻辑
- 支持从对象格式中提取代码进行验证
- 保持对字符串格式的兼容性

## 优势

1. **性能优化**：避免重复查询股票名称，提升加载速度
2. **用户体验**：界面直接显示完整的"代码+名称"信息
3. **向后兼容**：支持旧的配置格式，无缝升级
4. **数据完整性**：保存查询到的准确股票名称信息

## 使用示例

### 前端展示效果

用户在搜索并添加股票时，界面会显示：
```
002837 英维克  [×]
688702 盛科通信  [×]
```

### 配置文件保存格式

```json
{
  "source_id": "akshare_stock_a",
  "config": {
    "custom_stocks": [
      {"code": "002837", "name": "英维克"},
      {"code": "688702", "name": "盛科通信"}
    ],
    "stock_pool_type": "custom"
  }
}
```

## 兼容性说明

- **向后兼容**：旧的字符串数组格式仍然被支持
- **向前兼容**：新版本会自动保存为对象数组格式
- **数据迁移**：现有配置会自动转换为新格式，无需手动干预
