# 全面修复回测次数和优化次数未更新问题计划

## 问题描述
一键执行完整流程完成后，回测次数和优化次数依然未更新。

## 诊断目标
1. 检查前端统计更新函数是否正确调用和执行
2. 检查API请求是否成功，请求体和响应是否正确
3. 检查后端是否正确接收和处理请求
4. 检查数据库是否正确更新stats字段
5. 检查数据加载时是否正确合并stats字段

## 检查范围

### 1. 前端调用链
- 回测完成后是否正确调用 `updateStrategyBacktestStats`
- 应用优化后是否正确调用 `updateStrategyOptimizationStats`
- 函数内部逻辑是否正确执行
- API请求是否成功

### 2. 后端处理链
- PUT `/strategy/conceptions/{id}` 是否正确接收请求体
- `save_strategy_conception` 是否正确处理stats字段
- `UnifiedPersistence.save` 是否正确保存stats到data字段
- 数据库是否正确更新

### 3. 数据加载链
- GET `/strategy/conceptions` 是否正确返回stats字段
- `_list_from_database` 是否正确合并data字段中的stats
- 前端是否正确读取和显示stats

### 4. 数据验证
- 检查PostgreSQL中data字段的内容
- 检查文件系统中的数据
- 验证数据一致性
