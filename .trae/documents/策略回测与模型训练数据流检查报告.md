### 🔍 发现的关键问题

经过详细检查，发现**策略回测目前没有直接使用模型训练的结果作为输入**。存在以下问题：

#### 1. 数据流断裂 ❌

**当前数据流**:

```
模型训练 (ModelTraining) 
    ↓ ❌ 没有直接连接
策略回测 (StrategyBacktest)
    ↓
使用原始股票数据回测
```

**期望的数据流**:

```
模型训练 (ModelTraining) 
    ↓ ✅ 训练好的模型/预测信号
策略回测 (StrategyBacktest)
    ↓
使用模型预测结果进行回测
```

#### 2. 策略回测输入数据来源

**当前实现** (`src/gateway/web/backtest_service.py`):

* 回测使用历史股票数据进行模拟交易

* **没有使用模型训练的预测结果**作为交易信号

* 策略信号基于简单的技术指标或固定规则

### 📋 需要修复的问题

#### 问题1: 建立模型到回测的数据流

**优先级**: 🔴 高

**修复方案**:

1. 在回测服务中添加模型预测功能
2. 修改回测API，支持选择训练好的模型
3. 实现模型预测结果到交易信号的转换

**代码修改**:

```python
# src/gateway/web/backtest_service.py
async def run_backtest_with_model(
    strategy_id: str,
    model_id: str,  # 训练好的模型ID
    start_date: str,
    end_date: str,
    config: Dict[str, Any]
):
    # 1. 加载训练好的模型
    model = load_model(model_id)
    
    # 2. 加载回测数据
    data = load_stock_data(symbol, start_date, end_date)
    
    # 3. 使用模型进行预测
    predictions = model.predict(data)
    
    # 4. 将预测结果转换为交易信号
    signals = generate_signals_from_predictions(predictions, threshold=0.5)
    
    # 5. 使用信号进行回测
    result = backtest_engine.run_with_signals(data, signals, config)
    
    return result
```
