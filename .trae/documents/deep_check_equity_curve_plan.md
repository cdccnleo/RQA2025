# 累计收益率走势图表X轴标签深度检查计划

## 任务目标
深度检查并修复策略回测分析页面（strategy-backtest.html）累计收益率走势图表的X轴标签与实际交易日期不一致的问题。

## 当前状态
- 后端修复已生效：新的回测记录 `end_date` 已正确设置为实际最后交易日期（2026-02-13）
- 但问题依然存在：图表X轴标签可能仍与实际交易日期不一致

## 深度检查方向

### 1. 前端标签生成逻辑检查
- 检查 `updateReturnsChart` 函数的标签生成逻辑
- 验证是否使用了正确的 `start_date` 和 `end_date`
- 检查标签生成算法是否考虑了交易日历

### 2. 数据流向检查
- 追踪从后端返回的数据到前端图表的完整流程
- 检查数据在传递过程中是否被修改
- 验证 `equity_curve` 数据与标签的对应关系

### 3. 图表渲染检查
- 检查 Chart.js 的数据配置
- 验证X轴标签与数据点的对齐
- 检查是否有缓存导致旧数据残留

### 4. 边界情况检查
- 检查强制平仓日期的处理
- 验证周末和节假日的跳过逻辑
- 检查数据点数量与标签数量是否匹配

## 检查步骤

### 第一阶段：前端代码审查（15分钟）
1. 详细检查 `updateReturnsChart` 函数
2. 检查 `updateCharts` 函数的数据处理
3. 检查 `loadBacktestHistory` 函数的数据加载
4. 添加详细调试日志

### 第二阶段：数据流向追踪（15分钟）
1. 检查后端返回的完整数据结构
2. 验证前端接收的数据结构
3. 检查数据在 `updateCharts` 中的转换
4. 验证最终传递给 Chart.js 的数据

### 第三阶段：标签生成算法修复（20分钟）
1. 修复标签生成逻辑，确保与实际交易日期一致
2. 添加数据点与标签的验证机制
3. 确保强制平仓日期正确显示
4. 测试不同日期范围的回测数据

### 第四阶段：测试验证（10分钟）
1. 清除浏览器缓存，刷新页面
2. 检查累计收益率走势图表
3. 验证X轴标签与实际交易日期一致
4. 检查所有数据点都有对应的标签

## 关键检查点

### 必须验证的内容
- [ ] `result.end_date` 是否正确传递到前端
- [ ] 前端是否使用 `result.end_date` 生成标签
- [ ] 标签生成算法是否正确计算交易日
- [ ] 资金曲线数据点数量与标签数量是否匹配
- [ ] 最后一个标签是否对应最后交易日期

### 可能的问题点
1. 前端可能使用了错误的日期字段
2. 标签生成算法可能使用了自然日而非交易日
3. 数据点与标签的对应关系可能有偏移
4. 浏览器缓存可能显示旧数据

## 预期结果
累计收益率走势图表的X轴标签应该：
- 从实际开始日期（2026-02-03）开始
- 到实际结束日期（2026-02-13）结束
- 每个标签对应一个实际交易日
- 与交易记录明细的日期完全一致

## 风险与缓解
| 风险 | 缓解措施 |
|------|----------|
| 标签算法复杂 | 简化算法，直接使用数据点索引生成标签 |
| 数据格式不一致 | 添加数据验证和转换逻辑 |
| 浏览器缓存 | 强制刷新页面，清除缓存 |
| 时区问题 | 确保所有日期使用同一时区 |
