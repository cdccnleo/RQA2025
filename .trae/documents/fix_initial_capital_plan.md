# 回测初始资金与账户可用资金一致性修复计划

## 任务目标
确保策略回测分析页面（strategy-backtest.html）中：
1. 回测初始资金正确作为账户可用资金的起始值
2. 根据回测交易记录正确更新账户可用资金
3. 验证资金计算逻辑是否符合预期

## 当前状态
- 已添加"账户可用资金"列到交易记录表格
- 已实现资金计算逻辑
- 需要验证：初始资金是否正确获取，资金更新逻辑是否准确

## 问题分析

### 可能存在的问题
1. **初始资金获取问题**：可能未正确从回测结果中获取初始资金
2. **资金计算顺序问题**：交易记录排序可能影响资金计算结果
3. **买入/卖出逻辑问题**：资金增减计算可能有误
4. **强制平仓处理问题**：强制平仓的资金处理可能不正确
5. **多回测结果合并问题**：合并多个回测结果时初始资金计算可能有误

### 需要验证的内容
- [ ] 初始资金是否正确从回测结果获取
- [ ] 买入时资金是否正确扣除（价格×数量+手续费）
- [ ] 卖出时资金是否正确增加（价格×数量-手续费+盈亏）
- [ ] 强制平仓时资金处理是否正确
- [ ] 最终资金是否与回测结果一致

## 检查步骤

### 第一阶段：验证初始资金获取（10分钟）
1. 检查 `updateTradesDisplay` 函数中初始资金的获取逻辑
2. 验证 `results[0].initial_capital` 是否正确
3. 检查多个回测结果合并时的初始资金处理
4. 添加调试日志追踪初始资金值

### 第二阶段：验证资金计算逻辑（15分钟）
1. 逐笔验证买入交易的资金扣除计算
2. 逐笔验证卖出交易的资金增加计算
3. 验证手续费的扣除逻辑
4. 检查盈亏是否正确计入资金

### 第三阶段：修复发现的问题（15分钟）
1. 修复初始资金获取逻辑
2. 修复买入/卖出资金计算逻辑
3. 修复强制平仓资金处理
4. 添加数据验证和错误处理

### 第四阶段：测试验证（10分钟）
1. 刷新页面，检查交易记录表格
2. 验证初始资金显示正确
3. 逐笔验证资金变化是否符合预期
4. 验证最终资金与回测结果一致

## 资金计算逻辑验证

### 正确的资金计算逻辑
```
初始资金 = results[0].initial_capital

买入交易：
  账户可用资金 -= (成交价格 × 数量 + 手续费)

卖出交易：
  账户可用资金 += (成交价格 × 数量 - 手续费)
  // 盈亏已经包含在卖出金额中，不需要额外加减

强制平仓：
  同卖出交易处理
```

### 验证公式
```
最终资金 = 初始资金 + 所有卖出金额 - 所有买入金额 - 所有手续费
```

## 预期结果
- 第一笔交易前的账户可用资金 = 回测初始资金
- 每笔交易后的账户可用资金计算准确
- 最后一笔交易后的账户可用资金 = 回测最终资金
- 资金变化趋势与交易记录一致

## 风险与缓解
| 风险 | 缓解措施 |
|------|----------|
| 初始资金获取失败 | 添加默认值和错误处理 |
| 计算精度问题 | 使用高精度计算，避免浮点数误差 |
| 交易记录排序错误 | 确保按时间顺序计算资金 |
| 多回测结果冲突 | 分别计算每个回测的资金，或显示警告 |
