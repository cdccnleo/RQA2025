## 📋 **实施目标**

**核心目标：** 统一AKShare与BaoStock不同数据源的计算逻辑，确保量化交易系统数据分析的一致性

**具体目标：**
- ✅ 统一涨跌额(change)计算逻辑
- ✅ 统一振幅(amplitude)计算逻辑
- ✅ 统一涨跌幅(pct_change)计算逻辑
- ✅ 确保所有数据源返回标准化的数据格式

## 🔧 **实施步骤**

### **步骤1：增强数据源管理器统一计算逻辑**
**文件：** `src/core/integration/data_source_manager.py`

**修改内容：**
1. **新增标准化方法**：在 `_get_data_from_source` 方法中添加 `_normalize_stock_data` 方法
2. **统一计算逻辑**：
   ```python
   def _normalize_stock_data(self, df, source_name):
       """统一处理股票数据，计算标准字段"""
       normalized_data = []
       
       for _, row in df.iterrows():
           record = {
               'symbol': row.get('symbol', ''),
               'date': row.get('date', ''),
               'open': row.get('open', row.get('开盘', 0)),
               'high': row.get('high', row.get('最高', 0)),
               'low': row.get('low', row.get('最低', 0)),
               'close': row.get('close', row.get('收盘', 0)),
               'volume': row.get('volume', row.get('成交量', 0)),
               'amount': row.get('amount', row.get('成交额', 0)),
               'data_source': source_name
           }
           
           # 统一计算涨跌额
           if record['open'] > 0 and record['close'] > 0:
               record['change'] = record['close'] - record['open']
           
           # 统一计算振幅
           if record['open'] > 0 and record['high'] > 0 and record['low'] > 0:
               record['amplitude'] = (record['high'] - record['low']) / record['open'] * 100
           
           # 统一计算涨跌幅
           if record['open'] > 0 and record['close'] > 0:
               record['pct_change'] = (record['close'] - record['open']) / record['open'] * 100
           
           normalized_data.append(record)
       
       return normalized_data
   ```
3. **应用标准化**：在各数据源返回结果后调用此方法

## 🔒 **风险控制**

**潜在风险：**
1. **数据格式变更**：可能影响依赖旧格式的代码
2. **计算结果差异**：可能导致历史数据与新数据的计算结果不同

**应对措施：**
1. **渐进式部署**：先在测试环境验证，再部署到生产环境
2. **数据备份**：在修改前备份数据库中的历史数据
3. **兼容性检查**：全面检查代码库中依赖数据格式的部分
