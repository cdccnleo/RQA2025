# 修复 strategy-lifecycle 页面 strategies.map 错误计划

## 问题描述
策略生命周期页面首次打开时错误提示：`❌ 加载策略列表失败: strategies.map is not function`
- 强制刷新后无错误提示
- 再次打开时仍存在同样的错误提示

## 问题分析

### 可能原因
1. **Service Worker 缓存了错误的响应格式**
   - 首次访问时缓存了非数组格式的响应
   - 后续访问直接返回缓存的错误数据

2. **浏览器缓存了旧的 JavaScript 文件**
   - HTML 文件已更新，但浏览器仍使用缓存的 JS 逻辑

3. **加载顺序问题**
   - 脚本执行时机与 DOM 加载顺序不匹配
   - 变量未定义或覆盖

4. **API 响应格式不一致**
   - 首次请求和后续请求的响应格式不同

## 解决方案

### 阶段 1: 诊断问题根源
1. 检查 Service Worker 是否注册并缓存了数据
2. 检查浏览器控制台的具体错误堆栈
3. 检查网络面板的 API 响应数据

### 阶段 2: 修复 Service Worker 缓存
1. 更新 Service Worker 版本号，强制更新
2. 修改缓存策略，不缓存 API 响应
3. 添加缓存清理逻辑

### 阶段 3: 增强前端容错
1. 在调用 .map() 之前添加类型检查
2. 添加默认值处理
3. 添加详细的错误日志

### 阶段 4: 验证修复
1. 测试首次打开页面
2. 测试刷新页面
3. 测试重新打开页面

## 具体实施步骤

### 步骤 1: 修改 Service Worker (sw.js)
- 更新 CACHE_NAME 版本号
- 排除 `/strategy/conceptions` API 的缓存
- 添加激活时清理旧缓存逻辑

### 步骤 2: 修改 strategy-lifecycle.html
- 在调用 .map() 前添加防御性检查
- 添加 console.log 输出调试信息
- 确保变量正确定义和初始化

### 步骤 3: 添加调试日志
- 记录 API 响应数据类型
- 记录变量状态
- 记录错误堆栈

### 步骤 4: 测试验证
- 清除所有缓存后测试
- 模拟首次访问场景
- 验证多次打开是否正常

## 预期结果
- 首次打开页面不再出现 .map 错误
- 页面加载策略列表正常
- 多次打开页面保持一致性

## 风险评估
- **低风险**: 修改主要集中在前端容错处理
- **中风险**: Service Worker 更新可能影响其他页面缓存
- **缓解措施**: 逐步更新，保留回滚方案
