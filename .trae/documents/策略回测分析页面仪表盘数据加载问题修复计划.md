# 策略回测分析页面仪表盘数据加载问题修复计划

## 问题分析

经过深度检查，发现以下情况：

1. **后端API正常**：`/api/v1/backtest` 端点返回 200 OK 状态码和完整的回测数据
2. **前端代码已包含修复**：
   - 已在 DOM 加载完成后调用 `initCharts()` 函数
   - 已修复最大回撤计算错误
   - 已修复百分比计算括号位置错误
   - 已优化图表数据加载函数

## 可能的问题原因

1. **API调用路径问题**：前端可能使用了错误的API调用路径
2. **CORS配置问题**：跨域资源共享配置可能不正确
3. **初始化顺序问题**：前端初始化顺序可能存在问题
4. **错误处理问题**：前端错误处理可能不完善
5. **数据格式问题**：前端可能无法正确处理后端返回的数据格式

## 修复计划

- 验证 `getApiBaseUrl()` 函数是否返回正确的API基础路径
- 确保所有API调用使用正确的端点

### 2. 检查和修复CORS配置
- 验证后端CORS配置是否正确
- 确保前端可以正常访问后端API

### 3. 优化前端初始化顺序
- 确保图表初始化在数据加载之前完成
- 确保所有依赖项正确初始化

### 4. 增强前端错误处理
- 添加详细的错误日志
- 确保API调用失败时提供有用的错误信息
- 添加用户友好的错误提示

### 5. 验证数据格式处理
- 确保前端可以正确处理后端返回的数据格式
- 添加数据格式验证

### 6. 测试和验证
- 测试所有仪表盘是否能正常加载数据
- 验证所有数据计算是否正确
- 确保图表显示正确

- **API端点**：`/api/v1/backtest`
- **数据结构**：包含backtest_id、strategy_id、status、start_date、end_date、initial_capital、final_capital、total_return、annualized_return、sharpe_ratio、max_drawdown、win_rate、total_trades、equity_curve、trades、metrics等字段
