# 历史数据采集任务真实性问题检查计划

## 检查目标

全面检查数据采集监控页面中"最近完成的任务"仪表盘的以下问题：
1. 已完成的任务（如 task_20260223_0000、task_20260223_0001）的数据源、记录数、执行日志是否来自真实采集记录
2. source_2 等数据源ID在数据源配置中是否存在对应配置
3. 历史数据采集如何根据采集期类型计算并设置历史采集周期起止日期

## 问题分析

### 1. 已完成任务数据的真实性

**位置**: `src/gateway/web/datasource_routes.py` 第 2946-3004 行

**代码分析**:
```python
@router.get("/api/v1/data/scheduler/tasks/completed")
async def get_completed_tasks(limit: int = 20, offset: int = 0):
    """获取最近完成的任务列表"""
    try:
        from src.distributed.coordinator.unified_scheduler import get_unified_scheduler
        
        scheduler = get_unified_scheduler()
        
        # 获取调度器统计信息
        scheduler_stats = scheduler.get_scheduler_stats()
        
        # 从统一调度器获取已完成任务
        # 注意：统一调度器目前只保存统计数字，不保存详细历史
        # 这里我们生成模拟数据用于演示，后续可以扩展调度器保存详细历史  <-- 问题点
        completed_count = scheduler_stats.get("completed_tasks", 0)
        failed_count = scheduler_stats.get("failed_tasks", 0)
        
        # 生成模拟任务数据（实际应该从调度器或数据库获取）  <-- 问题点
        tasks = []
        import random
        from datetime import datetime, timedelta
        
        # 如果有实际完成的任务，生成模拟详情
        total_tasks = min(completed_count + failed_count, 100)  # 最多生成100条
        
        for i in range(min(limit, total_tasks - offset)):
            task_index = offset + i
            if task_index >= total_tasks:
                break
                
            # 随机生成任务数据  <-- 问题点
            is_success = random.random() > 0.1  # 90%成功率
            completed_time = datetime.now() - timedelta(minutes=random.randint(1, 1440))
            
            task = {
                "task_id": f"task_{completed_time.strftime('%Y%m%d')}_{task_index:04d}",  # <-- 生成模拟task_id
                "source_id": f"source_{random.randint(1, 10)}",  # <-- 随机生成source_1到source_10
                "status": "completed" if is_success else "failed",
                "records_count": random.randint(100, 10000) if is_success else 0,  # <-- 随机生成记录数
                "duration_ms": random.randint(1000, 30000),  # <-- 随机生成耗时
                "completed_at": completed_time.isoformat(),
                "error_message": None if is_success else "连接超时"
            }
            tasks.append(task)
```

**问题确认**: ✅ **任务数据完全来自模拟生成**
- 使用 `random.randint(1, 10)` 随机生成 source_id（source_1 到 source_10）
- 使用 `random.randint(100, 10000)` 随机生成记录数
- 使用 `random.randint(1000, 30000)` 随机生成耗时
- 注释明确说明"生成模拟数据用于演示"

### 2. 数据源ID与实际配置不匹配

**实际数据源配置检查**:
- 从之前的检查可知，实际配置的数据源包括：
  - `akshare_stock_a` - AKShare A股数据
  - `baostock_a股数据` - BaoStock A股数据
  - `akshare_stock_hk` - AKShare 港股数据
  - `akshare_index` - AKShare 指数数据
  - 等等（共16个数据源）

**问题确认**: ✅ **source_2 等ID与实际数据源配置不匹配**
- 代码生成的是 `source_1` 到 `source_10`
- 实际数据源ID是 `akshare_stock_a`、`baostock_a股数据` 等
- 完全没有对应关系

### 4. 历史采集周期起止日期计算

**采集期类型配置**:
- 位置: `datasource_routes.py` 第 3233 行
- 配置项: `collection_period_type`: `quarterly` 或 `yearly`

**当前实现分析**:
- 在 `data_collection_scheduler_manager.py` 中，使用 `should_collect()` 函数判断是否采集
- `should_collect()` 基于 `last_test` 时间和 `rate_limit` 计算
- **没有发现根据 `collection_period_type` 计算历史采集周期的代码**

**rate_limit_parser.py 分析**:
- `parse_rate_limit()` 解析 "1次/天"、"100次/分钟" 等格式
- `should_collect()` 基于上次采集时间和间隔判断是否采集
- **只支持固定间隔，不支持季度/年度周期计算**

**问题确认**: ✅ **没有实现根据采集期类型计算历史采集周期**
- `collection_period_type` 配置项存在但未被使用
- 没有季度/年度周期的起止日期计算逻辑
- 历史数据采集没有按季度或年度划分的概念

## 修复建议

### 短期修复（高优先级）

1. **使用真实数据源ID**
   - 修改 `get_completed_tasks()`，从 `DataSourceConfigManager` 获取实际启用的数据源
   - 使用真实数据源ID替代 `source_{random.randint(1, 10)}`

2. **从统一调度器获取真实任务数据**
   - 扩展统一调度器，保存已完成任务的详细信息
   - 或创建数据库表 `task_history` 存储任务执行记录

### 中期修复（中优先级）

3. **实现采集期类型计算**
   - 添加季度/年度周期计算函数
   - 根据 `collection_period_type` 计算历史采集起止日期
   - 示例：
     - quarterly: 采集最近4个季度的数据
     - yearly: 采集最近3年的数据

4. **保存真实采集记录**
   - 创建数据库表存储每次采集的详细记录
   - 包括：数据源ID、记录数、耗时、状态、日志等

### 长期改进（低优先级）

5. **任务执行追踪**
   - 实现完整的任务生命周期追踪
   - 包括：提交、开始、进度、完成、失败等状态
   - 支持实时日志流
