# 全面检查一键执行完整流程计划

## 目标
全面检查一键执行完整流程，确保回测次数和优化次数的更新逻辑正确实现，代码健壮可靠。

## 检查范围

### 1. 回测次数更新
- 回测完成后是否正确更新 `backtest_count`
- `last_backtest_at` 时间戳是否正确记录
- 更新失败时是否有适当的错误处理
- 前端是否正确显示回测次数

### 2. 优化次数更新
- 优化完成后是否正确更新 `optimization_count`
- `last_optimization_at` 时间戳是否正确记录
- 应用优化结果后是否更新优化次数
- 前端是否正确显示优化次数

### 3. 代码健壮性
- API端点路径是否正确
- 错误处理是否完善
- 日志记录是否清晰
- 降级机制是否有效

### 4. 数据一致性
- 回测次数在PostgreSQL和文件系统是否一致
- 优化次数在PostgreSQL和文件系统是否一致
- 统计数据是否正确持久化

## 实施步骤

### 第一阶段：检查回测次数更新逻辑（30分钟）
1. 检查前端回测步骤代码
2. 检查后端策略更新API
3. 验证API端点路径
4. 测试回测次数更新

### 第二阶段：检查优化次数更新逻辑（30分钟）
1. 检查前端优化步骤代码
2. 检查前端应用步骤代码
3. 添加优化次数更新逻辑
4. 验证API端点路径

### 第三阶段：代码健壮性检查（30分钟）
1. 检查所有API调用路径
2. 检查错误处理逻辑
3. 检查日志记录
4. 修复发现的问题

### 第四阶段：验证测试（20分钟）
1. 测试完整一键执行流程
2. 验证回测次数更新
3. 验证优化次数更新
4. 验证前端显示

## 预期产出
1. 回测次数更新流程文档
2. 优化次数更新流程文档
3. 修复后的代码
4. 测试验证报告
