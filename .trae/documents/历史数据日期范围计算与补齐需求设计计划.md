1. **无历史数据补齐机制**: 只采集最近N天数据，不检查数据库中已有数据的日期范围

## 历史数据补齐需求设计

### 需求1: 自动检测缺失数据时间段

**目标**: 自动识别数据库中每个股票缺失的数据日期范围

**实现方案**:
1. 查询数据库中该股票已有数据的最早日期和最新日期
2. 计算缺失时间段:
   - 历史缺失: 从上市日期到最早记录日期
   - 近期缺失: 从最新记录日期到当前日期
   - 中间缺失: 数据中的断档日期

**数据库查询**:
```sql
-- 查询某股票已有数据的日期范围
SELECT 
    MIN(date) as earliest_date,
    MAX(date) as latest_date,
    COUNT(DISTINCT date) as record_count
FROM akshare_stock_data 
WHERE symbol = '000001';

-- 查询缺失日期
SELECT generate_series(
    '2020-01-01'::date,
    CURRENT_DATE,
    '1 day'::interval
)::date AS date
EXCEPT
SELECT date FROM akshare_stock_data WHERE symbol = '000001';
```

### 需求3: 智能增量采集

**目标**: 只采集数据库中缺失的数据，避免重复下载

**实现方案**:
1. **检测已有数据**:
```python
def get_existing_data_range(symbol: str) -> Tuple[Optional[str], Optional[str]]:
    """获取数据库中已有数据的日期范围"""
    query = """
        SELECT MIN(date) as earliest, MAX(date) as latest
        FROM akshare_stock_data
        WHERE symbol = %s
    """
    # 返回 (earliest_date, latest_date)
```

2. **计算缺失范围**:
```python
def calculate_missing_ranges(
    symbol: str,
    target_start: str,
    target_end: str
) -> List[Tuple[str, str]]:
    """计算需要采集的缺失日期范围列表"""
    existing_start, existing_end = get_existing_data_range(symbol)
    
    missing_ranges = []
    
    # 历史缺失
    if existing_start and target_start < existing_start:
        missing_ranges.append((target_start, existing_start))
    
    # 近期缺失
    if existing_end and existing_end < target_end:
        missing_ranges.append((existing_end, target_end))
    
    # 中间缺失（检测断档）
    missing_dates = detect_gap_dates(symbol, existing_start, existing_end)
    for gap in missing_dates:
        missing_ranges.append(gap)
    
    return missing_ranges
```

3. **分段采集**:
```python
# 对每个缺失范围进行采集
for start, end in missing_ranges:
    success = self._collect_and_save(symbol, start, end)
```
