### 4. 数据采集调度管理器检查

**文件**: `src/gateway/web/data_collection_scheduler_manager.py`

#### 4.1 核心功能

| 功能 | 实现状态 | 说明 |
|------|----------|------|
| 定时检查数据源 | ✅ 已实现 | `_scheduler_loop()` 方法，默认60秒间隔 |
| 根据频率生成任务 | ✅ 已实现 | `_check_and_schedule()` 方法 |
| 提交任务到统一调度器 | ✅ 已实现 | `_submit_collection_task()` 方法 |
| 避免重复任务 | ✅ 已实现 | `_has_pending_task()` 方法，按天去重 |
| 任务清理 | ✅ 已实现 | `_cleanup_completed_tasks()` 方法，保留3天记录 |

#### 4.2 任务提交流程

```
1. _scheduler_loop() 定时触发
2. _check_and_schedule() 获取所有启用数据源
3. _check_source() 检查单个数据源
   - 调用 should_collect() 判断是否需要采集
   - 调用 _has_pending_task() 检查是否已有待处理任务
4. _submit_collection_task() 提交任务到统一调度器
   - 使用 TaskType.DATA_COLLECTION
   - 优先级 TaskPriority.NORMAL
```

### 5. 统一调度器集成检查

**统一调度器**: `src/distributed/coordinator/unified_scheduler.py`

#### 5.1 任务类型
- `TaskType.DATA_COLLECTION` - 数据采集任务

#### 5.2 任务优先级
- `TaskPriority.NORMAL` - 正常优先级

#### 5.3 调度器统计
通过 `scheduler.get_scheduler_stats()` 获取:
- total_tasks: 总任务数
- pending_tasks: 待处理任务数
- running_tasks: 运行中任务数
- completed_tasks: 已完成任务数
- failed_tasks: 失败任务数
- queue_sizes: 各队列大小

### 7. 数据流分析

#### 7.1 任务创建流程

```
用户点击"立即采集"
    ↓
frontend: triggerImmediateCollection()
    ↓
API: POST /api/v1/data/scheduler/control
    ↓
backend: control_scheduler() (datasource_routes.py:210)
    ↓
    如果 action == "trigger_immediate"
        调用统一调度器功能

或者：

自动调度器定时检查
    ↓
backend: DataCollectionSchedulerManager._scheduler_loop()
    ↓
backend: _check_and_schedule()
    ↓
backend: _submit_collection_task()
    ↓
backend: unified_scheduler.submit_task()
    ↓
任务进入队列等待执行
```

### 8. 改进建议

2. **实现任务详情查看**

3. **完善配置持久化**

#### 8.2 中期改进（中优先级）

1. **添加单个任务控制**
   - 暂停/恢复单个任务

   - 支持更多调度策略（时间窗口、依赖关系等）

2. **历史数据分析**
   - 采集趋势分析图表
   - 性能瓶颈识别
   - 预测性维护建议

## 检查结论

### 已实现功能 ✅

1. **基础调度控制**: 启动/停止自动采集
2. **任务提交**: 自动定时提交和手动立即触发
3. **状态监控**: 实时获取调度器状态和统计
4. **统一调度器集成**: 与分布式调度器良好集成
5. **避免重复**: 按天去重机制防止重复提交

### 待完善功能 ⚠️

1. **任务管理粒度**: 只能控制整个调度器，无法管理单个任务
2. **历史记录**: 无法查看已完成任务历史
3. **配置持久化**: 采集期类型等配置无法保存
4. **任务详情**: 无法查看任务执行详情和日志

### 整体评价

任务管理标签实现了**基础的调度控制功能**，能够完成历史数据采集的自动调度和手动触发。但在**精细化管理**方面还有较大提升空间，特别是单个任务控制和历史记录查看功能缺失。

建议优先实现**最近完成任务列表**和**配置持久化**功能，以提升用户体验。
