# 特征选择流程实施方案A - 验证仪表盘功能

## 1. 项目概述

### 1.1 目标
- 触发特征选择任务，生成特征选择历史数据
- 验证特征选择过程仪表盘的显示功能
- 完善系统特征工程流程

### 1.2 背景
- 当前特征提取已完成（100个特征）
- 特征选择历史表为空（0条记录）
- 特征选择过程仪表盘无法显示数据

## 2. 实施步骤

### 阶段1：检查特征选择API和任务类型（P0）
1. 检查统一调度器支持的任务类型
2. 确认特征选择任务的API接口
3. 检查特征选择器的实现状态

### 阶段2：准备特征选择任务参数（P0）
1. 确定待选择的股票代码列表
2. 配置特征选择方法（importance/correlation/LASSO等）
3. 设置选择参数（top_k、threshold等）

### 阶段3：触发特征选择任务（P0）
1. 通过API提交特征选择任务
2. 监控任务执行状态
3. 确认任务完成并生成历史记录

### 阶段4：验证仪表盘显示（P0）
1. 检查feature_selection_history表数据
2. 刷新监控页面查看仪表盘
3. 验证图表和数据展示

### 阶段5：完善和优化（P1）
1. 添加特征选择结果评估
2. 优化仪表盘展示效果
3. 文档更新

## 3. 技术实现

### 3.1 特征选择任务API调用
```bash
POST /api/v1/scheduler/tasks
{
  "task_type": "feature_selection",
  "name": "特征选择-测试任务",
  "params": {
    "symbols": ["000917", "300124", "300727"],
    "selection_method": "importance",
    "top_k": 10,
    "min_quality": 0.7
  },
  "priority": "high"
}
```

### 3.2 备选：直接调用特征选择服务
```python
from src.features.selection.feature_selector import FeatureSelector

selector = FeatureSelector()
selected_features = selector.select_features(
    symbols=["000917", "300124"],
    method="importance",
    top_k=10
)
```

## 4. 预期结果

### 4.1 数据库状态
- feature_selection_history表应有1+条记录
- 记录包含选择方法、输入特征数、输出特征数等

### 4.2 仪表盘显示
- 特征选择过程图表显示历史记录
- 图表展示已选择特征数变化趋势
- 时间轴显示选择操作时间点

### 4.3 系统流程
```
数据采集 -> 特征提取 -> 特征选择(新增) -> 待模型训练
```

## 5. 验收标准

- [ ] 特征选择任务成功提交
- [ ] 任务执行完成且状态为completed
- [ ] feature_selection_history表有数据
- [ ] 特征选择过程仪表盘显示图表
- [ ] 图表数据与数据库记录一致

## 6. 风险与应对

| 风险 | 可能性 | 应对措施 |
|------|--------|----------|
| 特征选择服务未实现 | 中 | 使用备选方案直接调用选择器 |
| 任务执行失败 | 低 | 检查日志，修复问题后重试 |
| 仪表盘仍不显示 | 低 | 检查前端数据格式，修复渲染逻辑 |
