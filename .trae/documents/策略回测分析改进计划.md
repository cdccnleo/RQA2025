## 阶段一：风险控制修复（高优先级）

### 1.1 增加止损止盈设置
**文件**: `src/gateway/web/backtest_routes.py`

**修改内容**:
- 在 `BacktestRequest` 模型中增加止损止盈参数
- 在模型驱动回测请求中增加止损止盈参数
- 默认止损5%，默认止盈10%

**具体修改**:
```python
class BacktestRequest(BaseModel):
    """回测请求模型"""
    strategy_id: str
    start_date: str
    end_date: str
    initial_capital: float = 100000.0
    commission_rate: float = 0.001
    slippage: float = 0.001
    market_impact: float = 0.001
    stop_loss: float = 0.05  # 默认5%止损
    take_profit: float = 0.10  # 默认10%止盈
```

- 默认最大仓位30%
- 支持多种仓位策略

**具体修改**:

    max_position_size: float = 0.3  # 默认最大30%仓位

### 1.3 增加风险敞口限制

## 阶段二：数据安全修复（高优先级）

### 2.1 增加日期范围限制
**文件**: `src/gateway/web/backtest_routes.py`

**修改内容**:
- 定义最大日期范围常量（5年）
- 在回测端点中验证日期范围
- 如果超出范围，返回400错误

**具体修改**:
```python
MAX_DATE_RANGE_DAYS = 365 * 5

# 在回测端点中验证
days = (end_date - start_date).days
if days > MAX_DATE_RANGE_DAYS:
    raise HTTPException(status_code=400, detail=f"日期范围不能超过{MAX_DATE_RANGE_DAYS}天")
```
