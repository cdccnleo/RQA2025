# 业务流程驱动架构完善计划 - 数据采集调度器优化

## 概述
本计划实现数据采集调度器的完善,确保在数据采集配置成功完成采集后自动更新最后测试时间,并深入检查当前数据采集配置是否未进行数据采集。

## 任务分解

### [ ] 任务1: 分析数据采集流程
- **Priority**: P0
- **Depends On**: None
- **Description**:
  - 分析现有的数据采集流程和相关代码
  - 理解数据采集调度器的工作原理
  - 理解数据采集配置的结构
- **Success Criteria**:
  - 完整理解数据采集流程和关键代码位置
- **Test Requirements**:
  - `programmatic` TR-1.1: 确认找到所有关键文件并记录位置
  - `human-judgment` TR-1.2: 理解数据采集的完整生命周期
- **Notes**: 重点分析 `data_collectors.py` 和 `data_source_config_manager.py`

### [ ] 任务2: 添加最后测试时间更新功能
- **Priority**: P0
- **Depends On**: 任务1
- **Description**:
  - 在数据采集成功完成后,自动更新数据源配置的 `last_test` 字段
  - 更新 `last_collection` 字段和相关统计
  - 更新 `collection_status` 状态
- **Success Criteria**:
  - 数据采集完成后,相应数据源配置的时间字段被正确更新
- **Test Requirements**:
  - `programmatic` TR-2.1: 验证字段更新逻辑正确
  - `human-judgment` TR-2.2: 测试完成时间戳格式正确

### [ ] 任务3: 实现未采集数据检查功能
- **Priority**: P0
- **Depends On**: 任务1
- **Description**:
  - 检查数据采集配置的采集状态
  - 识别长时间未采集的数据源配置
  - 添加告警和提示功能
- **Success Criteria**:
  - 能够准确识别未进行数据采集的配置
- **Test Requirements**:
  - `programmatic` TR-3.1: 检查逻辑正确识别未采集配置
  - `human-judgment` TR-3.2: 告警功能正常

### [ ] 任务4: 集成到业务流程驱动架构
- **Priority**: P1
- **Depends On**: 任务2, 任务3
- **Description**:
  - 将新增功能与现有的数据采集调度器集成
  - 确保与业务流程驱动架构的一致性
  - 确保与21个层级架构的一致性
- **Success Criteria**:
  - 功能完整集成,无冲突
- **Test Requirements**:
  - `programmatic` TR-4.1: 集成后的代码能够正常运行
  - `human-judgment` TR-4.2: 架构一致性检查通过

### [ ] 任务5: 测试和验证
- **Priority**: P1
- **Depends On**: 任务4
- **Description**:
  - 单元测试新增功能
  - 集成测试整个数据采集流程
  - 验证数据采集配置更新
  - 验证未采集检查
- **Success Criteria**:
  - 所有测试通过
- **Test Requirements**:
  - `programmatic` TR-5.1: 单元测试通过
  - `programmatic` TR-5.2: 集成测试通过
  - `human-judgment` TR-5.3: 功能验收通过
