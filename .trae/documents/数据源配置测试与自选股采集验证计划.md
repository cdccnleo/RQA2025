# 数据源配置测试与自选股采集验证计划

## 目标
1. 将已启用的数据源配置的测试时间回退1天
2. 检查数据采集是否按照数据源配置的自选股池进行
3. 验证数据是否正确持久化到数据库

## 背景
用户已删除 `000001` 的采集数据，现在需要：
1. 回退测试时间以触发数据采集
2. 验证系统是否按照配置的自选股池进行数据采集
3. 确保采集的数据正确持久化到 PostgreSQL 数据库

## 检查范围

### 1. 已启用数据源识别
- 检查所有 `enabled: true` 的数据源
- 确认数据源的自选股池配置（`custom_stocks`、`stock_pool` 等）
- 记录当前测试时间

### 2. 批量回退测试时间
- 使用已实现的批量回退 API
- 将所有已启用数据源的 `last_test` 回退1天
- 验证回退操作成功

### 3. 数据采集触发验证
- 触发数据采集（手动或自动）
- 监控采集过程日志
- 确认采集按照自选股池配置进行

### 4. 数据持久化验证
- 检查数据库中新增的数据
- 验证数据的 `source_id` 和 `symbol` 正确性
- 确认只采集了自选股池中的股票

## 实现方案

### 步骤1：识别已启用数据源
**操作**: 查询数据源配置，筛选 `enabled: true` 的数据源

**检查命令**:
```bash
# 查看所有启用的数据源
curl -s http://localhost:8000/api/v1/data/sources | python -m json.tool | grep -A 5 '"enabled": true'
```

**预期输出**:
- 列出所有启用的数据源ID
- 记录每个数据源的 `last_test` 时间
- 确认自选股池配置

### 步骤2：批量回退测试时间
**操作**: 调用批量回退 API

**API调用**:
```bash
# 批量回退所有已启用数据源的测试时间
curl -X POST http://localhost:8000/api/v1/data/sources/batch/rollback-test-time \
  -H "Content-Type: application/json" \
  -d '{
    "source_ids": ["akshare_stock_a", "akshare_index", ...],
    "days": 1
  }'
```

**预期结果**:
- 所有已启用数据源的 `last_test` 回退1天
- API 返回成功状态和详细信息

### 步骤3：触发数据采集
**操作**: 触发数据采集任务

**方式A - 手动触发**:
```bash
# 调用数据采集 API
curl -X POST http://localhost:8000/api/v1/data/sources/akshare_stock_a/collect \
  -H "Content-Type: application/json"
```

**方式B - 测试连接触发**:
- 在数据源配置页面点击"测试连接"
- 观察是否自动采集数据

**预期行为**:
- 采集任务按照自选股池配置执行
- 只采集配置中指定的股票代码
- 日志显示采集的股票列表

### 步骤4：验证数据持久化
**操作**: 检查数据库中的采集结果

**SQL查询**:
```sql
-- 查看最近采集的数据
SELECT source_id, symbol, COUNT(*) as count, MAX(date) as latest_date
FROM akshare_stock_data
WHERE collected_at > NOW() - INTERVAL '1 hour'
GROUP BY source_id, symbol
ORDER BY source_id, count DESC;

-- 对比自选股池配置
-- 检查是否只采集了配置中的股票
```

**预期结果**:
- 数据库中新增的数据只包含自选股池中的股票
- 没有非自选股的异常数据
- 数据的 `source_id` 正确对应数据源
