# 策略回测可用资金负数问题修复计划

## 任务目标
深入检查策略回测逻辑，解决交易记录中账户可用资金出现负数的问题。

## 当前问题
从验证结果可以看到，交易2后账户可用资金变为 **¥-2,564.82**（负数）：

```
交易1: 买入002837 -¥87,991.72 → 交易后: ¥12,008.28
交易2: 买入688702 -¥14,573.10 → 交易后: ¥-2,564.82 ❌ 负数！
```

## 问题分析

### 可能的原因
1. **杠杆交易**：回测可能使用了杠杆，允许资金为负
2. **资金计算逻辑错误**：买入时未正确检查可用资金
3. **交易顺序问题**：同一日期的多笔交易顺序可能导致资金计算错误
4. **后端数据问题**：交易记录的时间戳或数据不完整
5. **融资买入**：可能使用了融资功能

### 需要验证的内容
- [ ] 回测是否使用了杠杆或融资
- [ ] 交易记录的时间顺序是否正确
- [ ] 后端资金曲线数据是否也有负数
- [ ] 回测引擎的资金管理逻辑
- [ ] 是否需要添加资金不足检查

## 检查步骤

### 第一阶段：后端资金曲线检查（10分钟）
1. 检查资金曲线数据是否出现负数
2. 验证后端资金计算的准确性
3. 检查回测配置（杠杆、融资等）
4. 分析资金曲线与交易记录的对应关系

### 第二阶段：交易顺序分析（10分钟）
1. 检查同一日期的多笔交易顺序
2. 验证交易时间戳的准确性
3. 分析交易顺序对资金计算的影响
4. 检查是否需要按更细粒度排序（时间+毫秒）

### 第三阶段：前端显示修复（15分钟）
1. 如果后端允许负数（杠杆交易），前端添加说明
2. 添加资金不足警告提示
3. 优化资金显示，负数用红色标注
4. 添加杠杆/融资标识

### 第四阶段：回测逻辑检查（15分钟）
1. 检查回测引擎的资金管理代码
2. 验证是否使用了杠杆
3. 检查融资买入的逻辑
4. 确认资金负数的合理性

## 预期结果
- 明确资金负数的原因（杠杆/融资/数据错误）
- 如果是数据错误，修复资金计算逻辑
- 如果是杠杆交易，前端添加相应标识和说明
- 确保资金显示准确反映回测实际情况
