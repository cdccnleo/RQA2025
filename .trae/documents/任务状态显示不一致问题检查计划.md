# 任务状态显示不一致问题检查计划

## 检查目标

分析检查数据采集监控页面任务管理标签中，运行中的任务列表显示状态为"待处理"，但点击详情后显示状态为"失败"的问题。

## 问题分析

### 1. 前端代码分析

**运行中任务列表显示逻辑** (`data-collection-monitor.html` 第 1916-2009 行):
```javascript
function displayRunningTasks(tasks) {
    // ...
    const statusLabels = {
        'running': '运行中',
        'paused': '已暂停',
        'pending': '待处理'
    };
    // 只显示 running, paused, pending 三种状态
}
```

**任务详情显示逻辑** (`data-collection-monitor.html` 第 2175-2204 行):
```javascript
function renderTaskDetail(task) {
    const statusClass = task.status === 'completed' ? 'success' : 'error';
    const statusText = task.status === 'completed' ? '成功' : '失败';
    // ...
    <div><strong>状态:</strong> <span>${statusText}</span></div>
}
```

### 2. 后端API分析

### 3. 问题根因

**状态不一致的原因**:

1. **运行中任务列表**从 `_task_status_store` 获取状态
   - 只返回状态为 `running` 或 `pending` 的任务
   - 任务创建时初始状态为 `pending`
   - 任务执行过程中状态可能已更新为 `failed` 或 `completed`
   - 但 `_task_status_store` 中的状态没有及时更新

2. **任务详情**从 `task_history_manager` 获取状态
   - 任务历史管理器记录了任务的真实状态
   - 当任务执行失败时，历史记录更新为 `failed`
   - 但 `_task_status_store` 中可能仍显示为 `pending`

3. **状态同步缺失**
   - 任务执行完成（成功或失败）后，`_task_status_store` 中的状态没有更新
   - 导致运行列表显示旧状态（pending），详情显示真实状态（failed）

### 4. 数据流分析

```
任务创建
    ↓
_task_status_store[task_id] = {status: "pending"}  ← 运行列表读取
    ↓
任务执行
    ↓
任务失败
    ↓
task_history_manager.update_task_failed(status: "failed")  ← 详情读取
    ↓
_task_status_store[task_id] 仍为 {status: "pending"}  ← 未更新！
```

## 修复方案

### 方案1: 统一状态源（推荐）

**思路**: 运行中任务列表也从 `task_history_manager` 获取状态，而不是 `_task_status_store`

**修改**:
1. 修改 `get_running_tasks()` API，从任务历史管理器获取任务状态
2. 只返回状态为 `running` 或 `pending` 的任务
3. 确保状态一致性

### 方案2: 同步状态更新

**思路**: 当任务状态变更时，同步更新 `_task_status_store`

**修改**:
1. 在 `update_task_started()` 中更新 `_task_status_store[task_id]["status"] = "running"`
2. 在 `update_task_completed()` 中更新 `_task_status_store[task_id]["status"] = "completed"`
3. 在 `update_task_failed()` 中更新 `_task_status_store[task_id]["status"] = "failed"`
4. 在 `cancel_task()` 中更新 `_task_status_store[task_id]["status"] = "cancelled"`

### 方案3: 清理已完成任务

**思路**: 任务完成后从 `_task_status_store` 中移除

**修改**:
1. 在 `update_task_completed()` 中删除 `_task_status_store[task_id]`
2. 在 `update_task_failed()` 中删除 `_task_status_store[task_id]`
3. 运行列表只显示真正运行中的任务

## 建议实现

**推荐方案2（同步状态更新）**，因为：
1. 保持 `_task_status_store` 作为运行任务列表的数据源
2. 确保状态一致性
3. 可以追踪任务的完整生命周期
4. 便于调试和监控
