# 历史数据采集任务生成问题检查计划

## 问题描述

启动调度器并点击"强制立即采集"后，系统并未按照当前启用的数据源配置生成历史数据采集任务。

## 检查目标

1. 检查前端"立即采集"按钮调用的API和参数
2. 检查后端API是否正确处理`trigger_immediate`动作
3. 检查是否获取了启用的数据源列表
4. 检查是否为每个启用的数据源生成了采集任务
5. 检查任务是否正确提交到统一调度器

## 问题分析

### 1. 前端调用分析

**位置**: `web-static/data-collection-monitor.html` 第 1633-1654 行

```javascript
async function triggerImmediateCollection(force = false) {
    const confirmText = force ? '强制立即采集将跳过时间窗口检查，确定要执行吗？' : '确定要立即采集吗？';
    if (!confirm(confirmText)) return;

    try {
        const response = await fetch(`${API_BASE_URL}/api/v1/data/scheduler/control`, {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({ action: 'trigger_immediate', force: force })  // <-- 发送trigger_immediate动作
        });
        const result = await response.json();

        if (result.success) {
            showNotification('采集任务已触发', 'success');
            refreshAllData();
        } else {
            showNotification(result.message || '采集失败', 'warning');
        }
    } catch (error) {
        showNotification('采集失败', 'error');
    }
}
```

**问题发现**: 前端发送了 `action: 'trigger_immediate'`，但后端API可能没有正确处理这个动作。

### 2. 后端API分析

**位置**: `src/gateway/web/datasource_routes.py` 第 211-265 行

```python
@router.post("/api/v1/data/scheduler/control")
async def control_scheduler(request: dict):
    """控制调度器启动/停止（使用统一调度器）"""
    try:
        action = request.get("action", "status")
        
        # 使用统一调度器（符合架构设计）
        from src.distributed.coordinator.unified_scheduler import get_unified_scheduler
        from src.distributed.registry import get_unified_worker_registry, WorkerType
        
        scheduler = get_unified_scheduler()
        registry = get_unified_worker_registry()

        if action == "start":
            # ... 启动逻辑
        elif action == "stop":
            # ... 停止逻辑
        else:  # <-- 注意：这里没有处理 trigger_immediate 动作
            # 获取数据采集器数量
            data_collectors = registry.get_workers_by_type(WorkerType.DATA_COLLECTOR)
            
            return {
                "success": True, 
                "action": "status", 
                "running": scheduler._running,
                "scheduler_type": "unified_scheduler",
                "data_collectors_count": len(data_collectors)
            }
```

**问题确认**: ✅ **后端API没有处理 `trigger_immediate` 动作**

当 `action` 为 `trigger_immediate` 时，代码进入了 `else` 分支，只返回了状态信息，没有实际触发采集任务。

### 3. 正确的工作流程分析

根据 `data_collection_scheduler_manager.py` 的代码，正确的任务生成流程应该是：

```
1. 获取所有启用的数据源
   - 调用 get_data_source_config_manager().get_data_sources()
   - 过滤 enabled = True 的数据源

2. 为每个启用的数据源生成采集任务
   - 调用 _submit_collection_task(source_id, source_config)
   - 提交到统一调度器

3. 任务提交到统一调度器
   - scheduler.submit_task(TaskType.DATA_COLLECTION, ...)
```

### 4. 问题根因

**根因**: `control_scheduler` API 缺少 `trigger_immediate` 动作的处理逻辑。

当前实现只处理了：
- `start` - 启动调度器
- `stop` - 停止调度器
- `status`（默认）- 获取状态

缺少：
- `trigger_immediate` - 立即触发采集任务

## 修复方案

### 方案1: 在control_scheduler中添加trigger_immediate处理

```python
@router.post("/api/v1/data/scheduler/control")
async def control_scheduler(request: dict):
    """控制调度器启动/停止（使用统一调度器）"""
    try:
        action = request.get("action", "status")
        force = request.get("force", False)  # 获取force参数
        
        from src.distributed.coordinator.unified_scheduler import get_unified_scheduler
        from src.distributed.registry import get_unified_worker_registry, WorkerType
        
        scheduler = get_unified_scheduler()
        registry = get_unified_worker_registry()

        if action == "start":
            # ... 现有代码
        elif action == "stop":
            # ... 现有代码
        elif action == "trigger_immediate":  # <-- 添加这个分支
            # 获取启用的数据源
            from src.gateway.web.data_source_config_manager import get_data_source_config_manager
            config_manager = get_data_source_config_manager()
            sources = config_manager.get_data_sources()
            enabled_sources = [s for s in sources if s.get("enabled", False)]
            
            if not enabled_sources:
                return {
                    "success": False,
                    "action": "trigger_immediate",
                    "message": "没有启用的数据源",
                    "tasks_created": 0
                }
            
            # 为每个启用的数据源创建采集任务
            tasks_created = 0
            for source in enabled_sources:
                try:
                    # 准备任务数据
                    task_data = {
                        "source_id": source["id"],
                        "source_config": source,
                        "collection_type": "immediate",
                        "force": force,
                        "submitted_at": datetime.now().isoformat()
                    }
                    
                    # 提交任务到统一调度器
                    task_id = scheduler.submit_task(
                        task_type=TaskType.DATA_COLLECTION,
                        data=task_data,
                        priority=TaskPriority.HIGH if force else TaskPriority.NORMAL,
                        metadata={
                            "source_id": source["id"],
                            "submitted_by": "immediate_trigger",
                            "force": force
                        }
                    )
                    
                    tasks_created += 1
                    logger.info(f"✅ 立即采集任务已创建: {task_id} (数据源: {source['id']})")
                    
                except Exception as e:
                    logger.error(f"为数据源 {source['id']} 创建任务失败: {e}")
            
            return {
                "success": True,
                "action": "trigger_immediate",
                "message": f"已创建 {tasks_created} 个采集任务",
                "tasks_created": tasks_created,
                "sources_count": len(enabled_sources),
                "force": force
            }
        else:
            # ... 现有status代码
```
