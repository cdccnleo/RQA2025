# 数据采集自动调度功能实施计划

## 目标
实现数据采集的自动调度功能，根据数据源配置的采集频率自动生成采集任务，并在采集成功后更新最后测试时间。

## 问题总结
1. **调度器运行但无任务执行**: 调度器状态 `running: true`，但 `total_tasks: 0`
2. **最后测试时间未更新**: 已启用的2个数据源最后测试时间停留在2月15日
3. **缺少自动任务生成**: 调度器只负责任务路由，不根据采集频率生成新任务
4. **rate_limit未解析**: 采集频率配置（如"1次/天"）未被解析和使用

## 实施步骤

### 阶段1: 创建采集频率解析器
**文件**: `src/gateway/web/rate_limit_parser.py`

**功能**:
- 解析 `rate_limit` 字符串（如"1次/天"、"100次/分钟"）
- 转换为秒数间隔
- 支持多种格式：
  - "X次/天" → 86400秒
  - "X次/小时" → 3600秒
  - "X次/分钟" → 60秒
  - "X秒/次" → X秒

**接口**:
```python
def parse_rate_limit(rate_limit_str: str) -> int:
    """解析采集频率，返回间隔秒数"""

    """检查是否应该执行采集"""
```

### 阶段3: 修改数据采集工作器
**文件**: `src/distributed/coordinator/data_collector_worker.py`

**修改内容**:
- 在 `_complete_task` 方法中添加更新 `last_test` 的逻辑
- 采集成功后调用配置管理器更新数据源配置
- 记录采集历史

**修改点**:
```python
def _complete_task(self, task_id: str, result: Any = None, error: str = None):
    """完成任务并更新最后采集时间"""
    # 1. 完成任务
    scheduler.complete_task(task_id, result, error)
    
    # 2. 更新数据源的 last_test
    if result and not error:
        self._update_last_collection_time(source_id)
        
    # 3. 记录采集历史
    self._record_collection_history(source_id, result, error)
```

## 技术要点

### 1. 线程安全
- DataCollectionSchedulerManager 使用独立线程
- 更新配置时使用锁机制
- 避免与手动测试冲突

### 2. 任务去重
- 使用任务ID或数据源ID+时间戳去重
- 检查任务队列中是否已有相同数据源的任务
- 避免重复采集

### 3. 错误处理
- 采集失败时记录错误信息
- 连续失败时暂停自动调度
- 提供手动恢复机制

### 4. 性能考虑
- 每分钟检查一次，避免过于频繁
- 批量提交任务，减少调度开销
- 异步执行，不阻塞主线程
