## 六、修复建议

### 6.1 风险控制修复（高优先级）

**建议1**: 修改前端默认值

```javascript
// riskAssessmentCheck 默认勾选
const riskAssessment = document.getElementById('riskAssessmentCheck').checked; // 默认true

// anomalyDetectionCheck 默认勾选
const anomalyDetection = document.getElementById('anomalyDetectionCheck').checked; // 默认true
```

**建议2**: API增加风险控制验证

```python
# 在model_training_routes.py中增加
deployment = config.get("deployment", {})
risk_management = deployment.get("risk_management", {})

if not risk_management.get("assess_risk"):
    logger.warning("风险评估未启用，强制启用")
    risk_management["assess_risk"] = True

if not risk_management.get("anomaly_detection"):
    logger.warning("异常检测未启用，强制启用")
    risk_management["anomaly_detection"] = True
```

### 6.2 数据安全修复（中优先级）

**建议3**: 增加股票白名单验证

```python
# 在model_training_routes.py中增加
ALLOWED_STOCKS = {'000001', '000002', ...}  # 白名单

symbols = config.get("symbols", [])
invalid_symbols = set(symbols) - ALLOWED_STOCKS
if invalid_symbols:
    raise HTTPException(status_code=400,
        detail=f"不支持的股票代码: {invalid_symbols}")
```

**建议4**: 增加日期范围限制

```python
# 在model_training_routes.py中增加
MAX_DATE_RANGE_DAYS = 365 * 5  # 最多5年

time_range = config.get("time_range", {})
start_date = time_range.get("start_date")
end_date = time_range.get("end_date")

if start_date and end_date:
    days = (end_date - start_date).days
    if days > MAX_DATE_RANGE_DAYS:
        raise HTTPException(status_code=400,
            detail=f"日期范围不能超过{MAX_DATE_RANGE_DAYS}天")
```
