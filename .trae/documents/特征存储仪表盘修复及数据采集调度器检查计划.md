# 特征存储仪表盘修复及数据采集调度器检查计划

## 任务概述

1. **特征存储仪表盘修复**：检查并修复特征工程监控页面中特征存储仪表盘显示硬编码特征名称（如 `feature_任务ID`）而非实际特征名称（如 `SMA_5`、`EMA_10`）的问题

2. **数据采集调度器检查**：检查数据采集调度器在完成数据采集后是否正确更新最后测试时间，并深入检查当前数据采集配置是否存在未进行数据采集的情况

## 问题分析

### 问题1：特征存储仪表盘显示硬编码名称

**当前实现状态**：
- ✅ 已创建 `feature_store` 表（特征存储表）
- ✅ `feature_task_executor.py` 在任务完成时调用 `save_features_to_store()` 保存特征列表
- ✅ `feature_engineering_service.py` 的 `get_features()` 函数优先从 `feature_store` 表读取特征
- ✅ 特征名称已从硬编码 `feature_{task_id}` 改为实际的特征名称（如 SMA_5）

**已实现的功能**：
```python
# feature_task_persistence.py 第606-648行
# 解析特征名称，提取特征类型和参数
match = re.match(r'([A-Za-z]+)_(\d+)', feature_name)
if match:
    feature_type = match.group(1).upper()
    parameters['period'] = int(match.group(2))
```

**潜在问题**：
- 特征名称解析可能不完整（只处理了 `XXX_N` 格式）
- 部分动态生成的特征名称可能无法正确解析

### 问题2：数据采集调度器更新最后测试时间

**当前实现状态**：
- ✅ `data_collection_orchestrator.py` 第583-646行实现了 `_update_data_source_last_test_time` 方法
- ✅ 数据采集成功/失败后都会调用该方法更新 `last_test` 字段
- ✅ 使用并发控制器保护配置更新操作

**实现逻辑**：
```python
# data_collection_orchestrator.py 第166行
await self._update_data_source_last_test_time(source_id, success=True)
# 数据采集成功后更新
await self._update_data_source_last_test_time(source_id, success=False)
# 数据采集失败后更新
```
