# 策略回测分析页面历史回测记录仪表盘更新计划

## 任务目标
在策略回测分析页面（strategy-backtest.html）运行回测后，除了更新现有的仪表盘（累计收益率走势、风险收益分布、策略多维度对比、详细性能指标、交易记录明细）外，还需要更新历史回测记录仪表盘。

## 当前状态
运行回测后，以下仪表盘会更新：
- ✅ 累计收益率走势
- ✅ 风险收益分布
- ✅ 策略多维度对比
- ✅ 详细性能指标
- ✅ 交易记录明细
- ❌ 历史回测记录（需要添加）

## 实现方案

### 方案：在回测完成后自动刷新历史回测记录列表

在运行回测的 JavaScript 函数中，在回测成功后调用 `loadBacktestHistory()` 函数来刷新历史回测记录仪表盘。

## 实施步骤

### 第一阶段：定位回测完成后的处理逻辑（10分钟）
1. 在 strategy-backtest.html 中找到运行回测的函数
2. 找到回测成功后的回调处理逻辑
3. 确定历史回测记录列表的更新函数

### 第二阶段：添加历史回测记录刷新逻辑（15分钟）
1. 在回测成功后调用 `loadBacktestHistory()` 函数
2. 确保刷新时显示加载状态
3. 添加错误处理

### 第三阶段：测试验证（15分钟）
1. 运行回测，验证历史回测记录列表是否自动更新
2. 检查新回测记录是否正确显示
3. 验证无错误发生

## 检查清单
- [ ] 定位到回测完成后的处理函数
- [ ] 添加 `loadBacktestHistory()` 调用
- [ ] 测试回测后历史记录自动刷新
- [ ] 验证新回测记录正确显示
- [ ] 验证无 JavaScript 错误

## 预期结果
运行回测后，历史回测记录仪表盘会自动刷新，显示最新的回测记录，无需手动刷新页面。

## 风险与缓解
| 风险 | 缓解措施 |
|------|----------|
| 刷新时数据不一致 | 确保在回测数据保存完成后再刷新 |
| 性能问题 | 使用异步加载，避免阻塞UI |
| 重复刷新 | 添加防抖或状态检查 |
