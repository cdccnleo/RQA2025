### 3. 训练任务执行器分析

**文件**: `src/gateway/web/training_job_executor.py`

**当前实现问题**:

1. **模拟训练而非真实训练** (第511-564行):
```python
# 模拟训练过程，更新进度
for epoch in range(1, epochs + 1):
    await asyncio.sleep(0.1)  # 模拟每个epoch的处理时间
    
    # 模拟准确率和损失值
    simulated_accuracy = min(0.95, 0.6 + (epoch / epochs) * 0.35)
    simulated_loss = max(0.1, 0.5 - (epoch / epochs) * 0.4)
```

2. **缺少量化交易特定功能**:
   - 没有回测集成
   - 没有夏普比率、最大回撤等量化指标
   - 没有策略信号生成
   - 没有交易模拟

3. **模型保存为模拟对象** (第580行):
```python
model_object = self._create_mock_model(model_type, result, feature_columns)
```

### 4. 量化交易模型设计需求

根据量化交易系统设计，模型训练应该包含:

#### 4.1 模型类型适配
- **LSTM**: 时序预测，适合价格趋势预测
- **Transformer**: 注意力机制，适合多因子模型
- **CNN**: 图像识别，适合K线形态识别
- **XGBoost/LightGBM**: 梯度提升，适合特征重要性分析

#### 4.2 量化交易特定功能
- **特征工程**: 技术指标（MACD、RSI、布林带等）
- **标签生成**: 未来收益率、趋势方向
- **回测集成**: 训练后自动回测
- **量化指标**: 夏普比率、最大回撤、胜率、盈亏比
- **策略信号**: 生成买卖信号

#### 4.3 训练流程
1. 数据加载（历史行情数据）
2. 特征工程（计算技术指标）
3. 数据分割（训练集/验证集/测试集）
4. 模型训练
5. 回测验证
6. 量化指标计算
7. 模型保存

## 建议改进方案

### 阶段1: 修复模拟训练问题（高优先级）

1. **集成真实ML框架**
   - 使用 PyTorch/TensorFlow 实现真实训练
   - 或使用 scikit-learn 实现传统机器学习模型

2. **实现特征工程**
   - 集成技术指标计算（TA-Lib）
   - 实现特征标准化/归一化

3. **实现真实训练流程**
   - 数据加载和预处理
   - 模型训练和验证
   - 早停机制

### 阶段2: 添加量化交易功能（中优先级）

1. **回测集成**
   - 训练后自动回测
   - 计算回测指标

2. **量化指标计算**
   - 夏普比率
   - 最大回撤
   - 胜率、盈亏比
   - 年化收益率

3. **策略信号生成**
   - 生成买卖信号
   - 信号可视化

### 阶段3: 完善页面功能（低优先级）

1. **训练配置增强**
   - 添加更多超参数
   - 支持自定义损失函数
   - 支持自定义评估指标

2. **可视化增强**
   - 训练曲线实时显示
   - 回测结果可视化
   - 特征重要性分析

3. **模型对比**
   - 多模型性能对比
   - 模型选择建议
