# 策略回测页面数据加载和一致性问题修复计划

## 问题诊断

通过深度代码检查，发现以下关键问题：

### 1. 回测运行后数据更新不完整
**问题位置**：`runBacktest()` 函数
**问题描述**：回测成功后调用 `updateDashboard(results)`，但 `updateDashboard` 内部已调用 `updateCharts()` 和 `updateTradesDisplay()`，逻辑正确。但需要检查后端返回的数据格式。

### 2. 累计收益率走势图数据问题
**问题位置**：`updateReturnsChart()` 函数
**问题描述**：
- 依赖 `result.equity_curve` 数据
- 如果后端返回的数据中没有这个字段，图表无法显示
- 虽然有降级处理使用 `result.returns`，但需要确保数据格式正确
- 时间标签生成逻辑可能有问题（使用 `start_date` + i天）

### 3. 数据一致性问题
**问题位置**：`updateDashboard()` 函数
**问题描述**：
- 计算平均年化收益时过滤了0值：`results.map(r => r.annualized_return || 0).filter(r => r !== 0)`
- 但显示详细指标时直接使用 `results[0]`，没有过滤
- 这可能导致显示不一致

### 4. 交易记录无数据
**问题位置**：`updateTradesDisplay()` 函数
**问题描述**：
- 虽然函数被调用，但如果 `result.trades` 为空或不存在，交易记录不会显示
- 需要检查后端是否返回交易数据

### 5. 风险收益分布图数据问题
**问题位置**：`updateRiskReturnChart()` 函数
**问题描述**：
- 依赖 `result.annual_volatility` 或 `result.metrics.volatility`
- 如果这些字段不存在，会从权益曲线计算，但计算逻辑可能不准确

## 修复方案

### 1. 增强数据验证和调试
- 在 `updateDashboard` 中添加详细的调试日志
- 验证每个结果对象的关键字段是否存在
- 输出数据加载状态

### 2. 修复数据一致性问题
- 统一使用过滤后的数据计算平均值
- 确保详细指标显示与概览卡片一致
- 处理缺失数据的默认值

### 3. 优化图表数据更新
- 增强 `equity_curve` 数据的验证
- 添加备用数据源（如 `daily_returns`）
- 修复时间标签生成逻辑

### 4. 增强交易记录处理
- 添加交易数据存在性检查
- 输出交易数据加载日志
- 优化空状态提示

### 5. 添加数据加载状态提示
- 显示数据加载进度
- 提示用户数据加载状态
- 错误时显示重试按钮
