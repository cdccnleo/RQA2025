# 策略性能评估页面检查计划

## 目标
检查策略性能评估(strategy-performance-evaluation)页面的策略对比实现逻辑，以及各策略性能字段(总收益率、夏普比率、最大回撤、年化收益、胜率)的取数逻辑。

## 检查内容

### 1. 策略对比实现逻辑
- 策略列表加载逻辑
- 策略对比表格渲染逻辑
- 策略筛选和排序逻辑
- 策略详情查看逻辑

### 2. 性能字段取数逻辑
- 总收益率(total_return)数据来源
- 夏普比率(sharpe_ratio)数据来源
- 最大回撤(max_drawdown)数据来源
- 年化收益率(annual_return)数据来源
- 胜率(win_rate)数据来源

### 3. 数据流检查
- 后端API返回的数据结构
- 前端数据处理和转换逻辑
- 数据缓存和更新机制

## 实施步骤

### 第一阶段：代码审查（30分钟）
1. 检查策略对比表格渲染代码
2. 检查性能字段绑定逻辑
3. 检查数据加载和处理逻辑
4. 检查后端API数据返回格式

### 第二阶段：数据流追踪（30分钟）
1. 追踪从API到前端的完整数据流
2. 验证各性能字段的数据来源
3. 检查数据转换和计算逻辑
4. 确认数据一致性

### 第三阶段：问题修复（如有需要）（30分钟）
1. 修复发现的数据绑定问题
2. 修复字段映射错误
3. 优化数据展示逻辑
4. 验证修复效果

## 预期产出
1. 策略对比实现逻辑文档
2. 性能字段取数逻辑文档
3. 问题修复报告（如有）
4. 优化建议
