# 问题分析

通过分析代码，我发现了训练任务队列始终等待未被调度的可能原因：

1. **任务提交逻辑不完整**：训练任务可能没有被正确提交到调度器的任务队列中
2. **调度器任务分配逻辑问题**：调度器可能没有正确处理任务队列中的任务
3. **工作节点任务获取逻辑问题**：工作节点可能没有正确从调度器获取任务
4. **任务状态更新不一致**：任务状态可能没有被正确更新，导致任务被卡住

# 修复计划

## 1. 检查训练任务提交逻辑
- 检查模型训练路由中的任务提交代码
- 确保训练任务被正确创建并提交到调度器的任务队列
- 验证任务提交时的参数和格式是否正确

## 2. 修复调度器任务分配逻辑
- 检查 `TaskScheduler.get_task` 方法的实现
- 确保任务队列中的任务能够被正确取出并分配给工作节点
- 验证任务状态更新逻辑是否正确

## 3. 优化工作节点任务获取逻辑
- 检查 `TrainingJobExecutor._execution_loop` 方法
- 确保工作节点能够持续从调度器获取任务
- 验证工作节点的心跳更新是否正常

## 4. 添加任务调度监控和日志
- 在调度器和工作节点中添加详细的日志
- 监控任务队列状态和任务分配情况
- 确保任务调度过程可观测

## 5. 测试修复后的系统
- 验证训练任务能够被正确提交到队列
- 验证调度器能够将任务分配给工作节点
- 验证工作节点能够执行任务并更新状态
- 验证任务完成后能够被正确清理

# 预期结果

修复后，当模型训练调度器启动且存在活跃工作节点时，训练任务队列中的任务应该能够被正常调度和执行，不再出现始终等待的情况。
