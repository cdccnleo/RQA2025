# 数据采集仪表盘迁移计划

## 目标
将数据源配置管理页面（data-sources-config.html）中涉及数据采集及调度的仪表盘功能迁移到数据采集监控仪表板页面（data-collection-monitor.html），实现功能职责的清晰分离。

## 背景
根据用户要求，需要明确两个页面的职责边界：
- **data-collection-monitor.html**: 负责数据采集各项功能及监控（核心职责）
- **data-sources-config.html**: 负责数据源配置管理（核心职责）

当前 data-sources-config.html 中包含了一些数据采集相关的仪表盘功能，需要迁移到 data-collection-monitor.html。

## 迁移范围

### 1. 识别需要迁移的功能
在 data-sources-config.html 中检查以下功能是否需要迁移：
- [ ] 数据源状态监控仪表盘
- [ ] 采集统计信息展示
- [ ] 实时采集状态显示
- [ ] 采集调度相关功能
- [ ] 数据质量监控指标

### 2. 保留在 data-sources-config.html 的功能
- [ ] 数据源列表展示
- [ ] 数据源CRUD操作
- [ ] 批量操作功能
- [ ] 测试连接功能
- [ ] 自选股池配置
- [ ] 数据类型配置
- [ ] 基础状态指示（简单状态）

## 检查步骤

### 步骤1：详细检查 data-sources-config.html 中的监控功能
**检查内容**:
- 页面中的所有图表和仪表盘
- 监控相关的JavaScript函数
- 数据采集统计展示
- 实时状态更新功能

**检查命令**:
```bash
# 查找监控相关的代码
grep -n "dashboard\|chart\|monitor\|统计\|监控" web-static/data-sources-config.html

# 查找数据采集相关的函数
grep -n "collect\|采集\|调度" web-static/data-sources-config.html
```

### 步骤2：详细检查 data-collection-monitor.html 的现有功能
**检查内容**:
- 现有的监控仪表盘
- 采集任务展示
- 历史记录功能
- 数据质量监控

**检查命令**:
```bash
# 查看监控页面结构
grep -n "dashboard\|chart\|monitor\|task\|history" web-static/data-collection-monitor.html
```

### 步骤3：制定迁移清单
根据检查结果，列出需要迁移的具体功能模块：

| 功能模块 | 当前位置 | 迁移目标 | 优先级 |
|---------|---------|---------|--------|
| ? | data-sources-config | data-collection-monitor | ? |

## 迁移实施步骤

### 阶段1：功能识别与评估
1. 详细列出 data-sources-config.html 中的所有监控功能
2. 评估每个功能是否属于数据采集监控范畴
3. 确定需要迁移的功能清单

### 阶段2：代码迁移
1. 将监控相关的HTML代码迁移到 data-collection-monitor.html
2. 将监控相关的JavaScript函数迁移
3. 将监控相关的CSS样式迁移
4. 更新API调用路径

### 阶段3：功能增强
1. 在 data-collection-monitor.html 中增强监控功能
2. 添加数据源筛选功能
3. 添加实时刷新功能
4. 添加数据质量分析功能

### 阶段4：清理与优化
1. 从 data-sources-config.html 中移除已迁移的功能
2. 优化页面加载性能
3. 更新页面间的跳转链接
4. 测试功能完整性
