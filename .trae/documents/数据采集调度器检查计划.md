## 1. 检查结果摘要

经过对 `src/gateway/web/api.py` 和相关文件的检查，发现了以下关键问题：

### 🔴 严重问题

#### 1.1 调度器API端点不完整
- **现状**: 后端仅提供了 `/api/v1/scheduler/status` 一个端点
- **前端期望**: 30+ 个端点（dashboard、auto-collection、tasks、analytics等）
- **影响**: 前端 `data-collection-monitor.html` 无法正常显示数据

#### 1.2 统一调度器与数据采集调度器混淆
- **现状**: 代码中同时存在两种调度器
  - `src.core.orchestration.business_process.service_scheduler` - 数据采集调度器
  - `src.distributed.coordinator.unified_scheduler` - 分布式统一调度器
- **问题**: 两种调度器没有正确集成，可能导致冲突

#### 1.3 数据采集自动调度未正确配置
- **现状**: 自动采集逻辑分散在多个地方
- **问题**: 没有统一的自动采集任务注册机制

@app.post("/api/v1/data/scheduler/auto-collection/start")
async def start_auto_collection():
    """启动自动采集"""
    try:
        from src.core.orchestration.business_process.service_scheduler import get_data_collection_scheduler
        scheduler = get_data_collection_scheduler()
        
        if scheduler.is_running():
            return {"success": True, "message": "自动采集已在运行中"}
        
        success = await scheduler.start()
        return {
            "success": success,
            "message": "自动采集已启动" if success else "启动失败"
        }
    except Exception as e:
        logger.error(f"启动自动采集失败: {e}")
        raise HTTPException(status_code=500, detail=str(e))

@app.post("/api/v1/data/scheduler/auto-collection/stop")
async def stop_auto_collection():
    """停止自动采集"""
    try:
        from src.core.orchestration.business_process.service_scheduler import get_data_collection_scheduler
        scheduler = get_data_collection_scheduler()
        
        if not scheduler.is_running():

### 3.2 方案B: 创建独立路由模块

创建 `src/gateway/web/scheduler_routes.py` 文件，将所有调度器相关路由独立出来。

#### 优点
- 代码结构更清晰
- 便于维护和测试
- 符合现有路由拆分模式（如 `datasource_routes.py`）

#### 缺点
- 需要修改 `api.py` 导入新路由
- 工作量相对较大

### 3.3 方案C: 统一两种调度器

整合 `service_scheduler` 和 `unified_scheduler`，使用统一的调度器架构。

#### 建议架构
