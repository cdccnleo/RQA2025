# 根据量化策略开发流程优化训练任务创建字段设计

## 现状分析

### 当前字段设计
- **前端表单字段**：
  - 模型类型（LSTM、Transformer、CNN、RandomForest、XGBoost、LightGBM）
  - 学习率
  - 批次大小
  - 训练轮数
  - 任务描述（可选）
- **后端处理**：
  - `model_type`：模型类型
  - `config`：包含超参数配置

### 量化策略开发流程要求
从dashboard面板的量化策略开发流程可以看到，完整流程包括：
1. **策略构思** → 2. **策略管理** → 3. **数据收集** → 4. **特征工程** → 5. **模型训练** → 6. **策略回测** → 7. **策略优化** → 8. **性能评估** → 9. **策略部署** → 10. **执行监控**

## 改进建议

### 1. 数据配置字段（对应流程步骤3和4）
- **训练数据来源**：下拉选择（历史数据、实时数据、自定义数据集）
- **数据预处理**：多选（标准化、归一化、特征缩放、缺失值处理）
- **特征选择方法**：下拉选择（None、PCA、互信息、递归特征消除）
- **时间范围**：日期选择器（开始日期、结束日期）

### 2. 模型配置字段（对应流程步骤5）
- **模型类型**：保持现有下拉选择
- **超参数配置**：
  - 学习率（保持现有）
  - 批次大小（保持现有）
  - 训练轮数（保持现有）
  - 优化器选择（SGD、Adam、RMSprop等）
  - 损失函数选择（根据模型类型自动调整）

### 3. 评估配置字段（对应流程步骤6和8）
- **验证集比例**：滑块（0.1-0.5，默认0.2）
- **评估指标**：多选（准确率、F1分数、AUC-ROC、均方误差等）
- **早停策略**：开关 + 耐心值输入
- **交叉验证**：开关 + K折输入

### 4. 计算资源配置（对应流程步骤5）
- **GPU使用**：开关
- **并行计算**：开关 + 线程数输入
- **内存限制**：数字输入（GB）

### 5. 部署配置字段（对应流程步骤9）
- **模型保存路径**：文本输入
- **版本控制**：开关
- **部署环境**：下拉选择（开发、测试、生产）

### 6. 风险管理配置（对应流程步骤10）
- **预测置信度阈值**：滑块（0-1，默认0.5）
- **模型风险评估**：开关
- **异常检测**：开关

### 7. 监控配置字段（对应流程步骤10）
- **日志级别**：下拉选择（DEBUG、INFO、WARNING、ERROR）
- **性能监控**：开关
- **告警阈值**：数字输入

## 技术实现

### 前端改进
- 重构创建任务模态框，使用分步表单或折叠面板组织字段
- 添加字段验证和默认值
- 实现条件显示（如根据模型类型显示不同的超参数）

### 后端改进
- 扩展`create_training_job`函数，支持新的配置字段
- 添加配置验证和处理逻辑
- 增强任务持久化，保存完整的配置信息
- 添加与数据层、特征层的集成

### 集成改进
- 与业务流程编排器集成，支持完整的模型训练生命周期
- 通过EventBus发布更详细的训练事件
- 与监控系统集成，提供实时训练状态监控

## 预期效果

通过这些改进，训练任务创建将更符合量化策略开发流程的要求，支持：
- 更灵活的数据配置，满足不同策略的数据需求
- 更精细的模型调优，提高模型性能
- 更完善的评估机制，确保模型质量
- 更严格的风险控制，符合金融行业要求
- 更便捷的部署管理，加速模型上线
- 更全面的监控体系，确保系统稳定运行
