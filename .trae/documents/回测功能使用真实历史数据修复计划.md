# 回测功能使用真实历史数据修复计划

## 问题诊断

通过深度代码检查，发现以下关键问题：

### 1. 回测服务使用模拟数据
**问题位置**：`src/gateway/web/backtest_service.py` 第528-540行
**问题描述**：
```python
# 生成模拟数据
import pandas as pd
from datetime import datetime as dt

date_range = pd.date_range(start=start_date, end=end_date)
mock_data = pd.DataFrame({
    'timestamp': date_range,
    'open': [100.0 + i * 0.1 for i in range(len(date_range))],
    'high': [100.1 + i * 0.1 for i in range(len(date_range))],
    'low': [99.9 + i * 0.1 for i in range(len(date_range))],
    'close': [100.0 + i * 0.1 for i in range(len(date_range))],
    'volume': [1000 + i * 10 for i in range(len(date_range))]
})
```
回测使用的是生成的模拟数据，而非已采集的真实历史数据。

### 2. 缺乏数据管理层集成
**问题描述**：
- 没有调用数据加载器（`src/data/core/data_loader.py`）
- 没有从数据湖（Data Lake）获取数据
- 没有使用数据源配置管理（`data_source_config_manager.py`）

### 3. 数据管理层架构
根据 `docs/architecture/data_layer_architecture_design.md`，数据管理层提供：
- 数据接入层（数据适配器、连接器、加载器）
- 数据处理层（验证器、转换器、处理器）
- 数据存储层（数据湖、缓存系统）

## 修复方案

### 1. 集成数据管理层加载历史数据
修改 `backtest_service.py`，使用数据管理层加载真实历史数据：
- 调用 `DataLoader` 从数据湖加载数据
- 使用 `DataSourceManager` 获取已配置的数据源
- 根据策略配置的时间范围加载对应的历史数据

### 2. 实现历史数据查询接口
创建或复用数据查询接口：
- 支持按时间范围查询
- 支持按股票代码查询
- 支持按数据类型查询（日线、分钟线等）

### 3. 数据验证和预处理
- 验证数据的完整性和准确性
- 处理缺失数据
- 数据格式转换（适配回测引擎需求）

### 4. 降级方案
当真实数据不可用时：
- 提示用户数据缺失
- 提供数据补全功能入口
- 可选：使用模拟数据并明确标注
