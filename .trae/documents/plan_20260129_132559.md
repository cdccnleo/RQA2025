# 📋 数据采集系统优化计划

## 1. 计划目标

基于之前的数据采集映射一致性修复，实现以下后续建议：

- ✅ **数据重新采集**：使用标准映射重新采集历史数据
- ✅ **自动化验证**：实现数据采集后的自动验证
- ✅ **监控机制**：建立数据质量监控系统
- ✅ **定期清理**：定期检查并清理不符合标准的数据

## 2. 实施计划

### 2.1 数据重新采集模块

**实现内容**：
- 创建 `src/core/orchestration/standard_data_collector.py`
- **集成已实现的AKShare服务** (`src/core/integration/akshare_service.py`)
- 实现标准映射的数据采集
- 支持批量采集和增量采集

**与现有系统的关系**：
- **集成现有AKShare服务**：直接使用 `get_akshare_service()` 获取全局实例
- **补充现有采集系统**：作为标准映射的采集补充
- **统一采集标准**：与日常增量采集和历史数据采集使用相同标准

**核心功能**：
- 使用标准数据源标识 `akshare_stock_a`
- 使用标准数据类型 `daily`/`weekly`/`monthly`
- 确保所有关键字段完整映射
- 支持错误重试和断点续传

### 2.2 自动化验证机制

**实现内容**：
- 扩展 `src/data/quality/unified_quality_monitor.py`
- 集成到现有的数据采集流程
- 实现采集后自动验证

**与现有系统的关系**：
- **集成现有验证器**：使用 `UnifiedQualityMonitor`
- **增强验证规则**：添加标准映射的验证规则
- **统一验证标准**：确保历史数据与增量数据使用相同验证标准

**核心功能**：
- 实时数据验证
- 质量指标计算
- 异常检测和告警
- 自动修复建议

### 2.3 监控机制部署

**实现内容**：
- 配置 `src/core/monitoring/data_quality_monitor.py`
- 集成Prometheus监控
- 实现数据质量仪表盘

**与现有系统的关系**：
- **集成现有监控**：使用 `get_historical_data_monitor()`
- **扩展监控指标**：添加数据质量相关指标
- **统一监控标准**：确保历史数据与增量数据使用相同监控标准

**核心功能**：
- 数据质量实时监控
- 历史趋势分析
- 阈值告警
- 自动健康检查

### 2.4 定期清理任务

**实现内容**：
- 创建 `src/data/maintenance/data_cleanup.py`
- 配置定时任务
- 实现数据质量检查和清理

**与现有系统的关系**：
- **补充现有维护**：作为现有数据维护的补充
- **统一清理标准**：确保历史数据与增量数据使用相同清理标准
- **集成现有存储**：直接操作 `akshare_stock_data` 表

**核心功能**：
- 定期检查数据一致性
- 清理不符合标准的数据
- 生成清理报告
- 备份清理前数据

## 3. 技术架构

### 3.1 数据流设计

```
┌─────────────────────┐     ┌──────────────────────┐     ┌─────────────────────┐
│ 标准数据采集器      │ ──> │ 数据质量验证器        │ ──> │ 数据持久化存储       │
│ (补充现有采集系统)   │     │ (增强现有验证系统)     │     │ (使用现有存储系统)   │
└─────────────────────┘     └──────────────────────┘     └─────────────────────┘
         ↑                          ↑                          ↑
         │                          │                          │
┌─────────────────────┐     ┌──────────────────────┐     ┌─────────────────────┐
│ 监控系统            │ <── │ 质量指标计算          │ <── │ 定期清理任务         │
│ (扩展现有监控)       │     │ (集成现有计算系统)     │     │ (补充现有维护系统)   │
└─────────────────────┘     └──────────────────────┘     └─────────────────────┘
```

### 3.2 集成点

- **AKShareService**：使用现有全局实例 (`get_akshare_service()`)
- **UnifiedQualityMonitor**：扩展现有验证器
- **HistoricalDataScheduler**：集成到现有调度系统
- **Prometheus**：使用现有监控系统
- **Grafana**：使用现有仪表盘系统
