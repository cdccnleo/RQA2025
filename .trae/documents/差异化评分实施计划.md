# 差异化评分实施计划

## 目标
实施基于特征类型的差异化评分方案，让质量分布真正反映特征的质量差异。

## 背景
- 当前所有特征使用统一评分 0.85，缺乏区分度
- 不同特征类型有不同的可靠性和稳定性
- 需要让质量分布仪表盘具有实际参考价值

## 实施方案

### 阶段1: 定义评分标准（1天）

#### 1.1 特征类型分类
根据技术指标的特性和可靠性，将特征分为以下类别：

| 类别 | 特征类型 | 基础评分 | 评分依据 |
|------|----------|----------|----------|
| 趋势类 | SMA, EMA | 0.90 | 计算简单，稳定性高，广泛应用 |
| 动量类 | RSI, MACD | 0.85 | 常用指标，效果良好，参数相对稳健 |
| 波动率类 | BOLL | 0.80 | 受异常值影响较大，需要更多数据验证 |
| 复杂指标 | KDJ | 0.82 | 计算复杂，参数敏感，但效果较好 |
| 成交量类 | OBV, VWAP | 0.78 | 受成交量异常影响，稳定性一般 |
| 其他 | 其他指标 | 0.80 | 默认评分 |

### 阶段2: 代码实现（2天）

#### 2.1 创建评分工具模块
**文件**: `src/features/quality/quality_scorer.py`

```python
"""
特征质量评分工具
提供基于特征类型的差异化评分
"""

# 特征类型到质量评分的映射
FEATURE_QUALITY_MAP = {
    # 趋势类 - 高稳定性
    'SMA': 0.90,
    'EMA': 0.90,
    'WMA': 0.88,
    
    # 动量类 - 良好效果
    'RSI': 0.85,
    'MACD': 0.85,
    'CCI': 0.83,
    
    # 波动率类 - 中等稳定性
    'BOLL': 0.80,
    'ATR': 0.80,
    
    # 复杂指标 - 参数敏感
    'KDJ': 0.82,
    'STOCH': 0.81,
    
    # 成交量类 - 受异常影响
    'OBV': 0.78,
    'VWAP': 0.78,
    'ADL': 0.77,
    
    # 默认评分
    'DEFAULT': 0.80
}


def get_feature_quality_score(feature_name: str) -> float:
    """
    根据特征名称获取质量评分
    
    Args:
        feature_name: 特征名称，如 "SMA_5", "RSI", "boll_upper"
    
    Returns:
        质量评分 (0.0-1.0)
    """
    # 提取基础特征名（去掉参数后缀）
    base_name = feature_name.split('_')[0].upper()
    
    # 处理 BOLL 的特殊情况（boll_upper, boll_middle, boll_lower）
    if base_name in ['BOLL', 'BOL']:
        return FEATURE_QUALITY_MAP.get('BOLL', FEATURE_QUALITY_MAP['DEFAULT'])
    
    # 处理 KDJ 的特殊情况（kdj_k, kdj_d, kdj_j）
    if base_name in ['KDJ']:
        return FEATURE_QUALITY_MAP.get('KDJ', FEATURE_QUALITY_MAP['DEFAULT'])
    
    # 处理 MACD 的特殊情况（macd_macd, macd_signal, macd_histogram）
    if base_name in ['MACD']:
        return FEATURE_QUALITY_MAP.get('MACD', FEATURE_QUALITY_MAP['DEFAULT'])
    
    # 从映射表获取评分
    return FEATURE_QUALITY_MAP.get(base_name, FEATURE_QUALITY_MAP['DEFAULT'])


def get_quality_category(score: float) -> str:
    """
    根据评分获取质量等级
    
    Args:
        score: 质量评分
    
    Returns:
        质量等级描述
    """
    if score >= 0.9:
        return "优秀"
    elif score >= 0.8:
        return "良好"
    elif score >= 0.7:
        return "一般"
    elif score >= 0.6:
        return "较差"
    else:
        return "差"


def calculate_quality_scores(features: list) -> dict:
    """
    为特征列表计算质量评分
    
    Args:
        features: 特征名称列表
    
    Returns:
        特征名到质量评分的字典
    """
    return {feature: get_feature_quality_score(feature) for feature in features}
```

#### 2.3 更新现有数据
**脚本**: `scripts/database/update_quality_scores.py`

```python
"""
更新现有特征的质量评分为差异化评分
"""

from src.gateway.web.postgresql_persistence import get_db_connection, return_db_connection
from src.features.quality.quality_scorer import get_feature_quality_score


def update_existing_quality_scores():
    """更新现有特征的质量评分"""
    conn = get_db_connection()
    cursor = conn.cursor()
    
    # 获取所有特征
    cursor.execute("SELECT feature_id, feature_name FROM feature_store")
    features = cursor.fetchall()
    
    updated_count = 0
    for feature_id, feature_name in features:
        # 计算新的质量评分
        new_score = get_feature_quality_score(feature_name)
        
        # 更新数据库
        cursor.execute(
            "UPDATE feature_store SET quality_score = %s WHERE feature_id = %s",
            (new_score, feature_id)
        )
        updated_count += 1
    
    conn.commit()
    cursor.close()
    return_db_connection(conn)
    
    print(f"✅ 已更新 {updated_count} 个特征的质量评分")
```
