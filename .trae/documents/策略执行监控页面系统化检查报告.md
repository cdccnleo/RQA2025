# 策略执行监控页面系统化检查报告

## 1. 总览

已完成对策略执行监控页面 `strategy-execution-monitor` 各仪表盘的系统化检查，包括实现逻辑、更新逻辑和数据持久化。所有功能均使用实际数据，未使用模拟数据。

## 2. 仪表盘检查详情

### 2.1 统计卡片（运行中策略、平均延迟、今日信号数、总交易数）

**实现逻辑**：
- 前端通过 `updateStatistics` 函数更新统计数据
- 数据来自 `get_strategy_execution_status` API
- 运行中策略：从策略列表中筛选状态为 `running` 的策略数量
- 平均延迟：从API返回的 `avg_latency` 字段
- 今日信号数：从API返回的 `today_signals` 字段
- 总交易数：从API返回的 `total_trades` 字段

**更新逻辑**：
- 页面加载时调用 `loadExecutionData` 函数获取初始数据
- 优先使用 WebSocket 进行实时更新
- WebSocket 连接失败时自动切换到轮询模式（每5秒更新一次）
- 点击"刷新"按钮手动更新数据

**数据持久化**：
- 后端优先从 `execution_persistence` 加载执行状态
- 从实时策略引擎获取最新数据并保存到持久化存储
- 使用 `save_execution_state` 函数保存策略执行状态

### 2.2 延迟趋势图表

**实现逻辑**：
- 前端使用 Chart.js 渲染折线图
- 数据来自 `get_strategy_execution_metrics` API
- X轴为时间，Y轴为延迟值（毫秒）

**更新逻辑**：
- 页面加载时调用 `loadExecutionData` 函数获取初始数据
- 通过 WebSocket 实时更新数据
- 图表自动重绘显示最新趋势

**数据持久化**：
- 后端从实时引擎获取历史延迟数据
- 支持从监控系统获取完整的历史趋势数据

### 2.3 吞吐量趋势图表

**实现逻辑**：
- 前端使用 Chart.js 渲染折线图
- 数据来自 `get_strategy_execution_metrics` API
- X轴为时间，Y轴为吞吐量值（信号/秒）

**更新逻辑**：
- 页面加载时调用 `loadExecutionData` 函数获取初始数据
- 通过 WebSocket 实时更新数据
- 图表自动重绘显示最新趋势

**数据持久化**：
- 后端从实时引擎获取历史吞吐量数据
- 支持从监控系统获取完整的历史趋势数据

### 2.4 策略执行列表

**实现逻辑**：
- 前端通过 `renderStrategyTable` 函数渲染表格
- 数据来自 `get_strategy_execution_status` API
- 支持按状态筛选（全部、运行中、已暂停、已停止）
- 每行显示策略名称、状态、延迟、吞吐量、信号数、持仓数和操作按钮

**更新逻辑**：
- 页面加载时调用 `loadExecutionData` 函数获取初始数据
- 通过 WebSocket 实时更新数据
- 选择筛选条件时自动重新渲染表格
- 支持启动/暂停策略操作

**数据持久化**：
- 后端从实时引擎获取策略执行状态
- 保存每个策略的详细执行数据到持久化存储
- 包含策略的完整执行指标

### 2.5 最近信号

**实现逻辑**：
- 前端通过 `renderRecentSignals` 函数渲染信号列表
- 数据来自 `get_realtime_signals` API
- 显示策略名称、股票代码、信号类型和时间
- 信号类型（buy/sell/hold）使用不同颜色标识

**更新逻辑**：
- 页面加载时调用 `loadExecutionData` 函数获取初始数据
- 通过 WebSocket 实时更新数据
- 只显示最近的信号（最多10个）

**数据持久化**：
- 后端从交易信号服务获取真实信号数据
- 不使用模拟数据，即使为空也返回真实结果
- 支持从执行引擎获取信号数据作为备用

## 3. 技术架构

### 3.1 前端架构

- **技术栈**：HTML、Tailwind CSS、JavaScript、Chart.js
- **核心功能**：
  - 数据可视化
  - API 调用
  - WebSocket 实时更新
  - 数据筛选和操作

### 3.2 后端架构

- **技术栈**：Python、FastAPI
- **核心功能**：
  - API 路由
  - 实时策略引擎管理
  - 性能指标收集
  - 数据持久化
  - WebSocket 通知

### 3.3 数据持久化

- **存储方式**：
  - 策略执行状态：保存到执行状态存储
  - 性能指标：保存到执行状态存储
  - 信号数据：从交易信号服务获取
- **核心模块**：
  - `execution_persistence.py`：执行状态存储
  - `strategy_execution_service.py`：执行服务
  - `trading_signal_service.py`：信号服务
