# 策略开发流程数据持久化迁移计划

## 1. 迁移目标

将量化策略开发流程中各个环节的数据统一持久化到 PostgreSQL 数据库，同时保留文件系统作为降级存储方案，确保数据可靠性和系统稳定性。

## 2. 迁移范围

### 2.1 覆盖环节
- ✅ 策略构思 (strategy-conception)
- ✅ 策略管理 (strategy-management)
- ✅ 策略回测 (strategy-backtest)
- ✅ 策略优化 (strategy-optimizer)
- ✅ 策略性能评估 (strategy-performance-evaluation)
- ✅ 策略部署 (strategy-lifecycle)
- ✅ 策略执行监控 (strategy-execution-monitor)

### 2.2 当前存储状况
- **策略构思**：仅文件系统 (`data/strategy_conceptions`)
- **策略回测**：双存储模式 (文件系统 + PostgreSQL)
- **其他环节**：预计主要使用文件系统存储

### 4.2 降级机制实现

```python
def save_with_fallback(data, table_name, primary_key, file_dir):
    """
    保存数据到数据库，失败后降级到文件系统
    
    Args:
        data: 要保存的数据
        table_name: 数据库表名
        primary_key: 主键字段名
        file_dir: 文件系统存储目录
    
    Returns:
        是否成功保存
    """
    try:
        # 尝试保存到数据库
        if save_to_database(data, table_name, primary_key):
            return True
    except Exception as e:
        logger.warning(f"数据库保存失败: {e}")
    
    # 降级到文件系统
    try:
        return save_to_filesystem(data, file_dir, primary_key)
    except Exception as e:
        logger.error(f"文件系统保存失败: {e}")
        return False
```
