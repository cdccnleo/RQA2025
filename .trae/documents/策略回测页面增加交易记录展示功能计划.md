# 策略回测页面增加交易记录展示功能计划

## 现状分析

### 后端支持
- `BacktestResponse` 模型已包含 `trades: list` 字段
- API已返回交易记录数据
- 交易数据格式包含：交易时间、交易类型、价格、数量、盈亏等

### 前端现状
- 页面已展示回测结果和图表
- 没有交易记录展示区域
- `updateDashboard` 函数未处理交易数据

## 功能设计

### 1. 交易记录展示区域
**位置**：详细性能指标下方
**布局**：
- 标题：交易记录明细
- 统计卡片：总交易次数、盈利次数、亏损次数、胜率
- 交易表格：展示每笔交易的详细信息

### 2. 交易记录表格
**表头**：
- 序号
- 交易时间
- 交易类型（买入/卖出）
- 成交价格
- 成交数量
- 交易成本
- 盈亏金额
- 盈亏比例
- 状态（盈利/亏损）

**功能**：
- 按时间排序（最新在前）
- 分页显示（每页20条）
- 筛选功能（只看盈利/亏损）
- 导出功能（CSV格式）

### 3. 交易统计信息
- 总交易次数
- 盈利交易次数
- 亏损交易次数
- 胜率
- 平均盈利
- 平均亏损
- 盈亏比

### 4. 交互功能
- 点击交易行查看详情
- 展开/收起交易记录区域
- 刷新交易数据

## 实施步骤

1. **HTML结构**
   - 在性能指标下方添加交易记录区域
   - 添加统计卡片（4个）
   - 添加交易表格容器
   - 添加分页控件

2. **CSS样式**
   - 交易记录区域样式
   - 统计卡片样式
   - 表格样式（斑马纹、悬停效果）
   - 状态标签样式（盈利绿色、亏损红色）

3. **JavaScript功能**
   - `updateTradesTable()` 函数：渲染交易表格
   - `updateTradeStats()` 函数：更新交易统计
   - `renderTradeRow()` 函数：渲染单行交易数据
   - `filterTrades()` 函数：筛选交易记录
   - `exportTrades()` 函数：导出交易记录

4. **数据集成**
   - 在 `updateDashboard()` 中调用交易更新函数
   - 处理交易数据格式转换
   - 计算交易统计信息

## 预期效果

- 用户可以看到每笔交易的详细信息
- 可以验证回测结果的准确性
- 可以分析交易策略的优劣
- 可以导出交易记录进行进一步分析

## 技术实现

### 交易数据格式示例
```json
{
  "trades": [
    {
      "timestamp": "2024-01-15T10:30:00",
      "type": "buy",
      "price": 100.5,
      "quantity": 100,
      "cost": 10.05,
      "pnl": 0,
      "pnl_percent": 0
    },
    {
      "timestamp": "2024-01-16T14:20:00",
      "type": "sell",
      "price": 105.2,
      "quantity": 100,
      "cost": 10.52,
      "pnl": 470.43,
      "pnl_percent": 4.67
    }
  ]
}
```

### 统计计算
- 总交易次数 = trades.length
- 盈利次数 = trades.filter(t => t.pnl > 0).length
- 亏损次数 = trades.filter(t => t.pnl < 0).length
- 胜率 = 盈利次数 / 总交易次数
- 平均盈利 = 盈利总额 / 盈利次数
- 平均亏损 = 亏损总额 / 亏损次数
- 盈亏比 = 平均盈利 / |平均亏损|
