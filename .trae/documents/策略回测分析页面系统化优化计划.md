# 策略回测分析页面系统化优化计划

## 一、现状分析

### 1.1 现有页面功能
- **strategy-management**: 策略管理（查看、编辑、删除、导出）
- **data-sources-config**: 数据源配置管理
- **feature-engineering-monitor**: 特征工程监控
- **model-training-monitor**: 模型训练监控
- **strategy-backtest**: 策略回测分析（当前需要优化）

### 1.2 现有后端API
- `/api/v1/strategy/conceptions` - 获取策略列表
- `/api/v1/backtest` - 回测列表和运行回测
- `/api/v1/backtest/run` - 运行回测（支持commission_rate参数）
- 交易费用：默认0.1%佣金率

### 1.3 当前问题
1. **前端**：strategy-backtest页面缺少活跃策略列表展示
2. **前端**：缺少交易费用设置界面
3. **前端**：缺少回测时间范围设置
4. **后端**：回测引擎使用模拟数据而非真实数据

## 二、优化目标

### 2.1 前端优化
1. **增加活跃策略列表展示**
   - 从strategy-management页面借鉴策略列表展示方式
   - 显示策略名称、状态、创建时间等关键信息
   - 支持策略选择和批量操作

2. **增加交易费用设置**
   - 佣金率设置（默认0.1%）
   - 滑点设置（默认0.1%）
   - 市场冲击成本（默认0.1%）
   - 总费用实时计算和显示

3. **增加回测时间范围设置**
   - 开始日期选择器
   - 结束日期选择器
   - 预设时间范围（最近1个月、3个月、6个月、1年、全部）
   - 日期范围验证

4. **优化仪表盘展示**
   - 活跃策略数：显示可回测的策略数量
   - 平均年化收益：基于选定策略的历史表现
   - 夏普比率：风险调整后的收益指标
   - 最大回撤：风险控制指标

### 2.2 后端优化
1. **扩展回测请求模型**
   ```python
   class BacktestRequest(BaseModel):
       strategy_id: str
       start_date: str
       end_date: str
       initial_capital: float = 100000.0
       commission_rate: float = 0.001  # 佣金率
       slippage: float = 0.001         # 滑点
       market_impact: float = 0.001    # 市场冲击
   ```

2. **改进回测引擎**
   - 连接真实数据源（从data-sources-config获取）
   - 实现真实的策略执行逻辑
   - 计算实际的交易成本和滑点
   - 生成真实的回测结果

3. **增加策略列表API**
   - 获取活跃策略列表
   - 支持分页和筛选
   - 返回策略基本信息和状态
