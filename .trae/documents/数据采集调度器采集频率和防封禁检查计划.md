# 数据采集调度器采集频率和防封禁检查计划

## 问题描述

1. **采集队列任务始终等待执行** - 最近活动仪表盘显示采集队列有2个任务，但始终处于等待执行状态
2. **采集频率控制检查** - 需要验证采集调度是否严格按照配置的采集频率执行
3. **防封禁机制检查** - 需要确保采集频率不会过于频繁导致数据源封禁

## 检查范围

### 1. 任务执行问题
- 检查统一调度器是否正确分配任务给工作节点
- 检查数据采集工作器是否正确接收和执行任务
- 检查任务状态是否正确更新
- 检查工作节点是否正常运行

### 2. 采集频率控制
- 检查采集频率解析器是否正确解析配置（如"1次/天"）
- 检查调度管理器是否正确计算下次采集时间
- 检查是否严格按照采集频率执行，避免过于频繁
- 检查任务去重机制是否正常工作

### 3. 防封禁机制
- 检查是否有最小采集间隔限制（如最小60秒）
- 检查是否有随机延迟机制
- 检查是否有请求频率限制
- 检查错误处理和重试机制

## 实施步骤

### 阶段1: 任务执行问题排查
1. 检查统一调度器的任务分配逻辑
2. 检查数据采集工作器的任务接收和执行
3. 检查任务队列状态和工作节点状态
4. 检查日志，查找任务未执行的原因

### 阶段2: 采集频率控制验证
1. 验证rate_limit_parser的解析逻辑
2. 验证DataCollectionSchedulerManager的调度逻辑
3. 验证should_collect函数的判断逻辑
4. 验证任务去重机制

### 阶段3: 防封禁机制检查
1. 检查最小采集间隔限制
2. 检查是否有随机延迟机制
3. 检查请求频率限制
4. 检查错误处理和重试机制

### 阶段4: 修复和优化
1. 修复任务执行问题
2. 优化采集频率控制
3. 添加防封禁机制（如需要）
4. 测试验证

## 预期输出
- 任务执行问题诊断报告
- 采集频率控制验证报告
- 防封禁机制检查报告
- 修复和优化方案
