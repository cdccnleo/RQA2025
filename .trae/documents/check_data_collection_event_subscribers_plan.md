# 检查数据采集完成后业务处理计划

## 目标
根据业务流程驱动架构设计，全面检查数据清洗、特征工程、策略回测等模块是否订阅 DATA_COLLECTION_COMPLETED 事件进行后续业务处理。

## 背景
- 数据源 `akshare_stock_a` 已成功采集今日数据
- 需要确保数据采集完成后，后续业务流程（数据清洗、特征工程、策略回测）能够正确响应
- 基于业务流程驱动架构，这些模块应该订阅 DATA_COLLECTION_COMPLETED 事件

## 检查范围

### 1. 数据清洗模块 (Data Cleaning)
**路径**: `src/data/quality/`, `src/data/validation/`
**检查内容**:
- 是否订阅 DATA_COLLECTION_COMPLETED 事件
- 是否有数据清洗处理逻辑
- 是否触发数据质量检查

### 2. 特征工程模块 (Feature Engineering)
**路径**: `src/features/`, `src/core/core_optimization/`
**检查内容**:
- 是否订阅 DATA_COLLECTION_COMPLETED 事件
- 是否有特征计算处理逻辑
- 是否更新特征缓存

### 3. 策略回测模块 (Strategy Backtesting)
**路径**: `src/backtest/`
**检查内容**:
- 是否订阅 DATA_COLLECTION_COMPLETED 事件
- 是否有策略回测触发逻辑
- 是否更新回测结果

### 4. 其他业务模块
**检查内容**:
- 数据同步服务 (src/data/sync/)
- 数据缓存服务 (src/data/cache/)
- 机器学习服务 (src/ml/)

## 实施步骤

### 步骤 1: 搜索现有事件订阅
- 搜索所有订阅 DATA_COLLECTION_COMPLETED 的代码
- 搜索所有订阅 DATA_COLLECTED 的代码（向后兼容）
- 记录每个订阅者的处理逻辑

### 步骤 2: 检查业务流程完整性
- 验证数据清洗流程是否完整
- 验证特征工程流程是否完整
- 验证策略回测流程是否完整

### 步骤 3: 识别缺失的订阅者
- 对比业务流程驱动架构设计
- 识别应该订阅但未订阅的模块
- 列出需要添加事件订阅的模块

### 步骤 4: 实现缺失的事件订阅
- 为数据清洗模块添加事件订阅
- 为特征工程模块添加事件订阅
- 为策略回测模块添加事件订阅

### 步骤 5: 验证和测试
- 验证事件发布和订阅机制
- 测试数据采集后的完整业务流程
- 确认所有模块正确响应事件

## 预期结果
- 所有业务模块正确订阅 DATA_COLLECTION_COMPLETED 事件
- 数据采集完成后自动触发后续业务流程
- 符合业务流程驱动架构设计要求

## 时间估计
- 步骤 1-2: 30分钟
- 步骤 3: 20分钟
- 步骤 4: 60分钟
- 步骤 5: 30分钟
- 总计: 约2.5小时
