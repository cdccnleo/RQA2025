# 数据源配置仪表盘回退测试时间功能计划

## 目标
在数据源配置管理页面（data-sources-config.html）的数据源配置仪表盘**批量操作按钮列表**中，增加"回退1天"批量操作按钮，允许用户选择多个数据源配置进行批量回退1天操作。

## 背景
用户需要能够批量回退数据源的最后测试时间，这样可以：
1. 同时测试多个数据源的数据采集时间边界条件
2. 批量测试数据源的增量采集逻辑
3. 批量验证历史数据补采功能
4. 提高测试效率，避免逐个操作

## 需求分析

### 功能需求
1. **复选框选择**：在每个数据源卡片上添加复选框，允许用户选择多个数据源
2. **批量操作栏**：在页面顶部或底部添加批量操作工具栏
3. **回退1天按钮**：在批量操作按钮列表中添加"回退1天"按钮
4. **批量操作**：点击按钮后，将选中的所有数据源的 `last_test` 时间向后回退1天
5. **操作反馈**：显示操作成功/失败的反馈提示，包括成功数量和失败数量
6. **全选功能**：支持全选/取消全选所有数据源

### 界面需求
- **复选框位置**：数据源卡片左上角或标题栏左侧
- **批量操作栏位置**：页面顶部（数据源列表上方）或固定在底部
- **按钮样式**：使用次要按钮样式（如灰色或浅蓝色）
- **按钮图标**：使用时钟或日历相关的图标（如 `fa-history` 或 `fa-calendar-minus`）
- **选中计数**：显示已选择的数据源数量

### 技术需求
1. 前端：维护选中数据源ID列表
2. 前端：调用后端批量 API 更新数据源的 `last_test` 字段
3. 后端：提供批量 API 端点支持修改多个数据源的 `last_test` 时间
4. 后端：支持批量操作的事务处理（部分失败时回滚或继续）

## 实现方案

### 1. 后端 API 设计

#### 新增批量 API 端点
```
POST /api/v1/data/sources/batch/rollback-test-time
```

#### 请求体
```json
{
  "source_ids": ["akshare_stock_a", "baostock_stock"],
  "days": 1
}
```

#### 响应格式
```json
{
  "success": true,
  "message": "批量回退完成",
  "data": {
    "total": 2,
    "success": 2,
    "failed": 0,
    "results": [
      {
        "source_id": "akshare_stock_a",
        "success": true,
        "last_test": "2026-02-22 14:30:00",
        "previous_last_test": "2026-02-23 14:30:00"
      },
      {
        "source_id": "baostock_stock",
        "success": true,
        "last_test": "2026-02-22 10:00:00",
        "previous_last_test": "2026-02-23 10:00:00"
      }
    ]
  }
}
```

### 2. 后端实现

#### 修改文件
- `src/gateway/web/datasource_routes.py`: 添加新的批量路由处理函数

#### 实现逻辑
1. 从请求体中获取数据源ID列表和回退天数
2. 加载数据源配置
3. 遍历每个数据源：
   - 验证数据源是否存在
   - 计算新的 `last_test` 时间
   - 更新数据源配置
4. 保存配置到文件/数据库
5. 返回批量操作结果（成功/失败列表）

### 3. 前端实现

#### 修改文件
- `web-static/data-sources-config.html`

#### 实现内容

##### 3.1 数据源卡片添加复选框
```html
<div class="flex items-center mb-2">
    <input type="checkbox" 
           class="source-checkbox mr-2" 
           data-source-id="${source.id}"
           onchange="updateBatchSelection()">
    <h3 class="text-lg font-semibold">${source.name}</h3>
</div>
```

##### 3.2 批量操作工具栏
```html
<!-- 批量操作工具栏 -->
<div id="batch-toolbar" class="hidden bg-gray-100 p-4 rounded-lg mb-4 flex items-center justify-between">
    <div class="flex items-center">
        <input type="checkbox" id="select-all" onchange="toggleSelectAll()" class="mr-2">
        <span class="text-gray-700">全选</span>
        <span class="ml-4 text-gray-600">已选择 <span id="selected-count">0</span> 个数据源</span>
    </div>
    <div class="flex space-x-2">
        <button onclick="batchRollbackTestTime()" 
                class="bg-gray-500 hover:bg-gray-600 text-white px-4 py-2 rounded text-sm transition duration-300 flex items-center"
                title="将选中数据源的最后测试时间回退1天">
            <i class="fas fa-history mr-2"></i>回退1天
        </button>
        <!-- 其他批量操作按钮 -->
    </div>
</div>
```

##### 3.3 JavaScript 功能实现
- `updateBatchSelection()`: 更新选中状态，显示/隐藏批量操作栏
- `toggleSelectAll()`: 全选/取消全选
- `batchRollbackTestTime()`: 批量回退测试时间
- `getSelectedSourceIds()`: 获取选中的数据源ID列表

## 风险评估

### 低风险
- 功能仅修改 `last_test` 字段，不影响实际数据采集
- 操作可逆，可以重新测试数据源恢复当前时间
- 批量操作提高效率，减少重复操作

### 注意事项
- 需要确保只有管理员或有权限的用户才能执行批量操作
- 建议添加操作确认对话框，防止误操作
- 批量操作失败时，需要明确告知用户哪些数据源操作失败
- 回退时间不应早于数据源创建时间
