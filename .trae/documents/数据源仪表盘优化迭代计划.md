# 数据源仪表盘优化迭代计划

## 目标
按照检查报告中的优化建议，对数据源配置管理页面的仪表盘进行迭代优化，实现监控数据的正常收集和展示。

## 优化内容

### 1. 添加性能指标记录点
**目标**: 在数据采集流程中添加性能指标记录

#### 1.1 数据源测试时记录指标
- **文件**: `src/gateway/web/datasource_routes.py`
- **位置**: `test_data_source` 函数
- **实现**: 在测试完成时记录延迟指标

#### 1.2 数据采集时记录指标
- **文件**: `src/data/collectors/akshare_collector.py`
- **位置**: `collect_and_save` 方法
- **实现**: 记录采集延迟、吞吐量、错误率

### 2. 集成监控服务
**目标**: 使用 DataCollectionMonitor 监控数据新鲜度和完整性

#### 2.1 数据采集监控
- **文件**: `src/infrastructure/monitoring/services/data_collection_monitor.py`
- **实现**: 在数据采集完成后调用监控检查

#### 2.2 定期监控任务
- **文件**: `src/gateway/web/api.py`
- **实现**: 启动定期监控任务，更新指标

### 3. 前端降级显示
**目标**: 当监控数据为空时提供更好的用户体验

#### 3.1 空数据处理
- **文件**: `web-static/data-sources-config.html`
- **实现**: 
  - 显示"暂无数据"而不是 0
  - 添加数据收集状态提示
  - 提供手动刷新按钮

#### 3.2 数据收集进度
- **实现**: 显示数据收集进度条或状态指示器

## 实施步骤

### 阶段1: 后端指标记录
1. 修改 `datasource_routes.py`，在测试数据源时记录延迟
2. 修改 `akshare_collector.py`，在采集时记录性能指标
3. 验证指标数据是否正确记录到 PerformanceMonitor

### 阶段2: 监控服务集成
1. 在数据采集流程中集成 DataCollectionMonitor
2. 添加定期监控任务到 api.py
3. 验证监控数据是否正确更新

### 阶段3: 前端优化
1. 修改仪表盘数据显示逻辑
2. 添加"暂无数据"状态提示
3. 添加手动刷新功能
4. 测试前端展示效果

### 阶段4: 集成测试
1. 测试完整数据流
2. 验证仪表盘数据展示
3. 检查性能影响

1. 后端指标记录代码
2. 监控服务集成代码
3. 前端优化代码
4. 测试验证报告
