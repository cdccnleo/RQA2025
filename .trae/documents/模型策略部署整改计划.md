# 模型策略部署整改计划

**计划日期**: 2026-02-25  
**目标**: 根据量化交易系统合规检查报告，对模型策略部署模块进行全面整改  
**预期结果**: 合规性评分从32分提升至85分以上

---

## 1. 整改目标

### 1.1 核心目标

- 修复5个高优先级问题
- 修复4个中优先级问题
- 实现量化交易系统合规要求

| 检查维度 | 当前得分 | 目标得分 | 提升措施 |

| 模型管理 | 50 | 90 | 添加模型验证、质量检查 |

## 2. 整改任务清单

### 阶段一：高优先级问题修复（立即实施）

#### 任务 1.1: 添加模型验证逻辑
**目标**: 在部署接口中添加模型存在性、状态、质量验证
**文件**: `src/gateway/web/strategy_lifecycle_routes.py`
**预计工时**: 2小时

**实施步骤**:
1. 创建模型验证函数 `validate_model_for_deployment()`
2. 验证模型存在性（数据库+文件系统）
3. 验证模型训练状态（status == 'trained'）
4. 验证模型质量指标（accuracy >= 0.6）
5. 验证模型版本有效性
6. 在 `deploy_model_strategy_api` 中调用验证

**验收标准**:
- [ ] 无效模型ID返回400错误
- [ ] 未训练模型返回400错误
- [ ] 低质量模型（accuracy < 0.6）返回400错误
- [ ] 有效模型正常部署

---

#### 任务 1.2: 添加参数范围验证
**目标**: 对所有部署参数进行严格的范围验证
**文件**: `src/gateway/web/strategy_lifecycle_routes.py`
**预计工时**: 2小时

**实施步骤**:
1. 创建参数验证函数 `validate_deployment_params()`
2. 验证 prediction_threshold: [0.1, 0.9]
3. 验证 confidence_threshold: [0.5, 0.95]
4. 验证 position_sizing: ['equal', 'confidence_based', 'kelly']
5. 验证 max_position_size: [0.05, 0.30]（降低上限）
6. 使用 Pydantic 模型进行类型和范围验证

**验收标准**:
- [ ] 非法参数返回400错误并提示具体错误
- [ ] 边界值测试通过
- [ ] 参数类型错误返回400错误

---

#### 任务 1.3: 添加强制风险控制参数
**目标**: 强制要求配置止损、止盈、最大回撤等风控参数
**文件**: 
- `src/gateway/web/strategy_lifecycle_routes.py`
- `web-static/strategy-lifecycle.html`
**预计工时**: 3小时

**实施步骤**:
1. 后端修改:
   - 在部署接口中添加必需风控参数
   - stop_loss: [0.01, 0.10]（止损 1%-10%）
   - take_profit: [0.02, 0.20]（止盈 2%-20%）
   - max_drawdown: [0.05, 0.15]（最大回撤 5%-15%）
   - 验证所有风控参数存在且有效

2. 前端修改:
   - 添加风控参数配置表单
   - 添加参数范围提示
   - 添加默认值（stop_loss=0.05, take_profit=0.10, max_drawdown=0.10）

**验收标准**:
- [ ] 缺少风控参数返回400错误
- [ ] 风控参数范围验证通过
- [ ] 前端表单显示风控参数配置
- [ ] 部署后策略包含完整风控配置

---

#### 任务 1.4: 实现模拟交易强制验证
**目标**: 禁止直接部署到实盘，必须经过模拟交易验证
**文件**: 
- `src/gateway/web/strategy_lifecycle_routes.py`
- `src/gateway/web/strategy_lifecycle.py`
**预计工时**: 4小时

**实施步骤**:
1. 修改部署接口逻辑:
   - 部署后默认进入 `paper_trading` 状态
   - 禁止直接转换到 `live_trading`

2. 添加模拟交易验证检查:
   - 检查模拟交易运行时间（至少7天）
   - 检查模拟交易收益（夏普比率 > 1.0）
   - 检查最大回撤（< 10%）

3. 添加手动审批接口:
   - 创建 `/api/v1/strategy/{strategy_id}/lifecycle/approve_live` 接口
   - 需要管理员权限
   - 记录审批日志

4. 修改状态转换规则:
   - 从 `paper_trading` 到 `live_trading` 需要满足验证条件

**验收标准**:
- [ ] 新部署策略默认进入模拟交易
- [ ] 未满足验证条件无法进入实盘
- [ ] 满足条件后可申请进入实盘
- [ ] 审批流程记录完整日志

---

- [ ] 前端滑块最大值为30%

- [ ] 显示风险提示信息
