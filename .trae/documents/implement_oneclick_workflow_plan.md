# 实现真实一键执行流程功能开发计划

## 目标
将策略管理页面的一键执行功能从模拟实现改为真实API调用，实现完整的回测→优化→应用流程。

## 功能需求

### 1. 回测步骤
- 调用回测API启动回测
- 轮询回测进度
- 获取回测结果
- 保存回测结果到策略

### 2. 优化步骤
- 基于回测结果调用优化API
- 轮询优化进度
- 获取优化结果
- 保存优化结果

### 3. 应用步骤
- 调用应用API将优化结果应用到策略
- 更新策略参数
- 增加优化次数统计

### 4. 状态管理
- 实时显示各步骤进度
- 错误处理和重试机制
- 取消功能
- 结果展示

## 技术方案

### API调用流程
```
1. 回测步骤
   POST /api/v1/backtest/run
   → 获取 backtest_id
   → 轮询 GET /api/v1/backtest/{backtest_id}
   → 保存结果

2. 优化步骤
   POST /api/v1/strategy/optimization/start
   → 获取 task_id
   → 轮询 GET /api/v1/strategy/optimization/progress
   → 获取结果 GET /api/v1/strategy/optimization/results

3. 应用步骤
   POST /api/v1/strategy/optimization/{task_id}/apply
   → 应用最优参数到策略
```

### 前端实现
- 使用 async/await 处理异步流程
- 使用 setInterval 轮询进度
- 使用 try-catch 处理错误
- 更新UI显示实时进度
