# 回测数据加载优先级调整计划

## 问题诊断

通过深度代码检查，发现以下关键问题：

### 1. 数据持久化实现
**PostgreSQL数据表**：`akshare_stock_data`
**数据字段**：date, open_price, high_price, low_price, close_price, volume, amount, pct_change, change, turnover_rate, amplitude
**查询函数**：`query_stock_data_from_postgresql()` 在 `postgresql_persistence.py.container` 中

### 2. 当前数据加载逻辑问题
**问题位置**：`backtest_service.py`
**问题描述**：
- 使用模拟数据作为降级方案
- 数据加载优先级不正确
- 没有优先从PostgreSQL加载真实数据

### 3. 量化交易系统要求
**严格要求**：禁止使用模拟数据，必须使用真实历史数据

## 修复方案

### 1. 调整数据加载优先级
新的数据加载优先级：
1. **MiniQMT适配器** - 从MiniQMT数据源加载历史数据
2. **PostgreSQL数据库** - 从PostgreSQL加载已采集的AKShare数据（使用 `query_stock_data_from_postgresql`）
3. **数据管理器** - 从数据管理层加载数据
4. **本地数据文件** - 检查 `/app/data/historical` 目录的CSV文件有效性

### 2. 禁止使用模拟数据
- 移除模拟数据生成逻辑
- 当所有数据源都不可用时，返回错误信息而非模拟数据
- 提示用户先进行数据采集

### 3. 集成PostgreSQL数据查询
- 导入 `query_stock_data_from_postgresql` 函数
- 优先从PostgreSQL查询已采集的股票数据
- 验证数据完整性和有效性

### 4. 数据验证
- 检查数据是否为空
- 验证数据日期范围
- 验证数据字段完整性

## 实施步骤

1. 修改 `backtest_service.py` 导入PostgreSQL查询函数
2. 调整 `_load_historical_data_from_data_manager` 函数的数据加载优先级
3. 集成PostgreSQL数据查询
4. 移除模拟数据降级方案
5. 添加数据验证逻辑
6. 添加错误处理和用户提示
7. 测试验证真实数据加载

## 预期效果

- 回测严格使用真实历史数据
- 优先从已采集的数据源加载
- 当数据不可用时明确提示用户
- 符合量化交易系统数据质量要求
