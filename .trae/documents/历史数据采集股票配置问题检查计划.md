# 历史数据采集股票配置问题检查计划

## 检查目标

全面检查数据采集监控页面中历史数据采集任务采集的股票代码是否应从数据源配置中获取，而非硬编码000001。

## 问题分析

### 1. 当前实现分析

**位置**: `src/distributed/coordinator/data_collector_worker.py` 第 291-307 行

**代码分析**:
```python
# 从source_config中提取股票列表
symbols = []
if source_config:
    # 获取自定义股票列表
    custom_stocks = source_config.get("custom_stocks", [])
    if custom_stocks:
        symbols = [stock.get("code") for stock in custom_stocks if stock.get("code")]
    
    # 如果没有自定义股票，尝试其他配置
    if not symbols:
        # 尝试从strategy_config获取
        strategy_config = source_config.get("strategy_config", {})
        # 这里可以添加从股票池获取的逻辑
        
# 如果没有指定股票，使用默认测试股票
if not symbols:
    symbols = ["000001"]  # <-- 硬编码默认股票：平安银行
```

**问题确认**: ✅ **存在硬编码股票代码**
- 当数据源配置中没有 `custom_stocks` 时，默认使用 `["000001"]`
- 这是测试用的硬编码数据，不是从配置中动态获取

### 2. 数据源配置分析

**检查数据源配置文件**: `data/data_sources_config.json`

**发现**:
- 大部分数据源配置中没有 `custom_stocks` 字段
- 也没有 `symbols` 字段
- 配置中主要包含：
  - `akshare_function`: AKShare函数名
  - `description`: 描述
  - `default_days`: 默认采集天数
  - 其他数据源特定配置

**示例配置** (akshare_stock_a):
```json
{
  "id": "akshare_stock_a",
  "name": "AKShare A股数据",
  "config": {
    "akshare_category": "A股",
    "default_days": 30,
    "adjust_type": "qfq",
    "enable_incremental": true,
    "description": "AKShare A股数据采集接口",
    "akshare_function": "stock_zh_a_spot_em"
  }
}
```

### 3. 问题根因

**根因**: 数据源配置中没有定义要采集的具体股票列表

**当前行为**:
1. 任务提交时传递 `source_config`
2. 工作器尝试从 `source_config.custom_stocks` 获取股票列表
3. 如果没有配置，使用硬编码 `["000001"]`

**期望行为**:
1. 数据源配置中应定义股票列表（如 `custom_stocks` 或 `symbols`）
2. 或者根据数据源类型自动获取股票列表（如获取全部A股）
3. 工作器从配置中动态获取股票列表

### 4. 影响范围

**影响**:
- 所有历史数据采集任务都只采集000001（平安银行）
- 无法根据数据源配置采集不同的股票
- 用户配置的数据源股票列表不生效

## 修复方案

### 方案1: 在数据源配置中添加股票列表（推荐）

**步骤**:
1. 修改数据源配置，添加 `custom_stocks` 或 `symbols` 字段
2. 前端数据源配置界面支持编辑股票列表
3. 任务提交时传递完整的股票列表

**示例配置**:
```json
{
  "id": "akshare_stock_a",
  "name": "AKShare A股数据",
  "config": {
    "akshare_category": "A股",
    "default_days": 30,
    "custom_stocks": [
      {"code": "000001", "name": "平安银行"},
      {"code": "600519", "name": "贵州茅台"},
      {"code": "000858", "name": "五粮液"}
    ]
  }
}
```

### 方案2: 根据数据源类型自动获取股票列表

**步骤**:
1. 修改工作器，根据 `akshare_function` 自动获取股票列表
2. 例如：`stock_zh_a_spot_em` 获取全部A股列表
3. 或者提供股票池配置，按股票池采集

**示例代码**:
```python
# 根据akshare_function获取股票列表
if source_config.get("akshare_function") == "stock_zh_a_spot_em":
    # 获取全部A股列表
    symbols = get_all_a_stocks()
elif source_config.get("akshare_function") == "stock_hk_spot_em":
    # 获取全部港股列表
    symbols = get_all_hk_stocks()
```

### 方案3: 混合方案

**逻辑**:
1. 优先使用配置的 `custom_stocks`
2. 如果没有配置，根据数据源类型自动获取
3. 如果自动获取失败，使用默认股票（并记录警告）

```python
def get_symbols_from_config(source_config):
    # 1. 优先使用配置的custom_stocks
    custom_stocks = source_config.get("custom_stocks", [])
    if custom_stocks:
        return [stock.get("code") for stock in custom_stocks if stock.get("code")]
    
    # 2. 根据数据源类型自动获取
    akshare_function = source_config.get("akshare_function")
    if akshare_function:
        symbols = auto_get_symbols_by_function(akshare_function)
        if symbols:
            return symbols
    
    # 3. 使用默认股票并记录警告
    logger.warning("未配置股票列表，使用默认测试股票")
    return ["000001"]
```

## 建议实现

**推荐方案3（混合方案）**，因为：
1. 保持向后兼容（支持现有配置）
2. 提供灵活性（支持自定义股票列表）
3. 提供智能默认（自动获取股票列表）
4. 有降级方案（默认测试股票）
