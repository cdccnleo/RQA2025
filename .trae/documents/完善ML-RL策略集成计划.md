# 完善ML/RL策略集成计划

## 背景

根据前期数据流全面检查，发现ML/RL策略在`UnifiedStrategyService`中仅为占位实现，需要完善集成以实现完整的机器学习交易流程。

## 现状分析

### 已具备的基础架构

| 组件 | 状态 | 说明 |
|------|------|------|
| ModelManager | ✅ | 模型注册、加载、管理 |
| InferenceManager | ✅ | 推理管理、缓存 |
| ModelPredictor | ✅ | 模型预测服务 |
| ModelDrivenStrategy | ✅ | 模型驱动策略完整实现 |
| RL策略(DQN/PPO/A2C) | ✅ | 强化学习策略完整实现 |

### 集成缺口

| 问题 | 位置 | 影响 |
|------|------|------|
| `_execute_ml_strategy` 占位实现 | strategy_service.py#L662 | ML策略无法执行 |
| `_execute_rl_strategy` 占位实现 | strategy_service.py#L674 | RL策略无法执行 |
| 缺乏策略工厂 | 无 | 策略创建管理混乱 |

## 目标

实现ML/RL策略与统一策略服务的完整集成，支持：
1. ML策略：基于模型预测生成交易信号
2. RL策略：基于强化学习智能体生成交易信号
3. 特征工程到模型推理的完整数据流
4. 策略实例的统一管理和缓存

## 接口定义

### ML策略配置
```python
@dataclass
class MLStrategyConfig:
    strategy_id: str
    model_id: str                    # 模型ID
    prediction_threshold: float = 0.5  # 预测阈值
    confidence_threshold: float = 0.7  # 置信度阈值
    position_sizing: str = "equal"     # 仓位管理
    max_position_size: float = 0.2     # 最大仓位
    feature_config: Dict = None        # 特征配置
```

### RL策略配置
```python
@dataclass
class RLStrategyConfig:
    strategy_id: str
    agent_type: str = "dqn"           # dqn/ppo/a2c
    model_path: str = None            # 预训练模型路径
    state_size: int = 8               # 状态空间维度
    action_size: int = 3              # 动作空间维度
    position_size: int = 100          # 交易数量
```

## 验收标准

### 功能验收
- [ ] ML策略能正确加载模型并生成交易信号
- [ ] RL策略能正确加载智能体并生成交易信号
- [ ] 特征工程输出正确传递到模型推理
- [ ] 策略工厂能正确创建和管理策略实例
- [ ] 推理缓存能有效减少重复计算

### 性能验收
- [ ] ML策略推理延迟 < 100ms
- [ ] RL策略推理延迟 < 50ms
- [ ] 批量数据加载速度提升 50%+
- [ ] 内存占用合理，无内存泄漏

### 可靠性验收
- [ ] 模型加载失败有降级处理
- [ ] 推理异常有错误处理和日志
- [ ] 策略实例能正确释放资源
- [ ] 支持模型热切换

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 模型推理延迟高 | 交易延迟 | 实现推理缓存、GPU加速 |
| 特征工程计算量大 | CPU占用高 | 并行计算、特征缓存 |
| 模型版本不兼容 | 预测失败 | 版本管理、向后兼容 |
| 内存泄漏 | 系统不稳定 | 资源监控、定期清理 |
