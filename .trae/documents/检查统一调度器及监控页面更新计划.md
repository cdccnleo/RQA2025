# 检查统一调度器及监控页面更新计划

## 目标
检查统一调度器启动、工作节点创建逻辑，并验证以下监控页面是否按照统一调度器更新：
1. 数据源配置管理页面 (data-sources-config) - 数据采集调度器仪表盘
2. 历史数据采集监控页面 (data-collection-monitor) - 历史数据采集监控仪表盘
3. 特征工程监控页面 (feature-engineering-monitor) - 特征任务调度器状态面板
4. 模型训练监控页面 (model-training-monitor) - 模型训练调度器状态仪表盘

## 检查内容

### 1. 统一调度器启动逻辑检查

#### 1.1 检查统一调度器启动流程
**文件**: `src/distributed/coordinator/unified_scheduler.py`
- 检查 `start()` 方法是否正确启动调度器
- 检查 `_scheduler_loop()` 是否正常运行
- 检查 `_auto_route_tasks()` 是否正确路由任务

#### 1.2 检查工作节点注册
**文件**: `src/distributed/registry/unified_worker_registry.py`
- 检查 `register_worker()` 是否正确注册工作节点
- 检查 `get_available_workers()` 是否正确返回可用工作节点
- 检查 `get_workers_by_type()` 是否正确按类型返回工作节点
