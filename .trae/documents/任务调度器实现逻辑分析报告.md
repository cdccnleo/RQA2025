## 调度器工作节点创建逻辑分析报告

### 1. 特征任务调度器 (`FeatureTaskScheduler`)

**工作节点创建逻辑**:
- **方法**: `start_with_workers(worker_count=None)`
- **位置**: `src/features/distributed/task_scheduler.py:427`
- **默认工作节点数**: `max(2, os.cpu_count() or 2)`
- **创建方式**: 自动创建线程池工作节点

**代码逻辑**:
```python
def start_with_workers(self, worker_count: int = None) -> None:
    # 启动调度器
    if not self._running:
        self.start()
    
    # 计算工作节点数量
    if worker_count is None:
        worker_count = max(2, os.cpu_count() or 2)
    
    # 创建工作节点
    for i in range(worker_count):
        worker_id = f"worker_{i}_{int(time.time())}"
        capabilities = {
            "max_concurrent_tasks": 1,
            "supported_task_types": ["技术指标", "统计特征", "情感特征", "自定义特征"],
            "cpu_cores": 1,
            "max_memory": 512  # MB
        }
        # 启动工作节点线程
        worker_thread = threading.Thread(
            target=start_worker,
            args=(worker_id, self),
            daemon=True
        )
        worker_thread.start()
        # 注册工作节点
        worker_manager.register_worker(worker_id, capabilities)
```

**启动入口**:
- **函数**: `start_scheduler()` 
- **位置**: `src/gateway/web/feature_engineering_service.py:720`
- **调用时机**: 系统初始化时自动调用

### 2. 数据采集调度器 (`DataCollectionServiceScheduler`)

**工作节点创建逻辑**:
- **类型**: 异步协程调度，不使用固定工作节点池
- **并发控制**: `max_concurrent_tasks = 1`
- **任务执行**: 通过 `DataCollectionWorkflow` 编排器动态执行

**特点**:
- 不预创建工作节点
- 每次采集任务独立启动协程
- 通过 `active_tasks` 集合管理活跃任务

### 3. 模型训练调度器 (`TrainingJobExecutor`)

**工作节点创建逻辑**:
- **类型**: 专用执行器，单实例运行
- **工作节点ID**: `training_executor_{id(self)}`
- **任务获取**: 从持久化存储获取待处理训练任务

**特点**:
- 不向特征任务调度器注册
- 独立管理训练任务生命周期
- 通过 `_get_pending_training_task()` 获取任务

### 4. 工作节点状态管理

**统一注册表** (`UnifiedWorkerRegistry`):
- **位置**: `src/distributed/registry/unified_worker_registry.py`
- **功能**: 管理所有类型工作节点状态
- **类型区分**: 
  - `FEATURE_WORKER` - 特征工作节点
  - `TRAINING_EXECUTOR` - 训练执行器

**特征工作节点管理器** (`FeatureWorkerManager`):
- **位置**: `src/features/distributed/worker_manager.py`
- **功能**: 管理特征工作节点生命周期
- **心跳监控**: 自动检测离线节点

### 5. 任务调度流程

**特征任务调度流程**:
1. 任务提交 → `scheduler.submit_task()`
2. 任务入队 → `PriorityQueue`
3. 工作节点获取任务 → `scheduler.get_task(worker_id)`
4. 任务执行 → `execute_feature_task(task)`
5. 结果返回 → `scheduler.complete_task()`

**数据采集调度流程**:
1. 定时触发 → `service_scheduler` 检查
2. 任务启动 → `DataCollectionWorkflow.execute()`
3. 数据获取 → 通过数据层适配器
4. 数据存储 → PostgreSQL + 数据湖

**模型训练调度流程**:
1. 任务提交 → 持久化存储
2. 执行器轮询 → `_get_pending_training_task()`
3. 任务执行 → `model_trainer.train_model()`
4. 状态更新 → 持久化存储

### 6. 确保调度器正常调度的关键机制

✅ **特征任务调度器**:
- `start_with_workers()` 自动创建工作节点
- `get_available_workers()` 只返回特征工作节点
- 训练执行器不再注册到特征调度器

✅ **数据采集调度器**:
- 独立协程执行，不依赖工作节点池
- 自动管理并发任务数

✅ **模型训练调度器**:
- 独立执行循环
- 从持久化存储获取任务
