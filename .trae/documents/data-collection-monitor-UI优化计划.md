# 数据采集监控仪表板 UI 优化计划

## 目标
重新组织和优化 data-collection-monitor.html 页面的 UI 显示，使其更加清晰、专业、易于使用。

## 当前页面结构分析

### 现有功能模块
1. **整体健康状态** - 健康度、活跃数据源、采集成功率
2. **详细指标** - 动态生成的指标卡片
3. **缓存统计** - 缓存命中率、大小等
4. **告警信息** - 活跃告警、已解决告警
5. **历史数据采集监控** - 调度器状态、工作进程、队列任务、成功率
6. **任务控制面板** - 启动、停止、暂停、恢复、清理、导出、立即采集等
7. **历史数据采集配置** - 采集期类型、保存配置
8. **活跃任务列表** - 正在执行的任务
9. **任务队列状态** - 队列处理状态
10. **工作进程状态** - 工作进程详情
11. **性能指标** - 历史采集性能
12. **采集延迟监控** - 平均延迟、最大延迟、延迟分布、延迟趋势、告警阈值
13. **数据质量监控** - 完整性、准确性、时效性、质量评分
14. **异常自动恢复** - 重试次数、恢复次数、成功率、切换次数

### 当前问题
1. **信息层级不清晰** - 所有功能平铺展示，缺乏主次之分
2. **视觉混乱** - 不同功能的样式不一致
3. **操作区域分散** - 控制按钮分布在不同区域
4. **缺乏数据可视化** - 大量数字展示，缺少图表
5. **响应式不足** - 在移动端显示可能不佳

## 优化方案

### 1. 页面布局重新组织

采用 **Tab 导航 + 卡片式布局**，将功能分组：

```
┌─────────────────────────────────────────────────────────────┐
│  📊 数据采集监控仪表板                    [刷新] [配置]      │
├─────────────────────────────────────────────────────────────┤
│  [概览] [实时监控] [任务管理] [性能分析] [数据质量] [告警]   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Tab 内容区域                                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2. 功能分组

#### Tab 1: 概览 (Overview)
- 关键指标卡片（健康度、成功率、活跃数据源）
- 状态总览图表
- 最近活动摘要
- 快捷操作按钮

#### Tab 2: 实时监控 (Real-time)
- 调度器状态实时监控
- 工作进程状态
- 任务队列状态
- 活跃任务列表
- 实时数据流展示

#### Tab 3: 任务管理 (Tasks)
- 任务控制面板（集中所有控制按钮）
- 历史采集配置
- 任务历史记录
- 批量操作功能

#### Tab 4: 性能分析 (Performance)
- 性能指标图表
- 采集延迟监控
- 延迟趋势图
- 告警阈值设置

#### Tab 5: 数据质量 (Quality)
- 数据质量评分
- 完整性、准确性、时效性指标
- 质量详情分析
- 异常数据展示

#### Tab 6: 告警 (Alerts)
- 活跃告警列表
- 已解决告警
- 告警统计
- 告警规则配置

### 3. UI 优化细节

#### 3.1 视觉层次
- **一级标题**: 页面标题、Tab 标题
- **二级标题**: 卡片标题
- **三级标题**: 指标标签
- **数据展示**: 大号数字 + 小号说明

#### 3.2 颜色规范
- **成功/正常**: 绿色 (#28a745)
- **警告**: 黄色 (#ffc107)
- **错误/危险**: 红色 (#dc3545)
- **信息**: 蓝色 (#007acc)
- **中性**: 灰色 (#6c757d)

#### 3.3 卡片设计
- 统一圆角 (8px)
- 统一阴影 (0 2px 8px rgba(0,0,0,0.1))
- 左侧彩色边框标识状态
- 悬停效果

#### 3.4 图表集成
- 使用 Chart.js 绘制趋势图
- 仪表盘图展示健康度
- 柱状图展示性能指标
- 折线图展示延迟趋势

### 4. 响应式设计

- 桌面端: 多列布局
- 平板: 双列布局
- 手机: 单列布局
- Tab 导航在小屏幕下可横向滚动
