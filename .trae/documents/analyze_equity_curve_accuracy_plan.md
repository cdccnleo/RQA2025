# 累计收益率走势准确性深度分析计划

## 任务目标
深度分析策略回测分析页面（strategy-backtest.html）累计收益率走势的准确性，确保图表能够根据回测交易记录准确反应收益率走势。

## 当前状态
- 后端修复已生效：新的回测记录 `end_date` 已正确设置为实际最后交易日期
- 前端标签生成算法已修复：使用实际交易日序列生成标签
- 需要验证：累计收益率走势是否准确反映交易记录的实际盈亏

## 分析方向

### 1. 资金曲线数据来源分析
- 检查 `equity_curve` 数据的生成逻辑
- 验证资金曲线是否包含所有交易的实际盈亏
- 检查买入、卖出、强制平仓对资金曲线的影响

### 2. 收益率计算准确性
- 验证累计收益率的计算公式
- 检查初始资金和最终资金的对应关系
- 验证日收益率与累计收益率的一致性

### 3. 交易记录与资金曲线对应关系
- 分析每笔交易对资金曲线的具体影响
- 检查买入时的资金扣减是否正确
- 检查卖出时的盈亏计算是否正确
- 验证强制平仓的资金处理

### 4. 图表展示准确性
- 验证图表数据点与实际交易日的对应关系
- 检查图表趋势是否与实际交易盈亏一致
- 验证图表起始和结束点是否正确

## 检查步骤

### 第一阶段：资金曲线数据生成分析（15分钟）
1. 检查回测引擎的资金曲线生成逻辑
2. 分析每笔交易如何影响资金曲线
3. 验证买入、卖出、强制平仓的资金处理
4. 检查手续费和滑点的处理

### 第二阶段：收益率计算验证（15分钟）
1. 验证累计收益率计算公式
2. 检查日收益率与累计收益率的关系
3. 验证初始资金和最终资金的准确性
4. 对比交易盈亏总和与资金曲线盈亏

### 第三阶段：交易记录与图表对应分析（20分钟）
1. 逐笔分析交易记录对资金曲线的影响
2. 验证图表趋势与交易盈亏的一致性
3. 检查图表数据点与实际交易日的对应关系
4. 分析强制平仓对图表的影响

### 第四阶段：修复和优化（20分钟）
1. 修复发现的数据不一致问题
2. 优化图表展示逻辑
3. 添加数据验证和调试日志
4. 测试验证修复效果

## 关键验证点

### 必须验证的内容
- [ ] 资金曲线数据是否包含所有交易的实际盈亏
- [ ] 累计收益率计算是否准确
- [ ] 图表趋势是否与交易记录一致
- [ ] 买入、卖出、强制平仓的资金处理是否正确
- [ ] 图表起始和结束点是否正确

### 可能的问题点
1. 资金曲线可能未正确反映买入时的资金扣减
2. 卖出盈亏可能未正确计算到资金曲线
3. 强制平仓的资金处理可能有问题
4. 图表数据点可能与实际交易日不对应

## 预期结果
累计收益率走势应该：
- 准确反映每笔交易的实际盈亏
- 买入时资金曲线下降（扣除成本）
- 卖出时资金曲线上升（增加盈亏）
- 强制平仓时资金曲线正确反映最终盈亏
- 图表趋势与交易记录明细完全一致

## 分析方法

### 数据对比分析
```
交易记录：
- 2026-02-03: 买入 002837 (资金减少)
- 2026-02-04: 卖出 002837 (盈亏增加)
- ...

资金曲线：
- 点1: 100000 (初始)
- 点2: 99800 (买入后)
- 点3: 100500 (卖出后)
- ...

对比验证：
- 点2 - 点1 = 买入成本
- 点3 - 点2 = 卖出盈亏
```

### 图表趋势验证
- 买入后资金曲线应下降
- 盈利卖出后资金曲线应上升
- 亏损卖出后资金曲线应下降
- 整体趋势应与累计盈亏一致

## 风险与缓解
| 风险 | 缓解措施 |
|------|----------|
| 数据生成逻辑复杂 | 添加详细日志追踪每笔交易对资金曲线的影响 |
| 计算精度问题 | 使用高精度计算，避免浮点数误差 |
| 图表渲染问题 | 添加数据验证，确保图表数据与实际一致 |
| 时序问题 | 确保交易记录和资金曲线的时间顺序一致 |
