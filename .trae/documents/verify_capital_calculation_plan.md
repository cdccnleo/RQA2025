# 策略回测账号可用资金计算验证计划

## 任务目标
检查策略回测逻辑，根据初始资金及每笔回测交易记录，验证账号可用资金是否计算正确。

## 当前状态
- 已修复前端资金计算逻辑，正确处理 `cost` 字段
- 需要验证：修复后的资金计算是否完全正确
- 需要确认：最终资金是否与回测结果一致

## 验证方向

### 1. 资金计算逻辑验证
- 验证买入交易的资金扣除是否正确
- 验证卖出交易的资金增加是否正确
- 验证强制平仓的资金处理是否正确
- 检查手续费的处理方式

### 2. 数据一致性验证
- 验证初始资金是否正确获取
- 验证每笔交易后的资金变化是否正确
- 验证最终资金是否与回测结果一致
- 检查资金曲线数据是否匹配

### 3. 边界情况检查
- 检查第一笔交易前的初始资金
- 检查最后一笔交易后的最终资金
- 验证多笔同日期交易的资金计算
- 检查强制平仓的资金处理

### 4. 前端展示验证
- 验证交易记录表格中的账户可用资金显示
- 检查资金变化趋势是否合理
- 验证累计盈亏计算是否正确

## 检查步骤

### 第一阶段：后端数据验证（10分钟）
1. 获取最新回测记录的完整数据
2. 检查初始资金和最终资金
3. 检查每笔交易的价格、数量、cost字段
4. 验证资金曲线数据

### 第二阶段：前端计算验证（15分钟）
1. 在浏览器控制台查看资金计算日志
2. 逐笔验证交易的资金变化
3. 检查初始资金和最终资金是否匹配
4. 验证资金计算逻辑的正确性

### 第三阶段：问题修复（15分钟）
1. 修复发现的资金计算问题
2. 优化资金计算逻辑
3. 添加更详细的调试日志
4. 增强数据验证

### 第四阶段：最终验证（10分钟）
1. 重新部署并测试
2. 验证所有交易记录的资金计算
3. 确认最终资金与回测结果一致
4. 检查资金变化趋势是否合理

## 验证公式

### 正确的资金计算
```
初始资金 = results[0].initial_capital

买入交易：
  如果 cost ≈ 价格 × 数量（误差10%内）：
    账户可用资金 -= cost  // cost是买入总金额
  否则：
    账户可用资金 -= 价格 × 数量  // cost是手续费

卖出交易：
  手续费 = (cost < 交易金额×0.5) ? cost : 0
  账户可用资金 += (价格 × 数量 - 手续费)

最终资金应该 = results[0].final_capital
```

### 验证要点
- 第一笔交易前：账户可用资金 = 初始资金
- 每笔买入后：账户可用资金减少
- 每笔卖出后：账户可用资金增加
- 最后一笔交易后：账户可用资金 = 最终资金

## 预期结果
- 初始资金正确显示为 ¥100,000.00
- 每笔交易后的账户可用资金计算准确
- 最终资金与回测结果一致（¥111,033.35）
- 资金变化趋势与交易盈亏一致

## 风险与缓解
| 风险 | 缓解措施 |
|------|----------|
| 后端数据格式不一致 | 添加智能识别逻辑，适应不同数据格式 |
| 计算精度问题 | 使用高精度计算，避免浮点数误差 |
| 交易记录排序问题 | 确保按时间顺序计算资金 |
| 多回测结果合并 | 分别计算每个回测的资金，或显示警告 |
