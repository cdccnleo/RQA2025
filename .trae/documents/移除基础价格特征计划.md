# 移除基础价格特征计划

## 目标
修改特征提取任务，只保留技术指标作为特征，移除基础价格特征（open, high, low, close, volume）。

## 背景
在量化交易系统中，基础价格数据（open, high, low, close, volume）是原始输入数据，不是衍生特征。技术指标（如 SMA, EMA, RSI, MACD, KDJ, BOLL）已经包含了价格信息。保留基础价格特征会增加特征维度，可能导致过拟合。

## 当前状态
- 特征提取任务成功生成了 18 个特征
- 其中 5 个是基础价格特征（open, high, low, close, volume）
- 12 个是技术指标特征
- 1 个是日期特征（date）

## 实施步骤

### 步骤 1: 修改特征保存逻辑
**文件**: `src/features/distributed/worker_executor.py`
**位置**: `execute_feature_task` 函数中保存特征到数据库的代码
**修改内容**:
- 在保存特征到 `feature_store` 表之前，过滤掉基础价格特征
- 只保留技术指标特征和日期特征

### 步骤 2: 更新特征计数
**文件**: `src/features/distributed/worker_executor.py`
**位置**: 更新任务进度和结果的代码
**修改内容**:
- 确保 `feature_count` 只计算技术指标特征
- 更新 `features_list` 只包含技术指标特征

### 步骤 3: 验证修改
**操作**:
- 重新构建 Docker 容器
- 创建新的特征提取任务
- 验证只生成技术指标特征
- 确认基础价格特征已被移除

## 预期结果
- 特征提取任务只生成技术指标特征（12 个）
- 不再生成基础价格特征（open, high, low, close, volume）
- 可以保留日期特征（date）用于时间序列分析
- 合规性评分提升

## 技术细节

### 基础价格特征列表
```python
basic_price_features = ['open', 'high', 'low', 'close', 'volume']
```

### 过滤逻辑
```python
# 在保存特征之前，过滤掉基础价格特征
basic_price_features = ['open', 'high', 'low', 'close', 'volume']
features_to_save = [f for f in all_features if f not in basic_price_features]
```

## 风险评估
- **低风险**: 只修改特征保存逻辑，不影响技术指标计算
- **向后兼容**: 基础价格数据仍可从原始数据表获取
- **性能提升**: 减少特征维度，降低过拟合风险
