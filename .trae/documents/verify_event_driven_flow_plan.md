# 事件驱动流程验证计划

## 目标
验证手动触发数据采集后，各业务模块是否正确响应 DATA_COLLECTION_COMPLETED 事件。

## 背景
- 已修复 `data_collectors.py`，现在手动采集时会发布 `DATA_COLLECTION_COMPLETED` 事件
- 已验证事件发布成功
- 需要检查各业务模块是否正确订阅并响应事件

## 检查范围

### 1. 特征工程模块响应检查
**检查内容**:
- 是否正确订阅 `DATA_COLLECTION_COMPLETED` 事件
- 事件处理器 `on_data_collection_completed` 是否被调用
- 是否正确解析事件数据（source_id, source_config）
- 是否创建特征提取任务
- 任务是否正确提交到统一调度器
- 是否有错误日志

**相关文件**:
- `src/features/core/event_listeners.py`
- `src/features/core/feature_task_service.py`

**验证方法**:
1. 检查特征工程事件监听器初始化日志
2. 检查事件处理日志
3. 检查特征任务是否创建（查询 `feature_engineering_tasks` 表）
4. 检查统一调度器任务队列

### 2. 数据质量监控响应检查
**检查内容**:
- 是否正确订阅 `DATA_COLLECTION_COMPLETED` 事件
- 事件处理器是否被调用
- 是否执行数据质量检查
- 是否生成质量报告
- 是否有错误日志

**相关文件**:
- `src/data/quality/data_collection_event_handler.py`
- `src/data/quality/data_quality_monitor.py`

**验证方法**:
1. 检查数据质量事件处理器初始化日志
2. 检查质量检查执行日志
3. 检查质量报告生成情况

### 3. ML模块响应检查
**检查内容**:
- 是否正确订阅 `DATA_COLLECTION_COMPLETED` 事件
- 事件处理器是否被调用
- 是否更新特征工程
- 是否触发模型训练
- 模型训练条件判断是否正确（数据量>100）
- 是否有错误日志

**相关文件**:
- `src/ml/core/data_collection_event_handler.py`

**验证方法**:
1. 检查ML事件处理器初始化日志
2. 检查特征工程更新日志
3. 检查模型训练触发情况

### 4. 策略回测响应检查
**检查内容**:
- 是否正确订阅 `DATA_COLLECTION_COMPLETED` 事件
- 事件处理器是否被调用
- 是否触发策略回测
- 回测条件判断是否正确
- 是否有错误日志

**相关文件**:
- 策略回测相关的事件处理器

**验证方法**:
1. 检查策略回测事件处理器初始化日志
2. 检查回测触发情况

## 实施步骤

### 步骤 1: 检查事件监听器初始化
1. 查看应用启动日志，确认各模块事件监听器已初始化
2. 检查是否有订阅 `DATA_COLLECTION_COMPLETED` 事件的日志

### 步骤 2: 手动触发数据采集
1. 调用 API 触发数据采集
2. 确认 `DATA_COLLECTION_COMPLETED` 事件已发布

### 步骤 3: 检查特征工程响应
1. 查看特征工程事件处理日志
2. 检查特征任务是否创建
3. 查询数据库 `feature_engineering_tasks` 表

### 步骤 4: 检查数据质量监控响应
1. 查看数据质量检查日志
2. 检查质量报告生成情况

### 步骤 5: 检查ML模块响应
1. 查看ML模块日志
2. 检查模型训练触发情况

### 步骤 6: 检查策略回测响应
1. 查看策略回测相关日志
2. 检查回测触发情况

### 步骤 7: 问题诊断与修复
1. 如果发现模块未响应，检查事件订阅代码
2. 检查事件处理器注册逻辑
3. 修复发现的问题

## 预期结果
- 特征工程模块：收到事件后创建特征提取任务
- 数据质量监控：收到事件后执行质量检查
- ML模块：收到事件后更新特征工程并触发模型训练
- 策略回测：收到事件后触发回测（如满足条件）

## 时间估计
- 步骤 1-2: 10分钟
- 步骤 3-6: 40分钟
- 步骤 7: 30分钟
- 总计: 约1.5小时
