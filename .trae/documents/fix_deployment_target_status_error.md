# 修复策略部署失败：缺少目标状态错误

## 问题描述
策略部署失败，错误信息：`❌ 部署失败: 缺少目标状态`

## 诊断计划

### 1. 检查容器App运行日志
- [ ] 查看后端错误日志
- [ ] 定位错误发生位置
- [ ] 分析错误堆栈

### 2. 检查后端代码
- [ ] 检查 `deploy_model_strategy_api` 函数
- [ ] 检查生命周期状态转换逻辑
- [ ] 检查目标状态参数传递

### 3. 检查数据库状态
- [ ] 检查策略构思是否正确保存
- [ ] 检查生命周期记录是否正确创建
- [ ] 检查当前状态字段

### 4. 检查前端请求
- [ ] 检查前端发送的请求参数
- [ ] 确认是否缺少目标状态参数

## 可能原因

### 原因1：生命周期状态转换缺少目标状态
```python
# 可能的问题：transition_status 调用时缺少目标状态
transition_strategy_status(strategy_id, None)  # 错误
```

### 原因2：生命周期创建后未设置初始状态
```python
# 可能的问题：创建生命周期后未正确初始化状态
lifecycle.current_status = None  # 错误
```

### 原因3：前端请求参数不完整
```javascript
// 可能的问题：前端未发送目标状态
body: JSON.stringify({
    model_id: modelId,
    name: strategyName,
    // 缺少 target_status
})
```

## 修复方案

### 方案1：修复后端状态转换逻辑
- 确保生命周期创建时设置正确的初始状态
- 确保状态转换时提供目标状态

### 方案2：修复前端请求参数
- 确保前端发送完整的目标状态参数
- 添加默认值处理

### 方案3：添加错误处理和日志
- 添加详细的错误日志
- 添加参数验证

## 验证步骤

1. 查看容器日志定位错误
2. 修复代码问题
3. 重新构建部署
4. 测试策略部署
5. 验证数据持久化
