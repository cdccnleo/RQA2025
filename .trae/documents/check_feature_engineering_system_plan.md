# 特征工程系统全面检查计划

## 目标
全面检查特征工程模块是否正确响应数据采集完成事件并创建特征提取任务，以及特征任务调度器启动、特征提取任务处理和特征工程监控页面各仪表盘显示。

## 背景
- 数据采集已完成，需要验证特征工程系统的完整流程
- 基于业务流程驱动架构，特征工程应自动响应数据采集完成事件
- 需要确保从事件响应到监控展示的完整链路正常

## 检查范围

### 1. 事件响应机制检查
**检查内容**:
- 特征工程模块是否正确订阅 DATA_COLLECTION_COMPLETED 事件
- 事件处理器是否被正确初始化
- 事件数据格式是否正确解析（source_id, source_config）
- 事件处理日志是否正常输出

**相关文件**:
- `src/features/core/event_listeners.py`
- `src/gateway/web/api.py` (事件处理器初始化)

### 2. 特征提取任务创建检查
**检查内容**:
- 收到事件后是否调用 `_create_feature_task` 方法
- 任务配置是否正确构建（data_source, source_config, collection_time）
- 任务类型是否正确确定（技术指标/情感特征/统计特征）
- 任务是否正确提交到调度器或服务层
- 任务创建日志是否正常输出

**相关文件**:
- `src/features/core/event_listeners.py`
- `src/features/core/feature_task_service.py` (如存在)

### 3. 特征任务调度器检查
**检查内容**:
- 特征任务调度器是否正确启动
- 调度器是否能正确接收特征提取任务
- 调度器的任务队列状态
- 调度器的执行线程/进程状态
- 调度器日志是否正常

**相关文件**:
- `src/features/core/scheduler/` 或类似路径
- `src/core/orchestration/scheduler/` (统一调度器)

### 4. 特征提取任务处理检查
**检查内容**:
- 特征提取任务是否被正确执行
- 特征计算逻辑是否正常（技术指标计算）
- 特征数据是否正确保存到数据库
- 任务执行日志是否正常输出
- 任务状态是否正确更新（pending -> running -> completed/failed）

**相关文件**:
- `src/features/core/calculators/` (特征计算器)
- `src/features/core/extractors/` (特征提取器)
- `src/features/core/persistence/` (特征持久化)

### 5. 特征工程监控页面检查
**检查内容**:
- 监控页面 `/feature-engineering-monitor` 是否正常加载
- 各仪表盘是否正确显示数据：
  - 任务统计仪表盘（总任务数、运行中、已完成、失败）
  - 特征计算进度仪表盘
  - 数据源特征覆盖仪表盘
  - 系统性能仪表盘（CPU、内存、执行时间）
  - 最近任务列表
- WebSocket 实时更新是否正常
- 前端与后端数据交互是否正常

**相关文件**:
- `web-static/feature-engineering-monitor.html` (监控页面)
- `src/gateway/web/api.py` (监控API端点)
- `src/gateway/web/websocket_routes.py` (WebSocket端点)

## 实施步骤

### 步骤 1: 检查事件响应机制
1. 检查特征工程事件监听器是否正确订阅事件
2. 检查应用启动时事件监听器是否正确初始化
3. 查看日志确认事件是否被正确接收和处理
4. 验证事件数据解析是否正确

### 步骤 2: 检查特征提取任务创建
1. 检查 `_create_feature_task` 方法实现
2. 验证任务配置构建逻辑
3. 检查任务提交到调度器的逻辑
4. 查看日志确认任务是否被创建

### 步骤 3: 检查特征任务调度器
1. 检查调度器是否正确启动
2. 检查调度器状态和任务队列
3. 查看调度器日志
4. 验证调度器API端点

### 步骤 4: 检查特征提取任务处理
1. 检查特征计算器是否正确执行
2. 验证特征数据是否正确保存
3. 查看任务执行日志
4. 检查任务状态更新

### 步骤 5: 检查监控页面
1. 访问监控页面验证是否正常加载
2. 检查各仪表盘数据是否正确显示
3. 验证WebSocket实时更新
4. 检查API端点返回数据

### 步骤 6: 端到端测试
1. 触发数据采集完成事件（手动或等待自动采集）
2. 观察完整流程：事件 -> 任务创建 -> 调度 -> 执行 -> 监控
3. 验证整个链路是否正常

## 预期结果
- 特征工程模块正确响应数据采集完成事件
- 特征提取任务自动创建并提交到调度器
- 特征任务调度器正常运行并执行任务
- 特征数据正确计算并保存
- 监控页面各仪表盘正确显示实时数据

## 时间估计
- 步骤 1-2: 30分钟
- 步骤 3-4: 40分钟
- 步骤 5: 30分钟
- 步骤 6: 20分钟
- 总计: 约2小时
