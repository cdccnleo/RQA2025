# 修复回测引擎未返回有效结果的问题

## 问题分析

通过分析代码，我发现了以下问题：

1. **调用方式不匹配**：
   - `backtest_service.py` 中使用 `await engine.run_backtest()` 异步调用
   - 但 `backtest_engine.py` 中的 `run_backtest` 方法是同步方法

2. **参数传递不匹配**：
   - 服务层传递了多个参数：`strategy_id, start_date, end_date, initial_capital, commission_rate`
   - 引擎中的方法只接受两个参数：`strategy=None, data=None`

3. **返回格式不匹配**：
   - 引擎返回 `BacktestResult` 对象
   - 服务层期望返回字典格式

4. **方法重载问题**：
   - 引擎中有多个 `run_backtest` 方法定义，可能导致调用错误的方法

## 解决方案

修改 `backtest_service.py` 中的 `_execute_backtest` 函数，调整调用方式和参数传递：

1. **移除异步调用**：
   - 移除 `await` 关键字，直接调用同步方法

2. **调整参数传递**：
   - 根据引擎方法签名，只传递必要的参数
   - 或创建一个适配层，将服务层参数转换为引擎期望的格式

3. **处理返回格式**：
   - 正确处理 `BacktestResult` 对象，将其转换为字典格式
   - 确保返回的字典包含服务层需要的所有字段

4. **添加错误处理**：
   - 增强错误处理，确保即使引擎调用失败也能返回有意义的结果

## 具体修改步骤

1. 打开 `c:\PythonProject\RQA2025\src\gateway\web\backtest_service.py` 文件
2. 修改 `_execute_backtest` 函数：
   - 移除 `await` 关键字
   - 调整 `run_backtest` 方法调用，适配引擎的参数要求
   - 添加 `BacktestResult` 对象到字典的转换逻辑
   - 增强错误处理
3. 保存修改并测试回测功能

## 预期结果

修改后，回测引擎应该能够正确处理服务层的调用，返回有效的回测结果，而不是空数据。
