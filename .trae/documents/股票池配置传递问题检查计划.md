# 股票池配置传递问题检查计划

## 检查目标

分析检查任务 `b601eb7d-1317-4a4e-baef-8b37f4e6ea9c` 为何没有从数据源配置 `akshare_stock_a` 中获取自定义股票池（37只股票），而仍然采集了 `000001`。

## 问题分析

### 1. 配置结构分析

**API 返回的数据源结构**:
```json
{
  "id": "akshare_stock_a",
  "name": "AKShare A股数据",
  "type": "股票数据",
  "config": {
    "stock_pool_type": "custom",
    "custom_stocks": [
      {"code": "002837", "name": "英维克"},
      {"code": "688702", "name": "盛科通信"},
      ...
    ]
  }
}
```

**任务创建时传递的数据**:
```python
task_data = {
    "source_id": source["id"],
    "source_config": source,  # 传递了整个数据源对象
    ...
}
```

### 2. 问题根因

**数据结构不匹配**:

1. **任务创建** (`datasource_routes.py` 第284行):
   - `source_config = source` (整个数据源对象)
   - `source` 包含: `id`, `name`, `type`, `config`, ...
   - `custom_stocks` 在 `source["config"]` 中

2. **工作器处理** (`data_collector_worker.py` 第417-420行):
   - `source_config = task_data.get("source_config", {})`
   - 调用 `self._get_symbols_from_config(source_config)`

3. **获取股票列表** (`data_collector_worker.py` 第109-157行):
   - `source_config.get("custom_stocks", [])` 
   - 但 `custom_stocks` 在 `source_config["config"]` 中，不在 `source_config` 根级别
   - 所以获取不到，返回空列表
   - 最终使用默认股票 `000001`

### 3. 数据流分析

```
数据源配置 (API)
    ↓
{
  "id": "akshare_stock_a",
  "config": {
    "custom_stocks": [...]  ← 股票池在这里
  }
}
    ↓
创建任务 (datasource_routes.py)
    ↓
task_data = {
  "source_config": source  ← 传递整个对象
}
    ↓
工作器获取 (data_collector_worker.py)
    ↓
source_config.get("custom_stocks")  ← 返回空（因为custom_stocks在config中）
    ↓
使用默认股票 000001
```

## 修复方案

### 方案1: 修改任务创建时传递的数据（推荐）

**修改** `datasource_routes.py` 第284行:
```python
# 修复前
task_data = {
    "source_id": source["id"],
    "source_config": source,  # 传递整个对象
}

# 修复后
task_data = {
    "source_id": source["id"],
    "source_config": source.get("config", {}),  # 只传递config部分
}
```

### 方案2: 修改工作器处理逻辑

**修改** `data_collector_worker.py` 第417-420行:
```python
# 修复前
source_config = task_data.get("source_config", {})
symbols = self._get_symbols_from_config(source_config)

# 修复后
source_config = task_data.get("source_config", {})
# 如果source_config包含config字段，使用config部分
if "config" in source_config:
    source_config = source_config["config"]
symbols = self._get_symbols_from_config(source_config)
```

### 方案3: 修改 _get_symbols_from_config 方法

**修改** `data_collector_worker.py` 第109-157行:
```python
def _get_symbols_from_config(self, source_config: Dict[str, Any]) -> List[str]:
    # 如果source_config包含config字段，使用config部分
    if "config" in source_config:
        source_config = source_config["config"]
    
    # 原有逻辑...
    custom_stocks = source_config.get("custom_stocks", [])
    ...
```

## 建议实现

**推荐方案1**，因为：
1. 保持数据结构一致性
2. 工作器只关心配置部分，不需要整个数据源对象
3. 减少不必要的数据传递
4. 符合单一职责原则
