# 特征存储仪表盘修复计划

## 问题诊断

### 问题描述
特征提取任务 `feature_task_single_002837_1771753907` 完成后，PostgreSQL 库 `feature_store` 表未有特征存储数据。

### 根本原因分析

经过代码审查，发现以下关键问题：

#### 1. `feature_store` 表未创建
**位置**: `src/gateway/web/feature_task_persistence.py` 中的 `save_features_to_store` 函数

**问题**: 
- `save_features_to_store` 函数直接执行 INSERT 操作，假设 `feature_store` 表已存在
- 但代码中没有创建 `feature_store` 表的逻辑
- 当表不存在时，INSERT 操作会失败

**代码片段** (第544-555行):
```python
cursor.execute("""
    INSERT INTO feature_store 
    (feature_id, task_id, feature_name, feature_type, parameters, symbol, quality_score)
    VALUES (%s, %s, %s, %s, %s, %s, %s)
    ON CONFLICT (feature_id) DO UPDATE SET
        feature_type = EXCLUDED.feature_type,
        parameters = EXCLUDED.parameters,
        quality_score = EXCLUDED.quality_score,
        updated_at = CURRENT_TIMESTAMP
""", ...)
```

#### 2. 错误处理不完善
**位置**: `src/gateway/web/feature_task_executor.py` 中的 `_execute_task` 方法

**问题**:
- 虽然代码中调用了 `save_features_to_store`，但异常捕获可能掩盖了表不存在的错误
- 日志中可能只显示"保存特征到存储表失败"，而没有具体错误信息

#### 3. 缺少数据库迁移脚本
**位置**: `scripts/database/`

**问题**:
- 没有发现创建 `feature_store` 表的 SQL 迁移脚本
- 与 `feature_engineering_tasks` 表不同（该表在 `_save_to_postgresql` 函数中自动创建）

## 修复方案

### 方案概述
采用**方案3（创建单独的特征存储表）**的修复策略，符合特征层架构设计。

### 修复步骤

#### 步骤1: 创建数据库迁移脚本
**文件**: `scripts/database/create_feature_store_table.sql`

创建 `feature_store` 表，包含以下字段：
- `feature_id`: 主键，格式为 `{task_id}_{feature_name}`
- `task_id`: 关联的特征工程任务ID
- `feature_name`: 特征名称（如 SMA_5, EMA_10）
- `feature_type`: 特征类型（如 SMA, EMA, RSI, MACD）
- `parameters`: JSONB 类型，存储特征参数（如 {"period": 5}）
- `symbol`: 股票代码
- `quality_score`: 特征质量评分
- `importance`: 特征重要性
- `created_at`: 创建时间
- `updated_at`: 更新时间

#### 步骤2: 修改 `save_features_to_store` 函数
**文件**: `src/gateway/web/feature_task_persistence.py`

修改内容：
1. 在插入数据前，自动创建 `feature_store` 表（如果不存在）
2. 添加更详细的错误日志
3. 确保表结构与查询兼容

#### 步骤3: 验证修复
1. 执行数据库迁移脚本创建表
2. 创建新的特征提取任务
3. 验证任务完成后特征数据是否正确保存到 `feature_store` 表
4. 验证前端特征存储仪表盘是否正确显示特征名称
