## 检查方法

### API数据验证
1. 调用后端API获取原始数据
2. 对比页面显示值与API返回值
3. 验证数据转换逻辑正确

### 功能测试
1. 刷新按钮功能
2. 分页功能
3. 搜索功能
4. Tab切换功能
5. 图表交互功能

### 边界情况
1. 空数据状态
2. 加载中状态
3. 错误状态
4. 大量数据分页

## 检查步骤

1. **准备工作**
   - 确保应用正常运行
   - 准备测试数据
   - 打开浏览器开发者工具

2. **逐个检查仪表盘**
   - 按照上述检查范围逐项验证
   - 记录发现的问题
   - 截图保存异常情况

3. **数据一致性验证**
   - 对比API返回数据与页面显示
   - 验证计算逻辑正确
   - 检查数据刷新机制

4. **功能测试**
   - 测试所有交互功能
   - 验证边界情况处理
   - 检查错误处理

5. **问题修复**
   - 修复发现的数据问题
   - 修复显示问题
   - 重新验证修复结果

## 预期结果

- 所有仪表盘数据显示正确
- 数据与API返回值一致
- 交互功能正常
- 边界情况处理得当
- 页面性能良好

## 输出物

1. 检查报告（问题列表）
2. 修复记录
3. 验证结果
