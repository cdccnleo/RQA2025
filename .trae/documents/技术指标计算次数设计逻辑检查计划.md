# 技术指标计算次数设计逻辑检查计划

## 目标
全面深入检查 feature-engineering-monitor 页面技术指标计算状态仪表盘中，技术指标计算次数的合理设计逻辑与代码实现。

## 背景
当前系统中，技术指标计算次数的统计涉及多个组件：
1. **前端展示**: feature-engineering-monitor.html 页面显示计算次数
2. **API服务**: feature_engineering_routes.py 提供 /features/engineering/indicators 接口
3. **业务逻辑**: feature_engineering_service.py 的 get_technical_indicators() 函数
4. **数据跟踪**: indicator_calculation_tracker.py 记录真实计算次数
5. **持久化存储**: data/indicator_calculations.json 文件存储历史数据

## 检查维度

### 1. 数据流完整性检查
**检查内容**:
- 指标计算次数从产生到展示的完整数据流
- 每个环节的数据转换和传递是否正确
- 是否存在数据丢失或重复计算

**涉及文件**:
- worker_executor.py (记录计算次数)
- indicator_calculation_tracker.py (跟踪器)
- feature_engineering_service.py (服务层)
- feature_engineering_routes.py (API层)
- feature-engineering-monitor.html (前端)

### 2. 计算逻辑合理性检查
**检查内容**:
- 什么情况下应该计为一次"计算"
- 是按任务计数还是按实际指标生成计数
- 批量任务和单任务的计数方式是否一致
- 失败任务是否应该计入

**关键问题**:
- 当前逻辑: 每次调用 record_calculation() 计数+1
- 是否合理: 一个任务包含6个指标，应该计为6次还是1次
- 是否需要区分"任务次数"和"指标计算次数"

### 3. 数据一致性检查
**检查内容**:
- indicator_calculations.json 中的数据与任务实际执行情况是否一致
- 不同来源的数据（任务表 vs 跟踪器）是否一致
- 持久化数据与内存数据是否同步

**验证方法**:
- 对比任务表中的 completed 任务数
- 对比 indicator_calculations.json 中的计数
- 检查是否有遗漏或重复记录

### 4. 业务场景覆盖检查
**检查内容**:
- 单只股票任务: 6个指标 × 1只股票 = 6次计算？
- 批量股票任务: 6个指标 × N只股票 = 6N次计算？
- 多时间周期任务: 如何计数
- 重试机制: 失败重试是否重复计数

### 5. 前端展示逻辑检查
**检查内容**:
- 前端如何获取和展示计算次数
- 是否有过滤或聚合逻辑
- 显示格式是否符合用户预期

**涉及代码**:
- feature-engineering-monitor.html 中的 renderIndicatorsStatus()
- API 响应数据的处理

## 实施步骤

### 步骤 1: 数据流追踪
1. 从 worker_executor.py 开始，追踪 record_calculation 调用

2. 评估是否符合业务需求
3. 识别潜在的计数偏差
4. 提出优化建议

### 步骤 3: 数据一致性验证
1. 查询数据库中的任务完成情况
2. 对比 indicator_calculations.json 文件
3. 检查是否存在数据不一致
4. 分析不一致的原因

### 步骤 4: 业务场景测试
1. 单只股票任务测试
2. 批量股票任务测试
3. 验证计数是否符合预期

### 步骤 5: 前端展示验证
1. 检查前端获取数据的 API 调用
2. 验证展示逻辑是否正确
3. 确认用户体验是否符合预期
