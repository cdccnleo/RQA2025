# 技术指标计算次数实现逻辑检查计划

## 目标
检查 feature-engineering-monitor 页面技术指标计算状态仪表盘的实现逻辑与结果是否正确，确保在 PostgreSQL 库表 feature_engineering_tasks 中无任务时，仪表盘显示正确。

## 背景
- PostgreSQL 库表 `feature_engineering_tasks` 中已无任务
- 需要验证技术指标计算次数的统计逻辑是否正确
- 需要确保仪表盘显示的数据与实际计算记录一致

## 检查维度

### 1. 数据源检查
**检查内容**:
- 确认 `indicator_calculations.json` 文件中的计算记录
- 检查 `indicator_calculation_tracker.py` 中的数据加载逻辑
- 验证内存数据与持久化文件的一致性

**涉及文件**:
- `data/indicator_calculations.json`
- `src/features/monitoring/indicator_calculation_tracker.py`

### 2. 计算逻辑检查
**检查内容**:
- 什么情况下应该计为一次"计算"
- 计算次数的统计方式是否合理
- 是否需要区分"任务次数"和"指标计算次数"

**关键问题**:
- 当前逻辑: 每次调用 `record_calculation()` 计数+1
- 是否合理: 一个任务包含6个指标，应该计为6次还是1次
- 无任务时，计算次数是否应该清零或保持历史记录

### 3. 前端展示逻辑检查
**检查内容**:
- 前端如何获取和展示计算次数
- 是否有过滤或聚合逻辑
- 显示格式是否符合用户预期

**涉及代码**:
- `web-static/feature-engineering-monitor.html`
- `src/gateway/web/feature_engineering_service.py`
- `src/gateway/web/feature_engineering_routes.py`

### 4. 数据一致性检查
**检查内容**:
- `indicator_calculations.json` 中的数据与实际计算记录是否一致
- 不同来源的数据（任务表 vs 跟踪器）是否一致
- 持久化数据与内存数据是否同步

## 实施步骤

### 步骤 1: 检查持久化文件数据
1. 读取 `data/indicator_calculations.json` 文件
2. 统计各指标的计算次数
3. 检查数据格式和完整性

### 步骤 2: 检查跟踪器实现逻辑
1. 检查 `indicator_calculation_tracker.py` 的加载逻辑
2. 验证 `get_all_indicators_status()` 方法
3. 检查数据同步机制

### 步骤 3: 检查前端展示逻辑
1. 检查 `feature-engineering-monitor.html` 的数据获取
2. 验证 `get_technical_indicators()` 服务方法
3. 检查 API 响应数据的完整性

### 步骤 4: 验证数据一致性
1. 对比内存数据与文件数据
2. 检查是否有遗漏或重复记录
3. 验证无任务时的显示逻辑

### 步骤 5: 测试验证
1. 访问 feature-engineering-monitor 页面
2. 检查技术指标计算次数显示
3. 验证数据准确性和一致性

## 预期结果
- 明确计算次数的合理定义
- 识别当前实现的问题（如果有）
- 提出改进建议（如果需要）
- 确保数据准确性和一致性

## 交付物
1. 数据检查报告: 持久化文件和内存数据的统计
2. 逻辑分析报告: 当前计算逻辑的评估
3. 一致性检查报告: 各数据源对比结果
4. 改进建议: 优化方案（如果需要）
