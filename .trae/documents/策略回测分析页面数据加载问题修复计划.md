# 策略回测分析页面数据加载问题修复计划

## 问题分析

经过深度检查，我发现：

1. **后端API正常工作**：`/api/v1/backtest` 端点能够正常响应请求并返回完整的回测数据，包含10个回测任务的详细信息。

2. **Docker容器状态正常**：所有容器都在运行状态，应用容器显示为 "Up About an hour (healthy)"。

3. **前端代码已实现**：前端已实现 `loadBacktestStats()`、`loadPerformanceMetrics()` 和 `loadBacktestCharts()` 函数，但可能存在以下问题：
   - API调用路径错误
   - CORS跨域问题
   - 初始化顺序问题
   - 错误处理不完善

## 修复方案

### 1. 检查并修复前端API调用路径

**问题**：前端可能使用了错误的API路径，导致无法连接到后端API。

**解决方案**：
- 检查 `getApiBaseUrl()` 函数的实现，确保返回正确的API基础路径。
- 验证前端调用的API端点与后端API端点匹配。

### 2. 检查并修复CORS配置

**问题**：前端和后端可能存在跨域问题，导致API调用被浏览器阻止。

**解决方案**：
- 检查后端API的CORS配置，确保允许前端域名访问。
- 在后端API响应中添加正确的CORS头。

### 3. 检查并修复前端初始化顺序

**问题**：前端可能在DOM加载完成前就尝试调用API，导致无法找到DOM元素。

**解决方案**：
- 确保前端在DOM加载完成后再调用API。
- 检查 `document.addEventListener('DOMContentLoaded', ...)` 中的代码执行顺序。

### 4. 增强前端错误处理

**问题**：前端可能在API调用失败时没有正确处理错误，导致UI没有更新。

**解决方案**：
- 在前端API调用中添加更多的错误处理代码。
- 在控制台输出详细的错误信息，以便于定位问题。
- 确保在API调用失败时能够正确更新UI，显示错误信息。

### 5. 添加调试信息

**问题**：前端代码缺乏足够的调试信息，导致难以定位问题。

**解决方案**：
- 在前端代码中添加更多的调试信息，包括API调用路径、响应状态等。
- 使用浏览器开发者工具的网络面板和控制台，查看API调用的详细信息。

## 实施步骤

1. **检查前端API调用路径**：
   - 查看 `getApiBaseUrl()` 函数的实现。
   - 验证前端调用的API端点与后端API端点匹配。

2. **检查CORS配置**：
   - 查看后端API的CORS配置。
   - 确保允许前端域名访问。

3. **检查前端初始化顺序**：
   - 查看 `document.addEventListener('DOMContentLoaded', ...)` 中的代码执行顺序。
   - 确保在DOM加载完成后再调用API。

4. **增强前端错误处理**：
   - 在前端API调用中添加更多的错误处理代码。
   - 在控制台输出详细的错误信息。

5. **测试修复效果**：
   - 重新加载策略回测分析页面，查看数据是否能够正常加载。
   - 使用浏览器开发者工具的网络面板和控制台，查看API调用的详细信息。

修复后，策略回测分析页面的所有仪表盘应该能够正常加载数据，包括：
