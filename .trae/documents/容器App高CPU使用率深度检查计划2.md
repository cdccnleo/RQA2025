# 容器App高CPU使用率深度检查计划（第二阶段）

## 当前状态

CPU使用率从147.99%降低到65-87%，但仍偏高，需要进一步检查和优化。

## 检查目标

1. 识别剩余的高CPU消耗来源
2. 检查工作线程数配置是否合理
3. 检查监控任务的执行频率
4. 优化其他可能导致CPU占用的代码

## 检查步骤

### 步骤1: 检查EventBus工作线程数配置
- 检查 `max_workers` 配置
- 检查 `_process_events` 线程数
- 检查 `_batch_processor` 线程数

### 步骤2: 检查监控服务配置
- 检查 `monitoring_interval` 配置
- 检查 `collection_interval` 配置
- 检查监控任务的执行频率

### 步骤3: 检查异步任务和定时任务
- 检查 `asyncio.create_task` 调用
- 检查定时任务的执行频率
- 检查是否有重复的任务调度

### 步骤4: 检查数据库查询频率
- 检查是否有高频数据库查询
- 检查连接池配置
- 检查查询优化

### 步骤5: 检查日志和调试输出
- 检查是否有高频日志输出
- 检查调试信息是否关闭
- 检查日志级别配置

### 步骤6: 检查WebSocket连接
- 检查WebSocket连接数
- 检查心跳频率
- 检查连接管理

## 预期结果

1. CPU使用率降低到50%以下
2. 内存使用稳定在合理范围
3. 进程数控制在合理范围

## 验证方法

1. 使用 `docker stats` 持续监控CPU使用率
2. 观察日志输出频率
3. 测试系统响应时间
