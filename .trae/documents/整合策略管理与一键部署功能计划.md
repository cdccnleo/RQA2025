# 整合策略管理与一键部署功能计划

## 背景

当前系统有两个独立的功能模块：
1. **策略管理页面** (`strategy-management.html`): 提供策略生命周期管理和一键执行工作流（回测→优化→应用）
2. **一键式策略部署模块** (`one_click_strategy_deployment.py`): 提供策略打包、环境部署、版本管理

两个模块目前无直接关联，需要整合形成完整的"研发→验证→发布"闭环。

## 目标

整合两个功能，实现：
1. 在策略管理页面添加"发布"按钮，调用部署模块API
2. 部署前自动执行一键工作流，确保策略已验证
3. 在部署模块中记录策略版本，关联优化历史

## 计划详情

### Phase 1: 后端API扩展

#### 1.1 创建部署API路由
- **文件**: `src/gateway/web/deployment_routes.py` (新建)
- **功能**:
  - `POST /api/v1/strategy/{strategy_id}/deploy` - 部署策略到指定环境
  - `GET /api/v1/strategy/{strategy_id}/deployments` - 获取策略部署历史
  - `POST /api/v1/deployments/{deployment_id}/rollback` - 回滚部署
  - `GET /api/v1/deployments/{deployment_id}` - 获取部署详情
  - `POST /api/v1/strategy/{strategy_id}/publish` - 一键发布（验证+部署）

#### 1.2 扩展一键部署模块
- **文件**: `src/gateway/web/one_click_strategy_deployment.py` (修改)
- **新增功能**:
  - 集成策略验证流程（调用回测和优化API）
  - 记录策略版本和优化历史到部署元数据
  - 添加异步部署支持（后台任务）
  - 部署状态持久化到数据库

#### 1.3 数据库模型扩展
- **文件**: `src/infrastructure/persistence/models/deployment_models.py` (新建)
- **模型**:
  - `DeploymentRecord` - 部署记录表
  - `StrategyVersion` - 策略版本表（关联优化历史）

### Phase 2: 前端页面增强

#### 2.1 策略管理页面扩展
- **文件**: `web-static/strategy-management.html` (修改)
- **新增功能**:

  - 添加"发布历史"弹窗
  - 添加"一键发布"流程（验证+部署）
  - 部署进度实时监控
  - 环境选择（开发/测试/生产）

#### 2.2 创建部署监控组件

- **功能**:
  - WebSocket连接实时获取部署进度

  - 错误处理和重试机制

## 技术方案

### 部署流程设计

```
用户点击"发布"按钮
        ↓
选择目标环境（dev/staging/prod）
        ↓
[可选] 执行验证流程（回测→优化）
        ↓
打包策略（代码+配置+依赖）
        ↓
上传到目标环境
        ↓
安装依赖
        ↓
运行测试
        ↓
更新部署状态
        ↓
记录部署历史
```

### 数据流设计

```
策略管理页面
      ↓ HTTP API
部署路由 (deployment_routes.py)
      ↓ 调用
一键部署模块 (one_click_strategy_deployment.py)
      ↓ 读写
数据库 (DeploymentRecord, StrategyVersion)
```

## 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 部署失败 | 中 | 实现自动回滚机制 |
| 并发部署冲突 | 低 | 添加部署锁机制 |
| 大文件传输超时 | 中 | 分片上传+进度通知 |
| 环境配置错误 | 高 | 部署前环境检查 |

## 验收标准

1. ✅ 策略管理页面显示"发布"按钮
2. ✅ 点击发布弹出环境选择对话框
3. ✅ 部署进度实时显示
4. ✅ 部署成功后在部署历史中可查
5. ✅ 支持回滚到上一版本
6. ✅ 部署前自动验证策略（可选）
7. ✅ 部署记录包含策略版本和优化历史
