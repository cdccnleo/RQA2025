# 监控页面重构验收检查计划

## 检查目标

验证 `web-static/dashboard.html` 的前端展示是否与重构计划一致，确保标签页布局、链接优化、性能优化等功能正常工作。

## 检查项目

### 1. 标签页导航检查

#### 1.1 标签页存在性
- [ ] 页面顶部存在标签页导航栏
- [ ] 包含6个标签页：系统总览、策略监控、交易监控、数据监控、告警中心、架构监控
- [ ] 标签页有对应的图标

#### 1.2 标签页样式
- [ ] 活动标签页有下划线高亮
- [ ] 标签页有悬停效果
- [ ] 响应式布局：移动端只显示图标

#### 1.3 标签页功能
- [ ] 点击标签页可以切换内容
- [ ] 切换时有动画效果
- [ ] 支持键盘导航（Ctrl+左右箭头）

### 2. 内容区域检查

#### 2.1 系统总览标签页
- [ ] 显示关键指标卡片（活跃策略、今日收益、数据延迟、风险等级）
- [ ] 指标数据正常加载

#### 2.2 其他标签页
- [ ] 首次切换时显示加载动画
- [ ] 内容正确加载
- [ ] 已加载标签页切换时无需重新加载

### 3. 链接优化检查

#### 3.1 新窗口打开
- [ ] 仪表盘卡片链接有 `target="_blank"`
- [ ] 链接有 `rel="noopener noreferrer"`
- [ ] 点击后在新窗口打开

#### 3.2 视觉提示
- [ ] 卡片上有外部链接图标
- [ ] 悬停时图标可见

### 4. 性能优化检查

#### 4.1 加载性能
- [ ] 首页加载时间 < 3秒
- [ ] 仅加载当前标签页内容

#### 4.2 缓存机制
- [ ] 已加载标签页内容被缓存
- [ ] 切换回已加载标签页时无网络请求

### 5. 刷新功能检查

#### 5.1 刷新按钮
- [ ] 刷新按钮存在
- [ ] 点击后仅刷新当前标签页
- [ ] 有旋转动画反馈

## 检查方法

1. **视觉检查**：访问页面，逐项核对上述检查点
2. **开发者工具检查**：
   - 检查元素是否存在
   - 检查样式是否正确应用
   - 检查网络请求
3. **功能测试**：
   - 点击标签页测试切换
   - 点击卡片测试新窗口打开
   - 测试键盘导航

## 验收标准

- 所有检查项通过
- 页面加载流畅，无卡顿
- 用户体验符合预期

## 时间估算

- 检查时间：30分钟
- 问题修复：视情况而定
