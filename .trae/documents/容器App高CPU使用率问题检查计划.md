# 容器App高CPU使用率问题检查计划

## 问题描述

容器App运行CPU使用率持续在100%以上，需要全面深入检查原因并进行优化。

## 检查目标

1. 识别导致CPU使用率100%以上的进程/线程
2. 分析代码中的CPU密集型操作
3. 检查是否存在无限循环或频繁轮询
4. 优化或修复高CPU消耗的代码

## 检查步骤

### 步骤1: 检查容器内进程状态
- 使用 `docker exec` 进入容器
- 使用 `top`、`ps`、`htop` 等命令查看进程CPU使用情况
- 识别CPU使用率最高的进程

### 步骤2: 检查Python进程详情
- 使用 `py-spy` 或 `pyflame` 进行Python性能分析
- 检查Python线程状态
- 识别CPU密集型函数

### 步骤3: 检查代码中的潜在问题
- 搜索无限循环 (`while True`)
- 搜索频繁轮询 (`time.sleep` 间隔过小)
- 检查后台任务和定时任务
- 检查异步任务是否有阻塞操作

### 步骤4: 检查日志输出
- 分析日志输出频率
- 检查是否有高频日志输出
- 检查是否有异常循环

### 步骤5: 检查数据库连接
- 检查是否有数据库连接泄漏
- 检查是否有频繁的查询操作
- 检查连接池配置

### 步骤6: 检查第三方库
- 检查是否有第三方库的问题
- 检查库的版本是否有已知问题

## 预期结果

1. 识别导致高CPU使用率的具体原因
2. 优化或修复相关代码
3. CPU使用率降低到合理范围（< 50%）

## 验证方法

1. 使用 `docker stats` 监控CPU使用率
2. 使用 `top` 命令查看进程状态
3. 观察日志输出频率
4. 测试系统响应时间

## 可能的原因

1. **无限循环**：代码中存在没有正确退出的循环
2. **频繁轮询**：定时任务间隔过短
3. **日志风暴**：高频日志输出
4. **数据库问题**：频繁的查询或连接问题
5. **异步任务阻塞**：异步代码中的同步操作
6. **第三方库问题**：某些库的高CPU消耗
