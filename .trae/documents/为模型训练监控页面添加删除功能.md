# 为模型训练监控页面添加删除功能

## 问题分析
- 特征工程监控页面 (`feature-engineering-monitor.html`) 已实现删除任务功能
- 模型训练监控页面 (`model-training-monitor.html`) 缺少删除任务功能
- 模型训练 API 路由 (`model_training_routes.py`) 缺少删除任务的端点

## 改进方案

### 1. 前端改进 (`model-training-monitor.html`)
- 在训练任务表格的操作列添加删除按钮
- 添加 `deleteJob` 函数，参考特征工程的 `deleteTask` 函数实现
- 实现删除确认对话框和错误处理
- 确保删除后刷新任务列表

### 2. 后端改进 (`model_training_routes.py`)
- 添加 `DELETE /ml/training/jobs/{job_id}` API 端点
- 实现删除逻辑，包括：
  - 从持久化存储中删除任务
  - 发布删除事件到 EventBus
  - 通过 WebSocket 广播删除事件
- 添加适当的错误处理和日志记录

### 3. 实现细节
- 前端删除按钮使用与特征工程页面相同的样式和图标
- 后端删除逻辑参考特征工程的实现方式
- 确保删除功能支持所有状态的训练任务
- 添加适当的权限检查和错误处理

## 预期效果
- 模型训练监控页面将具有与特征工程监控页面相同的删除功能
- 用户可以删除不需要的训练任务，保持任务列表的整洁
- 删除操作将实时反映在界面上，通过 WebSocket 广播更新
- 所有删除操作都将被记录到日志中，便于审计
