## 二、严重问题详情

### 问题: 特征工程任务生成的特征数据未被使用

**现象**:
- 用户在创建训练任务时选择特征工程任务 (data_source='features')
- 前端正确传递 `feature_task_id` 参数
- API正确获取特征数据并存储到 `config['feature_data']`
- 但训练执行器 **完全忽略了这些配置**

**根本原因**:
在 `training_job_executor.py` 的 `_execute_training` 方法中:
```python
# 第510行 - 直接从数据库加载原始数据，忽略 feature_task_id
training_data = await self._load_training_data(config)
```

训练执行器应该:
1. 检查配置中是否存在 `feature_task_id`
2. 如果存在，调用 `get_feature_data_for_training()` 获取特征数据
3. 如果不存在，则回退到从数据库加载原始数据

**影响**:
- 用户选择特征工程任务后，实际上使用的是原始历史数据
- 特征工程任务生成的特征数据被完全忽略
- 违反了量化交易系统的数据处理流程设计

---

## 三、修复建议

### 修复方案

需要在 `training_job_executor.py` 的 `_execute_training` 方法中添加特征数据加载逻辑:

```python
# 在第509行之后添加:
# 检查是否使用特征工程任务的特征数据
feature_task_id = config.get('feature_task_id')
if feature_task_id:
    logger.info(f"从特征工程任务 {feature_task_id} 加载特征数据...")
    try:
        from .feature_engineering_service import get_feature_data_for_training
        feature_result = get_feature_data_for_training(feature_task_id)
        
        if "error" not in feature_result:
            # 使用特征数据
            training_data = feature_result.get("data")
            logger.info(f"成功加载特征数据: {feature_result.get('shape')}")
        else:
            logger.warning(f"获取特征数据失败: {feature_result['error']}，回退到原始数据")
            training_data = await self._load_training_data(config)
    except Exception as e:
        logger.warning(f"加载特征数据异常: {e}，回退到原始数据")
        training_data = await self._load_training_data(config)
else:
    # 使用原始历史数据
    training_data = await self._load_training_data(config)
```
