# 为模型训练监控页面增加调度器启动监控面板

## 分析结果

经过对比分析，我发现：

### 特征工程监控页面 (`feature-engineering-monitor.html`)

* ✅ 包含完整的调度器状态监控面板

* ✅ 显示调度器运行状态（运行中/已停止）

* ✅ 显示活跃工作节点数和队列大小

* ✅ 显示任务统计（总任务数、已完成、失败、等待中）

* ✅ 提供启动/停止调度器的按钮

* ✅ 包含完整的调度器状态加载和管理逻辑

### 模型训练监控页面 (`model-training-monitor.html`)

* ❌ 缺少调度器状态监控面板

* ❌ 没有调度器启动/停止功能

* ❌ 无法查看调度器的详细状态和统计信息

## 实施计划

### 1. 在模型训练监控页面添加调度器状态监控面板

* 在统计卡片下方添加调度器状态卡片

* 包含调度器状态显示、活跃工作节点、队列大小等信息

* 添加启动/停止调度器的按钮

### 2. 实现调度器状态加载和管理逻辑

* 添加 `loadSchedulerStatus()` 函数

* 添加 `updateSchedulerStatus()` 函数

* 添加 `startScheduler()` 函数

* 添加 `stopScheduler()` 函数

* 更新 `refreshData()` 函数，包含调度器状态刷新

### 3. 确保后端API支持

* 验证 `/ml/training/scheduler/status` API 端点存在

* 验证 `/ml/training/scheduler/start` API 端点存在

* 验证 `/ml/training/scheduler/stop` API 端点存在

### 4. 测试和验证

* 测试调度器状态加载功能

* 测试启动/停止调度器功能

* 验证调度器状态更新和显示

* 确保与特征工程监控页面的调度器管理功能保持一致

## 预期效果

* 模型训练监控页面将具有与特征工程监控页面相同的调度器管理功能

* 用户可以方便地查看和管理模型训练任务的调度器状态

* 提高系统的可管理性和用户体验

* 保持两个监控页面的功能一致性
