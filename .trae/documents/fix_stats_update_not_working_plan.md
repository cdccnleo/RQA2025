# 修复回测次数和优化次数未更新问题计划

## 问题描述
一键执行完整流程完成后，回测次数和优化次数依然未更新。

## 诊断目标
1. 检查前端统计更新函数是否被正确调用
2. 检查API请求是否成功
3. 检查后端是否正确保存统计数据
4. 检查数据库是否正确更新
5. 检查前端是否正确显示统计数据

## 检查范围

### 1. 前端调用链
- 回测完成后是否调用 `updateStrategyBacktestStats`
- 应用优化后是否调用 `updateStrategyOptimizationStats`
- 函数参数是否正确传递

### 2. API请求
- GET `/strategy/conceptions/{id}` 是否成功
- PUT `/strategy/conceptions/{id}` 是否成功
- 请求体格式是否正确

### 3. 后端处理
- 后端是否正确接收和处理PUT请求
- 是否正确更新stats字段
- 是否正确保存到数据库

### 4. 数据验证
- 检查PostgreSQL中的数据
- 检查文件系统中的数据
- 验证数据一致性

## 实施步骤

### 第一阶段：前端调试（30分钟）
1. 添加详细日志到统计更新函数
2. 检查函数调用时机
3. 验证API请求参数
4. 检查响应结果

### 第二阶段：后端调试（30分钟）
1. 检查策略更新API实现
2. 添加详细日志到保存逻辑
3. 验证stats字段处理
4. 检查数据库更新

### 第三阶段：数据验证（20分钟）
1. 检查PostgreSQL数据
2. 检查文件系统数据
3. 验证数据一致性
4. 测试手动更新

### 第四阶段：修复和测试（30分钟）
1. 修复发现的问题
2. 测试完整流程
3. 验证统计数据更新
4. 验证前端显示

## 预期产出
1. 问题根因分析报告
2. 修复后的代码
3. 测试验证报告
